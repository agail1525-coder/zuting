## 1. 认证体系

### 1.1 JWT Bearer Token

- 所有受保护端点使用 **JWT Bearer Token** 认证。
- Token 格式: `Authorization: Bearer <access_token>`
- 算法: HS256 (开发) / RS256 (生产推荐)
- JWT Payload 必须包含: `sub` (userId)、`roles`、`iat`、`exp`

### 1.2 Access Token / Refresh Token 双 Token 策略

| Token 类型 | 有效期 | 存储位置 | 说明 |
|-----------|--------|---------|------|
| Access Token | 15 分钟 | 内存 / Secure Cookie | 短效，用于 API 请求 |
| Refresh Token | 7 天 | HttpOnly Secure Cookie | 长效，仅用于换取新 Access Token |

### 1.3 Token 过期策略

```
Access Token 过期 → 前端静默用 Refresh Token 换新 Access Token
Refresh Token 过期 → 强制跳转登录页，清除本地状态
Refresh Token 单次使用 (Rotation) → 换新后旧 Token 立即失效
```

### 1.4 安全存储规范

- **Web**: Access Token 存 Memory/Redux；Refresh Token 存 `HttpOnly; Secure; SameSite=Strict` Cookie
- **Mobile (React Native)**: 使用 `expo-secure-store` 存储 Token，禁止存 `AsyncStorage`
- **小程序**: 使用微信 `wx.setStorageSync` 加密存储，勿明文写入
- **Admin**: 同 Web 规范，额外要求 Token 绑定 IP (可选)

## 2. 授权模型

> 铁律 [R-63] [R-74]

### 2.1 Guard 体系

```typescript
// 所有 Controller 默认受保护
@UseGuards(JwtAuthGuard, RolesGuard)
@Controller('xxx')
export class XxxController {}

// 公开路由使用 @Public() 标记
@Public()
@Get('list')
findAll() {}

// 角色受限路由
@Roles(Role.ADMIN)
@UseGuards(JwtAuthGuard, RolesGuard)   // [R-74] @Roles() 必须配合 @UseGuards
@Delete(':id')
remove(@Param('id') id: string) {}
```

### 2.2 角色定义

| 角色 | 权限范围 | 说明 |
|-----|---------|------|
| `USER` | 自己的行程/订单/日志 CRUD，浏览公开内容 | 普通注册用户 |
| `ADMIN` | 全部内容 CRUD，用户管理，数据统计 | 管理后台操作员 |
| `SUPER_ADMIN` | ADMIN 权限 + 角色分配 + 系统配置 | 最高权限 |

### 2.3 禁止事项

```
❌ Controller 缺少 JwtAuthGuard → 端点无保护
❌ 单独使用 @Roles() 而不搭配 @UseGuards(RolesGuard) → 装饰器无效 [R-74]
❌ 全局关闭认证而非精确标记 @Public()
```

## 3. IDOR 防护

> 铁律 [R-68]: 接受 userId 参数的端点必须验证访问权限，与 JWT 中的 userId 对比

### 3.1 标准写法

```typescript
@Get('my-trips')
async getMyTrips(@Request() req: AuthRequest) {
  // 从 JWT 取 userId，不信任客户端传参
  return this.tripService.findByUserId(req.user.sub);
}

@Patch(':id')
async update(
  @Param('id') id: string,
  @Request() req: AuthRequest,
  @Body() dto: UpdateTripDto,
) {
  // 先查资源，再验证 owner
  const trip = await this.tripService.findOneOrThrow(id);
  if (trip.userId !== req.user.sub) {
    throw new ForbiddenException('无权操作他人行程');
  }
  return this.tripService.update(id, dto);
}
```

### 3.2 检查清单

- [ ] `/trips/:id` — 验证 `trip.userId === jwt.sub`
- [ ] `/orders/:id` — 验证 `order.userId === jwt.sub`
- [ ] `/journals/:id` — 验证 `journal.userId === jwt.sub`
- [ ] 任何 `userId` 路径参数端点 — 必须对比 JWT

## 5. SQL 注入防护

### 5.1 Prisma ORM 天然防护

Prisma 使用参数化查询，所有用户输入通过类型安全的查询对象传递，不存在 SQL 拼接注入风险。

```typescript
// 安全: Prisma 参数化
const sites = await this.prisma.holySite.findMany({
  where: { name: { contains: userInput } },
});

// 禁止: 任何形式的 raw SQL 字符串拼接
// ❌ this.prisma.$queryRaw(`SELECT * FROM holy_site WHERE name = '${userInput}'`)
```

### 5.2 Raw SQL 使用规范

- **禁止**拼接用户输入到 `$queryRaw` / `$executeRaw`
- 必须使用 Prisma 的 `Prisma.sql` 模板标签:

```typescript
// 正确的 raw SQL 写法 (仅在必要时使用)
const result = await this.prisma.$queryRaw(
  Prisma.sql`SELECT * FROM "HolySite" WHERE id = ${siteId}`
);
```

## 6. XSS 防护

### 6.1 后端自动转义

NestJS 默认对 JSON 响应中的 HTML 字符进行转义，无需额外配置。

### 6.2 前端 DOMPurify

凡是将用户提交的富文本/HTML 内容渲染到 DOM，必须通过 DOMPurify 净化:

```typescript
import DOMPurify from 'dompurify';

// 渲染用户内容
const clean = DOMPurify.sanitize(userHtmlContent);
return <div dangerouslySetInnerHTML={{ __html: clean }} />;
```

### 6.3 CSP (Content Security Policy)

在 Helmet 中配置 CSP 头，限制脚本来源:

```typescript
helmet({
  contentSecurityPolicy: {
    directives: {
      defaultSrc: ["'self'"],
      scriptSrc: ["'self'", "'unsafe-inline'"], // 生产环境移除 unsafe-inline
      imgSrc: ["'self'", 'data:', 'https:'],
      connectSrc: ["'self'", 'https://api.joinus.com'],
    },
  },
})
```

## 7. CSRF 防护

### 7.1 SameSite Cookie

所有 Cookie 设置 `SameSite=Strict` 或 `SameSite=Lax`，阻止跨站请求携带 Cookie:

```
Set-Cookie: refreshToken=xxx; HttpOnly; Secure; SameSite=Strict; Path=/api/auth
```

### 7.2 CSRF Token (表单场景)

纯 JSON API 天然对 CSRF 有抵抗力 (浏览器不会跨站发 `Content-Type: application/json`)。
若存在传统表单提交场景，必须使用 CSRF Token:

```typescript
// NestJS csurf 中间件 (表单场景)
import * as csurf from 'csurf';
app.use(csurf({ cookie: { httpOnly: true, sameSite: 'strict' } }));
```

## 8. CORS 配置

> 铁律 [HH-S03]: helmet + CORS白名单 + RateLimiter 三件套

### 8.1 配置规范

```typescript
// main.ts
app.enableCors({
  origin: (origin, callback) => {
    const whitelist = process.env.CORS_ORIGINS?.split(',') ?? [];
    if (!origin || whitelist.includes(origin)) {
      callback(null, true);
    } else {
      callback(new Error(`CORS 拒绝: ${origin}`));
    }
  },
  credentials: true,
  methods: ['GET', 'POST', 'PUT', 'PATCH', 'DELETE', 'OPTIONS'],
  allowedHeaders: ['Content-Type', 'Authorization'],
});
```

### 8.2 环境配置

```bash
# .env.production
CORS_ORIGINS=https://joinus.com,https://www.joinus.com,https://admin.joinus.com

# .env.development
CORS_ORIGINS=http://localhost:3000,http://localhost:3003
```

### 8.3 禁止事项

```
❌ origin: '*' (生产环境全开放)
❌ 允许任意域名携带 credentials
❌ 开发环境配置泄露到生产
```

## 11. findMany 防 OOM

> 铁律 [R-64]: findMany 必须有 take 参数，列表 @Max(100)，防 OOM

### 11.1 强制分页规范

```typescript
// DTO 中强制 pageSize 上限
export class PaginationDto {
  @IsOptional()
  @Type(() => Number)
  @IsInt()
  @Min(1)
  @Max(100)  // [R-64] 最大100条
  pageSize?: number = 20;

  @IsOptional()
  @Type(() => Number)
  @IsInt()
  @Min(1)
  page?: number = 1;
}

// Service 中必须带 take
async findMany(dto: PaginationDto) {
  const take = Math.min(dto.pageSize ?? 20, 100); // 双重保险
  const skip = (dto.page - 1) * take;
  return this.prisma.holySite.findMany({ take, skip });
}
```

### 11.2 禁止无限查询

```typescript
// ❌ 禁止无 take 的 findMany
const all = await this.prisma.holySite.findMany();

// ✅ 必须有 take
const items = await this.prisma.holySite.findMany({ take: 20, skip: 0 });
```

## 12. 外部 HTTP 调用超时

> 铁律 [R-65]: 外部 HTTP 调用必须有超时

### 12.1 NestJS HttpService 超时配置

```typescript
// 全局默认超时
HttpModule.register({
  timeout: 10000,  // 10 秒
  maxRedirects: 3,
}),

// 单次调用覆盖超时 (小鸿 AI，允许更长)
this.httpService.post(url, payload, {
  timeout: 180000,  // 180 秒 (LLM 推理)
  headers: { 'Content-Type': 'application/json' },
});
```

### 12.2 超时场景处理

```typescript
try {
  const response = await firstValueFrom(
    this.httpService.get(url, { timeout: 10000 }),
  );
  return response.data;
} catch (error) {
  if (axios.isAxiosError(error) && error.code === 'ECONNABORTED') {
    throw new GatewayTimeoutException('外部服务超时，请稍后重试');
  }
  throw error;
}
```

## 13. 敏感数据处理

### 13.1 密码哈希

```typescript
import * as bcrypt from 'bcrypt';

const SALT_ROUNDS = 12;

// 注册时
const hashedPassword = await bcrypt.hash(plainPassword, SALT_ROUNDS);

// 登录时
const isValid = await bcrypt.compare(plainPassword, hashedPassword);
```

### 13.2 响应脱敏

```typescript
// User 实体排除密码字段
@Exclude()
password: string;

// Controller 使用 ClassSerializerInterceptor
@UseInterceptors(ClassSerializerInterceptor)
@Get('profile')
getProfile(@Request() req) {
  return req.user; // password 字段自动排除
}
```

### 13.3 日志脱敏

```typescript
// 禁止在日志中输出敏感字段
// ❌ this.logger.log(`用户登录: ${JSON.stringify(loginDto)}`);
// ✅ this.logger.log(`用户登录: userId=${userId}, email=${maskEmail(email)}`);

function maskEmail(email: string): string {
  const [name, domain] = email.split('@');
  return `${name.slice(0, 2)}***@${domain}`;
}
```

### 13.4 敏感字段清单

以下字段禁止出现在 API 响应和日志中:

| 字段 | 处理方式 |
|-----|---------|
| `password` | `@Exclude()` 序列化排除 |
| `refreshToken` | 不存数据库，或存哈希 |
| JWT Secret | 仅环境变量 |
| 支付卡号 | 仅显示后四位 |
| 手机号 | 中间四位脱敏: `138****8888` |

## 14. 安全头 (Helmet)

### 14.1 完整配置

```typescript
app.use(
  helmet({
    // 防止点击劫持 (iframe 嵌套)
    frameguard: { action: 'deny' },

    // 禁止 MIME 类型嗅探
    noSniff: true,

    // HSTS: 强制 HTTPS (生产环境)
    hsts: {
      maxAge: 31536000,         // 1 年
      includeSubDomains: true,
      preload: true,
    },

    // 禁用旧版 XSS 过滤 (现代浏览器用 CSP)
    xssFilter: true,

    // 隐藏 X-Powered-By
    hidePoweredBy: true,

    // Referrer Policy
    referrerPolicy: { policy: 'strict-origin-when-cross-origin' },

    // CSP (内容安全策略)
    contentSecurityPolicy: {
      directives: {
        defaultSrc: ["'self'"],
        scriptSrc: ["'self'"],
        styleSrc: ["'self'", "'unsafe-inline'"],
        imgSrc: ["'self'", 'data:', 'https:'],
        fontSrc: ["'self'", 'https://fonts.gstatic.com'],
        connectSrc: ["'self'", 'https://api.joinus.com'],
        frameSrc: ["'none'"],
        objectSrc: ["'none'"],
      },
    },
  }),
);
```

### 14.2 安全头检查表

| 响应头 | 推荐值 | 作用 |
|-------|--------|------|
| `X-Frame-Options` | `DENY` | 防点击劫持 |
| `X-Content-Type-Options` | `nosniff` | 防 MIME 嗅探 |
| `Strict-Transport-Security` | `max-age=31536000; includeSubDomains` | 强制 HTTPS |
| `X-XSS-Protection` | `1; mode=block` | 旧版 XSS 过滤 |
| `Referrer-Policy` | `strict-origin-when-cross-origin` | 控制 Referer |
| `Content-Security-Policy` | 见上方配置 | 防 XSS/注入 |
| `Permissions-Policy` | `camera=(), microphone=()` | 禁用危险 API |
