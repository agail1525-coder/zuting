## 1. 总体原则

```
[R-59] Prisma字段名必须查schema.prisma确认，禁止猜测
[R-21] 字段/关系命名: camelCase
[R-22] 枚举值命名: UPPER_SNAKE_CASE
[R-23] 时间字段: ISO8601 格式
[R-24] ID: cuid() 生成，禁止自增
[R-31] Schema变更: prisma db push (开发阶段)
[R-32] 新增字段必须有默认值或允许 null
[R-33] 枚举新增顺序: Schema → 后端DTO → 前端
```

**读 Schema 优先原则**：在任何涉及数据库字段名、类型、关系的代码编写前，必须先阅读 `services/api/prisma/schema.prisma`，以实际文件为准，禁止凭印象或猜测写字段名。

## 2. Prisma Schema 规范

### 2.1 模型命名

| 层级 | 规范 | 示例 |
|------|------|------|
| 模型名 | PascalCase | `User`, `HolySite`, `TripStatusHistory` |
| 字段名 | camelCase | `userId`, `createdAt`, `imageUrl` |
| 关系名 | camelCase | `holySites`, `guideProfile`, `statusHistory` |
| 枚举名 | PascalCase | `TripStatus`, `UserRole`, `SealSeries` |
| 枚举值 | UPPER_SNAKE_CASE | `IN_PROGRESS`, `CROSS_CULTURAL` |

### 2.2 标准模型结构

每个业务模型必须遵守以下字段顺序：

```prisma
model 模型名 {
  // 1. 主键
  id        String   @id @default(cuid())

  // 2. 业务外键（关联到其他模型）
  userId    String
  user      User     @relation(fields: [userId], references: [id])

  // 3. 业务字段（按重要程度排列）
  title     String
  content   String   @db.Text
  status    SomeEnum @default(DEFAULT_VALUE)

  // 4. 可选扩展字段
  imageUrl  String?
  metadata  Json?

  // 5. 时间戳（必须放最后）
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  // 6. 索引（放模型最底部）
  @@index([userId])
  @@index([status])
}
```

### 2.3 @db.Text 使用规则

凡是长文本字段（描述、内容、传记、JSON等）必须加 `@db.Text`，否则 PostgreSQL 会使用 `varchar(191)` 限制。

```prisma
// 正确
description String @db.Text
content     String @db.Text
itinerary   Json   // Json类型不需要 @db.Text

// 错误 — 可能被截断
description String
```

## 3. 字段类型选择

| 类型 | 使用场景 | 注意事项 |
|------|---------|---------|
| `String` | 文本、URL、电话、货币代码、状态字符串 | 长文本必须加 `@db.Text` |
| `Int` | 整数计数、金额（分）、顺序、人数 | 金额统一用**分(cents)**，禁止浮点存钱 |
| `Float` | GPS坐标、评分、UTC偏移 | 不用于金额 |
| `Boolean` | 开关状态、是否已读、是否激活 | 必须有 `@default(false/true)` |
| `DateTime` | 时间戳、日期 | 统一 ISO8601，时区存 UTC |
| `Json` | 非结构化数据、逐日行程、周边信息 | 需在代码层做类型断言，前端用 `unknown` |
| `String[]` | 标签、图片URL列表、认证列表 | PostgreSQL数组，只用于简单字符串集合 |
| `Enum` | 有限状态集合 | 见第6节枚举规范 |

### 3.1 金额字段规范

```
所有金额字段统一以"分（cents）"存储，类型为 Int。
禁止使用 Float 存储金额，避免浮点精度问题。
```

```prisma
// 正确
totalBudget  Int?   // 单位: 分 (¥100.00 = 10000)
priceFrom    Int    // 起价(分)
totalAmount  Int    // 订单总额(分)

// 错误 — 浮点精度不可靠
totalBudget  Float  // 禁止
```

## 4. ID 策略

```
[R-24] ID 必须使用 cuid() 生成，禁止自增 ID（Auto Increment）
```

```prisma
// 标准主键 — 所有业务表
id String @id @default(cuid())

// 例外: Seal 表使用自然数字ID（业务特殊需求，三十印序号）
model Seal {
  id Int @id   // 特殊: 曹溪三十印的自然编号1-30
  ...
}
```

**为什么用 cuid？**
- 全局唯一，避免分布式冲突
- 不暴露记录数量（安全）
- URL友好，可直接用作路由参数

## 5. 时间字段规范

```
[R-23] 时间字段使用 ISO8601 格式，存储 UTC 时间
```

### 5.1 必须字段

所有业务模型**必须包含** `createdAt` 和 `updatedAt`（只读模型如 Session 可仅有 `createdAt`）：

```prisma
createdAt DateTime @default(now())   // 自动设置创建时间
updatedAt DateTime @updatedAt        // Prisma 自动更新修改时间
```

### 5.2 可选时间字段命名规范

| 场景 | 字段名 | 类型 |
|------|--------|------|
| 支付完成时间 | `paidAt` | `DateTime?` |
| 取消时间 | `cancelledAt` | `DateTime?` |
| 退款完成时间 | `refundedAt` | `DateTime?` |
| 最后登录 | `lastLoginAt` | `DateTime?` |
| 已读时间 | `readAt` | `DateTime?` |
| 审核时间 | `reviewedAt` | `DateTime?` |
| 过期时间 | `expiresAt` | `DateTime` |
| 活动结束 | `endAt` | `DateTime` |

规律：**动词过去式 + At**（paidAt, cancelledAt），或 **名词 + At**（expiresAt, startAt）。

## 6. 枚举规范

```
[R-22] 枚举值命名: UPPER_SNAKE_CASE
[R-33] 枚举新增顺序: Schema → 后端DTO → 前端
```

### 6.1 现有枚举总览

| 枚举名 | 用途 | 值数量 |
|--------|------|--------|
| `UserRole` | 用户角色 | 4 |
| `TripStatus` | 行程状态机（12态） | 12 |
| `OrderStatus` | 订单状态 | 5 |
| `SealSeries` | 曹溪三十印五系 | 5 |
| `RouteCategory` | 路线分类 | 9 |
| `RouteDifficulty` | 路线难度 | 3 |
| `RouteStatus` | 路线发布状态 | 3 |
| `BookingStatus` | 路线预订状态 | 5 |

### 6.2 TripStatus — 12态状态机

```prisma
enum TripStatus {
  DRAFT           // 草稿 — 初始状态
  PLANNING        // 规划中
  SUBMITTED       // 已提交 — 用户提交待确认
  CONFIRMED       // 已确认 — 管理员确认
  PAID            // 已支付
  PREPARING       // 准备中 — 出发前准备
  IN_PROGRESS     // 朝圣中
  COMPLETED       // 已完成
  REVIEWING       // 评价中
  CANCELLED       // 已取消（终态）
  REFUNDING       // 退款中
  REFUNDED        // 已退款（终态）
}
```

终态（`COMPLETED` / `CANCELLED` / `REFUNDED`）不可再跳转，由 `TripStateMachine` 强制校验。

### 6.3 新增枚举值的标准流程

```
第1步: 修改 services/api/prisma/schema.prisma，在对应枚举末尾添加新值
第2步: 执行 prisma db push 同步到数据库
第3步: 更新后端 DTO（CreateDto/UpdateDto 中的 enum 字符串数组）
第4步: 更新前端代码中硬编码的枚举值引用
第5步: 更新种子数据（若有相关种子）
第6步: 提交 commit，遵循 [B-01] 铁律
```

## 7. 枚举与 Swagger 的陷阱

> **高频踩坑，务必熟记！**

Prisma 生成的枚举对象会引发 NestJS + Swagger 的 circular dependency 错误，禁止直接传入 `@ApiProperty`。

### 7.1 错误做法

```typescript
// 错误 — 直接传 Prisma enum 给 Swagger 装饰器
import { TripStatus } from '@prisma/client';

@ApiProperty({ enum: TripStatus })  // ❌ 会触发 circular dependency
status: TripStatus;
```

### 7.2 正确做法

```typescript
// 正确 — 用字符串数组代替 Prisma enum
@ApiProperty({
  enum: ['DRAFT', 'PLANNING', 'SUBMITTED', 'CONFIRMED', 'PAID',
         'PREPARING', 'IN_PROGRESS', 'COMPLETED', 'REVIEWING',
         'CANCELLED', 'REFUNDING', 'REFUNDED'],
  example: 'DRAFT',
})
status: string;  // 类型用 string

// 在 Service 层 cast 回枚举
const result = await this.prisma.trip.findMany({
  where: { status: status as TripStatus },
});
```

### 7.3 DTO 中枚举参数的完整示例

```typescript
// trips/dto/create-trip.dto.ts
import { IsString, IsIn } from 'class-validator';

export class TransitionTripDto {
  @ApiProperty({
    description: '目标状态',
    enum: ['PLANNING', 'SUBMITTED', 'CONFIRMED', 'PAID', 'CANCELLED'],
  })
  @IsString()
  @IsIn(['PLANNING', 'SUBMITTED', 'CONFIRMED', 'PAID', 'CANCELLED'])
  toStatus: string;
}
```

## 8. 关系设计规范

### 8.1 @relation 显式命名

凡是一个模型与另一模型存在**多条关系**，必须用命名 `@relation` 区分：

```prisma
// 示例: User 与 ContentReport 有两条关系
model ContentReport {
  reporterId  String
  reporter    User   @relation("ReportsMade",    fields: [reporterId], references: [id])
  reviewedBy  String?
  reviewer    User?  @relation("ReportsReviewed", fields: [reviewedBy], references: [id])
}

// User 模型中对应
model User {
  reportsMade     ContentReport[] @relation("ReportsMade")
  reportsReviewed ContentReport[] @relation("ReportsReviewed")
}
```

### 8.2 onDelete 策略选择

| 策略 | 含义 | 适用场景 |
|------|------|---------|
| `Cascade` | 父删子也删 | 严格归属关系（Session/User, RouteSite/Route, ConversationMessage/Conversation） |
| `SetNull` | 父删子字段置null | 可选关联（Trip.guideId 导游被删后行程保留） |
| `Restrict` | 父有子则拒绝删除 | 重要数据保护（默认行为，不写即为Restrict） |

```prisma
// Cascade 示例 — Session 随 User 删除
model Session {
  user User @relation(fields: [userId], references: [id], onDelete: Cascade)
}

// Cascade 示例 — RouteSite 随 Route 删除
model RouteSite {
  route Route @relation(fields: [routeId], references: [id], onDelete: Cascade)
}

// SetNull 示例 — 导游删除后，行程的 guideId 置 null
model Trip {
  guide GuideProfile? @relation("TripGuide", fields: [guideId], references: [id])
  // guideId 为 String? 允许 null
}
```

### 8.3 可选关联 vs 必选关联

```prisma
// 必选关联 — 外键不为 null
userId String
user   User   @relation(...)

// 可选关联 — 外键为 null（String?）
tripId String?
trip   Trip?  @relation(...)
```

## 9. 索引规范

### 9.1 强制索引场景

| 场景 | 索引类型 | 示例 |
|------|---------|------|
| 外键字段 | `@@index` | `@@index([userId])` |
| 唯一标识符 | `@unique` | `orderNo String @unique` |
| 频繁过滤字段 | `@@index` | `@@index([status])` |
| 时间范围查询 | `@@index` | `@@index([createdAt])` |
| 复合查询 | `@@index` 复合 | `@@index([targetType, targetId])` |
| 防重复提交 | `@@unique` 复合 | `@@unique([userId, targetType, targetId])` |

### 9.2 来自 schema.prisma 的索引示例

```prisma
// Session — 按用户查询 + 过期清理
@@index([userId])
@@index([expiresAt])

// VerificationCode — 按目标+类型查找验证码
@@index([target, type])

// Review — 防止一用户对同目标重复评价
@@unique([userId, targetType, targetId])

// Notification — 按用户+已读状态查未读消息
@@index([userId, isRead])

// CouponUsage — 防止一张券被同一用户对同一订单重复使用
@@unique([couponId, userId, orderId])
```

### 9.3 索引反模式（禁止）

```
- 不要对所有字段建索引（写入性能下降）
- 不要对低基数布尔字段单独建索引（isActive, isRead）
- 不要对 Json 类型字段建索引（PostgreSQL 不支持简单索引）
```

## 10. 默认值规范

```
[R-32] 新增字段必须有默认值或允许 null
```

### 10.1 常见默认值模式

```prisma
// 布尔字段 — 必须有默认值
isActive      Boolean @default(true)
isPublic      Boolean @default(false)
emailVerified Boolean @default(false)

// 计数字段 — 默认为0
likes         Int     @default(0)
usedCount     Int     @default(0)
totalTrips    Int     @default(0)
persons       Int     @default(1)    // 人数默认1人

// 枚举字段 — 指定初始状态
role          UserRole   @default(PILGRIM)
status        TripStatus @default(DRAFT)
status        OrderStatus @default(PENDING)

// 字符串字段 — 默认值或可空
language      String @default("zh-CN")
currency      String @default("CNY")
provider      String @default("local")

// 可选字段 — 加 ? 允许 null
imageUrl      String?
note          String? @db.Text
guideId       String?
```

### 10.2 新增字段清单

每次 Schema 变更新增字段时，必须确认以下三选一：
1. 有明确的 `@default(value)` ✓
2. 类型为可空（加 `?`）✓
3. 有充分的业务理由强制非空且无默认值（需注释说明）✓

## 11. Schema 变更流程

```
[R-31] 开发阶段使用 prisma db push
        生产阶段使用 prisma migrate deploy
```

### 11.1 开发阶段标准流程

```bash
# 第1步: 修改 services/api/prisma/schema.prisma

# 第2步: 推送 Schema 到数据库（开发阶段）
pnpm db:push
# 等价于: cd services/api && npx prisma db push

# 第3步: 重新生成 Prisma Client
npx prisma generate

# 第4步: 若有种子数据变更，重新执行种子
pnpm db:seed

# 第5步: 重启 API 服务（SWC编译，需重启）
pnpm dev:api
```

### 11.2 生产变更流程

```bash
# 生产环境禁止 db push，使用迁移
npx prisma migrate dev --name "add_field_xxx"   # 生成迁移文件
npx prisma migrate deploy                        # 应用迁移
```

### 11.3 破坏性变更处理

| 操作 | 风险 | 处理方式 |
|------|------|---------|
| 删除字段 | 高 | 先废弃（`@deprecated`注释），观察1周后删 |
| 重命名字段 | 高 | 新增字段→迁移数据→删旧字段（3步） |
| 更改字段类型 | 高 | 同上，需数据迁移脚本 |
| 新增枚举值 | 低 | 直接追加到枚举末尾 |
| 删除枚举值 | 高 | 先清空使用该值的数据，再删除 |

## 12. DTO 映射规范

### 12.1 三类 DTO

| DTO类型 | 作用 | 继承方式 |
|---------|------|---------|
| `CreateXxxDto` | 创建请求体校验 | 独立定义，`class-validator` 装饰器 |
| `UpdateXxxDto` | 更新请求体（全部可选） | `PartialType(CreateXxxDto)` |
| `XxxResponseDto` | API 响应格式，排除敏感字段 | 独立定义 |

### 12.2 CreateDto 示例

```typescript
// trips/dto/create-trip.dto.ts
import { IsString, IsOptional, IsInt, IsDateString, Min, Max } from 'class-validator';
import { ApiProperty, ApiPropertyOptional } from '@nestjs/swagger';

export class CreateTripDto {
  @ApiProperty({ description: '行程标题', example: '禅宗朝圣之旅' })
  @IsString()
  title: string;

  @ApiPropertyOptional({ description: '出发日期', example: '2026-04-01T00:00:00Z' })
  @IsOptional()
  @IsDateString()
  startDate?: string;

  @ApiPropertyOptional({ description: '人数', minimum: 1, maximum: 100 })
  @IsOptional()
  @IsInt()
  @Min(1)
  @Max(100)      // [R-64] 防 OOM
  persons?: number;
}
```

### 12.3 UpdateDto 示例

```typescript
// trips/dto/update-trip.dto.ts
import { PartialType } from '@nestjs/swagger';
import { CreateTripDto } from './create-trip.dto';

export class UpdateTripDto extends PartialType(CreateTripDto) {}
// 所有字段自动变为可选，Swagger 文档自动同步
```

### 12.4 ResponseDto 敏感字段排除

以下字段**禁止**出现在响应 DTO 中：

```
passwordHash    — 密码哈希
refreshToken    — 刷新令牌
idNumber        — 导游身份证号
callbackPayload — 支付回调原始数据（含敏感信息）
```

```typescript
// users/dto/user-response.dto.ts
export class UserResponseDto {
  id: string;
  nickname: string;
  avatar?: string;
  role: string;
  email?: string;
  phone?: string;
  createdAt: Date;
  // passwordHash ❌ 不暴露
  // refreshToken ❌ 不暴露
}
```

## 附录A: Schema 变更 Checklist

```

□ 所有字段名为 camelCase
□ 主键使用 cuid()（Seal 除外）

□ onDelete 策略已明确设置（Cascade/SetNull/Restrict）
```
