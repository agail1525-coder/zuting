## §2 作用域

重点闭环:
- 用户操作 → API → DB → 前端刷新 (CRUD闭环)
- 创建订单 → 支付 → 确认 → 状态更新 (交易闭环)
- Trip状态机: DRAFT → ... → COMPLETED (行程闭环)
- 收藏/评价 → 数据写入 → 列表/计数更新 (UGC闭环)
- 优惠券领取 → 使用 → 核销 → 库存更新 (促销闭环)

---

## §3 领域规则 — 6维度闭环检查

### F1 CRUD闭环
```
[FW-01] 创建成功后 → 列表invalidateQueries → 新数据出现
[FW-02] 更新成功后 → 详情页数据刷新
[FW-03] 删除成功后 → 列表invalidateQueries → 项目消失 (B10)
[FW-04] 创建/更新返回的数据 → 前端可直接使用(含ID)
[FW-05] 乐观更新(如有) → 失败时回滚到真实数据
```

### F2 状态机闭环
```
[FW-06] Trip每个状态 → 至少有一个UI操作可推进到下一状态
[FW-07] 状态变更后 → 前端展示新状态 + 可用操作按钮变化
[FW-08] 终态(COMPLETED/CANCELLED/REFUNDED) → 无进一步操作按钮
[FW-09] 状态变更 → 写审计日志 (R-09)
[FW-10] 并发状态变更 → 乐观锁或版本号保护
```

### F3 交易闭环
```
[FW-11] 订单创建 → 支付 → 确认 全链路数据一致
[FW-12] 支付成功 → 订单状态更新 → 库存/配额扣减
[FW-13] 退款 → 订单状态回退 → 库存/配额恢复
[FW-14] 优惠券使用 → 标记已用 → 不可重复使用
[FW-15] 积分消费 → 余额扣减 → 记录流水
```

### F4 UGC闭环
```
[FW-16] 评价提交 → 评价列表出现 → 平均分更新
[FW-17] 收藏操作 → 收藏列表同步 → 心形图标状态同步
[FW-18] 分享操作 → 分享计数+1 → 被分享页面可访问
```

### F5 事件链完整性
```
[FW-19] 定义了方法但无调用者 = 休眠方法 → 清除或接通
[FW-20] 事件发布后必须有至少一个消费者
[FW-21] WebSocket事件 → 前端必须有对应handler
[FW-22] 通知触发 → 用户端可收到 → 点击可跳转到源
```

### F6 数据一致性
```
[FW-23] 同一数据在列表/详情/搜索中显示一致
[FW-24] 缓存数据与DB数据有同步机制(Cache Invalidation)
[FW-25] 计数类字段(评价数/收藏数)与实际记录数一致
[FW-26] 跨表关联数据同步删除/更新(级联)
[FW-27] 多源竞争(多个API写同一资源) → 有冲突解决策略
[FW-28] 定时任务产生的数据变更 → 前端能感知到变化
```

## §4 执行流程

```
Step 0: 确定目标模块的核心闭环
Step 1: 追踪CRUD闭环 (F1)
  → 找到每个create/update/delete操作
  → 追踪: 前端操作 → API → Service → DB → Response → 前端刷新
  → 标记断裂点
Step 2: 追踪状态机+交易闭环 (F2-F3)
  → 画出Trip状态流转路径
  → 检查每个跳转的前端操作+后端处理+数据变更
  → 检查支付/退款/优惠券的完整链路
Step 3: 追踪UGC+事件+一致性 (F4-F6)
  → 检查评价/收藏/分享的写入→展示闭环
  → 检查事件发布/消费匹配
  → 检查计数字段与实际数据一致性
Step 4: 修复 (Executor)
  → 补缺的invalidateQueries
  → 接通休眠方法
  → 补缺的事件handler
  → 修复数据不一致
Step 5: 验证 (Verifier)
  → 端到端走一遍核心闭环
  → 确认数据一致
  → tsc验证
```

## §6 工具治理

```
允许工具:
  - Read: 追踪数据流 (Controller→Service→Prisma→Response)
  - Grep: 搜索方法调用关系/事件发布消费
  - Glob: 发现相关文件
  - Edit: 修复闭环断裂
  - Bash: tsc验证

核心检测命令:
  invalidate:   Grep "invalidateQueries|invalidate|refetch" --glob "*.tsx"
  休眠方法:     对比export列表与import引用
  事件发布:     Grep "emit|publish|dispatch" --glob "*.service.ts"
  事件消费:     Grep "on\(|subscribe|@OnEvent" --glob "*.ts"
  计数字段:     Grep "Count|_count|count" --path "services/api/src"
```
