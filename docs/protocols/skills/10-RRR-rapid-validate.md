> **触发**: 全审++ Step 1 / GOGOGO每Wave结束 / RRR {module}

## §0 元数据


---

## §1 角色定义

RRR是**速查工具**，不做深入分析，只做3分钟快速健康检查。
发现问题后标记给对应的++做深入处理。

---

## §2 作用域

全部代码: services/api/ + apps/ + packages/
重点: 最近修改的文件 (git diff)

---

## §3 领域规则 — 8点速查清单

```
[RRR-01] tsc编译 — 全端零错误?
  → cd services/api && npx tsc --noEmit
  → 任何tsc错误 = P0阻断

[RRR-02] 空壳检查 — 有无空组件/空页面/Alert占位?
  → Grep "TODO|FIXME|placeholder|空壳|EMPTY" --glob "*.tsx"
  → 空壳 = P1

[RRR-03] API连通 — 页面调用的API端点存在?
  → 对比前端fetch/api调用 vs Controller路由
  → 404 = P0

[RRR-04] 种子数据 — 新增的Model有种子数据?
  → 检查seed.ts是否覆盖新Model
  → 无种子 = P1 (列表为空)

[RRR-05] Guard覆盖 — 非Public路由有Guard?
  → Grep缺失@UseGuards的Controller
  → 缺失 = P0

[RRR-06] 空态可达 — 空列表时用户能创建第一条?
  → 检查列表页的创建入口
  → 鸡蛋死锁 = P1

[RRR-07] i18n基础 — 新增的文本有i18n key?
  → 检查最近添加的TSX是否使用t()
  → 硬编码文本 = P1

[RRR-08] import链 — 修改的export有没有破坏importer?
  → 修改了组件/函数签名 → 检查所有引用者
  → 引用断裂 = P0
```

---

## §4 执行流程

```
Step 0: git diff确定最近修改范围 (≤30秒)
Step 1: RRR-01 tsc编译 (≤60秒)
Step 2: RRR-02~04 快速Grep扫描 (≤60秒)
Step 3: RRR-05~08 关键安全+闭环检查 (≤60秒)
Step 4: 输出报告
  → P0: 立即修复 (阻断后续)
  → P1: 标记给对应++ (盲区/地狱/体验)
  → PASS: 继续下一步工作
```

**总耗时目标: ≤3分钟**

---

## §5 评分机制

| 维度 | 代号 | 结果 |
|------|------|------|
| RRR-D1 tsc | 编译 | PASS / FAIL(P0) |
| RRR-D2 空壳 | 完整 | 零空壳 / 有空壳(P1) |
| RRR-D3 API | 连通 | 全通 / 有断(P0) |
| RRR-D4 种子 | 数据 | 覆盖 / 缺失(P1) |
| RRR-D5 Guard | 安全 | 全覆盖 / 缺失(P0) |
| RRR-D6 空态 | 可达 | 可达 / 死锁(P1) |
| RRR-D7 i18n | 基础 | 无硬编码 / 有硬编码(P1) |
| RRR-D8 import | 链路 | 无断裂 / 有断裂(P0) |

综合: 8项全PASS = 5星 / 有P1无P0 = 4星 / 有P0 = 3星以下

---

## §6 工具治理

```
允许工具 (速度优先):
  - Bash: tsc --noEmit (最关键)
  - Grep: 快速模式匹配
  - Glob: 文件发现
  - Read: 仅在发现P0时精读

禁止 (速度限制):
  - 不做深入代码分析 (那是其他++的事)
  - 不做修复 (只标记问题)
  - 不读大文件 (>300行的跳过)
```

---

## §7 生命周期

```
L1 触发: 全审++预检 / GOGOGO Wave完成 / 手动
L2 执行: 8点速查 (≤3分钟)
L3 输出: RRR评分 + P0/P1清单
L4 反馈: P0→立即修复 / P1→标记给对应++
L5 沉淀: 新速查点 → 更新§3 (严格控制≤10点)
```

---

## §8 铁律清单

```
[RRR-R01] 3分钟时间预算 — 超时必须中止并输出已有结果
[RRR-R02] 只标记不修复 — RRR是侦察兵，不是工兵
[RRR-R03] P0必须阻断 — 有P0时停止后续工作先修复
[RRR-R04] 不做深入分析 — 发现疑似问题标记给对应++
[RRR-R05] 聚焦最近修改 — git diff范围优先，非全量扫描
```
