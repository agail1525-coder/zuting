## 1. 命名规范

### 1.1 组件名

| 场景 | 规则 | 示例 |
|------|------|------|
| 组件类名 | PascalCase | `HolySiteCard`, `TripStatusBadge` |
| 单文件组件 | `PascalCase.tsx` | `HolySiteCard.tsx` |
| 多文件组件目录 | `kebab-case/` + `index.tsx` | `holy-site-card/index.tsx` |
| Props 接口 | `I{Component}Props` | `IHolySiteCardProps` |
| 内部子组件 | `_{Component}` (不导出) | `_CardImage` |

```tsx
// 正确示例
interface IHolySiteCardProps {
  siteId: string;
  name: string;
  imageUrl?: string;
}

export function HolySiteCard(props: IHolySiteCardProps) { ... }
```

### 1.2 文件与目录

```
components/
├── HolySiteCard.tsx              # 简单单文件组件
├── trip-status-badge/            # 复杂多文件组件
│   ├── index.tsx                 # 导出入口
│   ├── TripStatusBadge.tsx       # 主组件
│   └── constants.ts              # 组件私有常量
└── page-layout/
    ├── index.tsx
    └── PageLayout.tsx
```

### 1.3 函数与变量 (遵循 [R-19][R-20])

- **组件函数**: PascalCase，使用 `function` 声明（便于 DevTools 识别）
- **事件处理函数**: `handle{Event}` 前缀，如 `handleSubmit`, `handleCardClick`
- **布尔 props**: `is/has/can` 前缀，如 `isLoading`, `hasError`, `canDelete`

## 2. 组件分类

### 2.1 基础组件 (`components/ui/`)

平台无关的纯 UI 原语，不包含业务逻辑，不调用 API。

| 组件 | 说明 |
|------|------|
| `Button` | 按钮，支持 variant/size/disabled/loading |
| `Input` | 输入框，支持校验状态 |
| `Card` | 通用卡片容器 |
| `Badge` | 标签/徽章 |
| `Avatar` | 用户/圣地头像 |
| `Modal` | 弹窗 |
| `Spinner` | 加载指示器 |
| `EmptyState` | 空数据占位 |

### 2.2 业务组件 (`components/business/`)

包含 JOINUS.COM 业务语义，可调用 API hooks，但不承担路由职责。

| 组件 | 说明 |
|------|------|
| `HolySiteCard` | 圣地卡片（图片+名称+信仰标签+评分） |
| `TripStatusBadge` | 行程状态徽章（映射12状态机颜色） |
| `ReligionTag` | 宗教标签（颜色+符号） |
| `PatriarchAvatar` | 祖师头像+简介 |
| `JournalEntry` | 朝圣日志条目 |
| `ReviewCard` | 用户评价卡片 |
| `PriceDisplay` | 价格展示（含币种/折扣） |

### 2.3 布局组件 (`components/layout/`)

负责页面骨架与区块划分，不包含具体业务数据。

| 组件 | 说明 |
|------|------|
| `PageLayout` | 页面容器（含 Header/Footer/导航） |
| `Section` | 带标题的页面分区 |
| `GridLayout` | 响应式网格容器 |
| `SidebarLayout` | 左侧边栏+主内容布局 |
| `HeroSection` | 大图 Hero 区块 |

### 2.4 页面组件 (`app/` 或 `pages/`)

对应路由节点，负责数据获取与组合，**不应被其他页面复用**。命名以 `Page` 结尾（如 `HolySiteDetailPage`）或直接作为路由文件。

## 3. Props 设计

### 3.1 强制规则

```tsx
// [R-01] 严禁 any — 必须定义 Interface
// 错误
function Card(props: any) { ... }

// 正确
interface ICardProps {
  title: string;
  description?: string;            // 可选字段加 ?
  children?: React.ReactNode;      // children 类型必须是 ReactNode
  onClick?: () => void;
  className?: string;              // Web 组件应支持 className 扩展
}

export function Card({ title, description = '', children, onClick }: ICardProps) { ... }
```

### 3.2 默认值策略

| 类型 | 默认值方式 | 示例 |
|------|-----------|------|
| 字符串 | 解构赋值 | `description = ''` |
| 布尔 | 解构赋值 | `isLoading = false` |
| 数组 | 解构赋值 | `items = []` |
| 数字 | 解构赋值 | `maxCount = 10` |
| 回调 | 不设默认值，调用前判断 | `onClick?.()` |

### 3.3 禁止模式

```tsx
// 禁止 any
interface IBadProps { data: any }

// 禁止直接扩展 HTMLElement（应明确声明需要的属性）
interface IBadProps extends React.HTMLAttributes<HTMLDivElement> { }

// 禁止 object 类型（必须具体化）
interface IBadProps { config: object }
```

## 4. 状态管理

### 4.1 局部状态 — useState

用于组件内部 UI 状态（展开/收起、选中、表单值等），**不应用于服务端数据**。

```tsx
const [isExpanded, setIsExpanded] = useState(false);
const [selectedTab, setSelectedTab] = useState<'info' | 'reviews'>('info');
```

### 4.2 服务端数据 — useQuery

遵循 **[B1] 禁止 useQuery + useEffect 双写**。

```tsx
// 错误 — 双写反模式 [B1]
const { data } = useQuery({ queryKey: ['site', id], queryFn: () => fetchSite(id) });
useEffect(() => { setSite(data); }, [data]); // 禁止

// 正确 — 直接消费 data
const { data: site, isLoading, isError } = useQuery({
  queryKey: ['holy-site', id],
  queryFn: () => apiClient.holySites.getById(id),
  enabled: !!id,   // [B4] enabled 条件控制，防止空 id 请求
});
```

### 4.3 enabled 条件控制 [B4]

```tsx
// 错误 — id 为空时发起无效请求
const { data } = useQuery({ queryKey: ['detail', id], queryFn: () => api.getDetail(id) });

// 正确
const { data } = useQuery({
  queryKey: ['detail', id],
  queryFn: () => api.getDetail(id!),
  enabled: !!id,
});
```

### 4.4 数据变更 — useMutation + invalidate [B10]

```tsx
const queryClient = useQueryClient();

const deleteMutation = useMutation({
  mutationFn: (id: string) => apiClient.trips.delete(id),
  onSuccess: () => {
    // [B10] 删除后必须 invalidate，禁止遗漏
    queryClient.invalidateQueries({ queryKey: ['trips'] });
  },
});
```

## 5. 空安全编程

### 5.1 可选链与空值合并

```tsx
// 可选链 ?.
const name = user?.profile?.displayName;
const firstItem = items?.[0];

// 空值合并 ??（优于 || ，避免 0/false 被误判）
const score = item.score ?? 0;
const label = item.nameZh ?? item.name ?? '未知';
```

### 5.2 数值方法调用前 null 检查 [B7]

```tsx
// 错误 — score 可能为 null/undefined
<span>{item.score.toFixed(1)}</span>

// 正确
<span>{(item.score ?? 0).toFixed(1)}</span>
```

### 5.3 数组 map 前 isArray 检查 [B8]

```tsx
// 错误 — data.items 可能不是数组
{data.items.map(item => <Card key={item.id} {...item} />)}

// 正确
{(Array.isArray(data?.items) ? data.items : []).map(item => (
  <Card key={item.id} {...item} />
))}
```

### 5.4 对象属性访问

```tsx
// 访问深层嵌套对象时，逐层保护
const lat = site?.location?.coordinates?.lat ?? 0;
const tag = religion?.tags?.[0]?.label ?? '';
```

## 6. 样式方案

### 6.1 Web — TailwindCSS v4

```tsx
// 使用 Tailwind 工具类，禁止行内 style（除动态计算值）
<div className="flex items-center gap-4 rounded-xl bg-slate-800 p-6 shadow-lg">
  <span className="text-gold-400 font-semibold">{title}</span>
</div>

// 动态样式：使用 cn() 工具函数合并类名
import { cn } from '@/lib/utils';

<button className={cn(
  'rounded-lg px-4 py-2 font-medium transition-colors',
  isActive ? 'bg-amber-500 text-white' : 'bg-slate-700 text-slate-300',
  disabled && 'opacity-50 cursor-not-allowed',
)}>
```

**禁止**:
- 混用 `style={{}}` 和 Tailwind 覆盖同一属性
- 使用魔法数字颜色（应使用设计 token）
- 在组件内写 `<style>` 标签

### 6.2 Mobile — React Native StyleSheet

```tsx
import { StyleSheet } from 'react-native';

// 所有样式集中在文件底部 styles 对象
const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: '#0f172a',
    padding: 16,
  },
  title: {
    fontSize: 18,
    fontWeight: '600',
    color: '#D4A855',  // 金色点缀 token
  },
});

// 禁止直接写行内对象（每次渲染重新创建）
// 错误: style={{ flex: 1, padding: 16 }}
// 正确: style={styles.container}
```

### 6.3 小程序 — SCSS Modules

```scss
// Component.module.scss
.container {
  display: flex;
  background-color: #0f172a;
  padding: 32rpx;
}

.title {
  font-size: 36rpx;
  color: #D4A855;
}
```

```tsx
import styles from './Component.module.scss';
<View className={styles.container}>
  <Text className={styles.title}>{title}</Text>
</View>
```

### 6.4 Admin — Ant Design Tokens

```tsx
import { theme } from 'antd';

// 使用 Ant Design Design Token，禁止硬编码颜色
const { token } = theme.useToken();

<div style={{ color: token.colorPrimary, background: token.colorBgContainer }}>
```

## 7. 加载状态三态

**[HH-E01 Q4] 页面的 loading / error / empty 三态必须全部处理。**

### 7.1 标准三态模板

```tsx
export function HolySiteList() {
  const { data, isLoading, isError, error } = useQuery({
    queryKey: ['holy-sites'],
    queryFn: () => apiClient.holySites.getAll(),
  });

  // 1. Loading 态
  if (isLoading) {
    return (
      <div className="flex items-center justify-center py-20">
        <Spinner size="lg" />
        <span className="ml-3 text-slate-400">加载圣地中...</span>
      </div>
    );
  }

  // 2. Error 态
  if (isError) {
    return (
      <div className="flex flex-col items-center py-20 text-center">
        <p className="text-red-400">加载失败：{(error as Error).message}</p>
        <Button variant="outline" className="mt-4" onClick={() => refetch()}>
          重试
        </Button>
      </div>
    );
  }

  // 3. Empty 态
  if (!data?.items?.length) {
    return (
      <EmptyState
        icon="temple"
        title="暂无圣地数据"
        description="管理员正在录入，请稍后再来"
      />
    );
  }

  // 4. 正常渲染
  return (
    <div className="grid grid-cols-1 gap-4 sm:grid-cols-2 lg:grid-cols-3">
      {data.items.map(site => (
        <HolySiteCard key={site.id} {...site} />
      ))}
    </div>
  );
}
```

### 7.2 各端三态组件对照

| 端 | Loading | Error | Empty |
|----|---------|-------|-------|
| Web | `<Spinner>` + 骨架屏 | 红色提示 + 重试按钮 | `<EmptyState>` 组件 |
| Mobile | `<ActivityIndicator>` | `<ErrorBanner>` | `<EmptyView>` |
| 小程序 | `<Loading>` | `<AtMessage>` | 空状态 View |
| Admin | Ant Design `<Spin>` | `<Alert type="error">` | `<Empty>` |

## 8. 响应式布局

### 8.1 移动优先原则

所有 Web 组件从最小屏幕开始设计，逐步扩展。

```tsx
// 移动优先 — 基础样式适配手机，断点样式适配大屏
<div className="
  grid grid-cols-1 gap-4
  sm:grid-cols-2
  md:grid-cols-3
  lg:grid-cols-4
  xl:grid-cols-5
">
```

### 8.2 Tailwind 断点规范

| 断点 | 最小宽度 | 适用场景 |
|------|---------|---------|
| (无前缀) | 0px | 移动端基础样式 |
| `sm:` | 640px | 大屏手机/小平板 |
| `md:` | 768px | 平板 |
| `lg:` | 1024px | 笔记本/桌面 |
| `xl:` | 1280px | 大屏桌面 |
| `2xl:` | 1536px | 超宽屏 |

### 8.3 禁止模式

```tsx
// 禁止 — 固定像素宽度导致移动端溢出
<div style={{ width: '1200px' }}>

// 禁止 — 忽略移动端，只写桌面样式
<div className="grid grid-cols-4 gap-8">

// 正确
<div className="w-full max-w-screen-xl mx-auto px-4 sm:px-6 lg:px-8">
```

## 9. 国际化

### 9.1 文本 i18n 化

```tsx
import { useTranslations } from 'next-intl';  // Web
// 或 import { t } from '@/i18n';              // 其他端

export function HolySiteCard({ site }: IHolySiteCardProps) {
  const t = useTranslations('holySite');

  return (
    <Card>
      {/* 正确 — 使用 i18n key */}
      <h3>{t('viewDetail')}</h3>

      {/* 允许 — 来自 API 的数据（已含多语言字段） */}
      <p>{site.nameZh ?? site.name}</p>

      {/* 禁止 — 硬编码 UI 文本 */}
      {/* <h3>查看详情</h3> */}
    </Card>
  );
}
```

### 9.2 允许例外

以下情况可以不走 i18n：

- 纯装饰性文字（如"···"、图标 emoji 替代）
- 来自 API 返回的数据字段（nameZh/nameEn 由后端管理）
- 调试/开发环境仅供内部使用的组件

### 9.3 i18n Key 命名规范

```
{模块}.{功能}.{描述}

示例:
holySite.card.viewDetail
trip.status.inProgress
common.button.submit
common.error.networkError
```

## 10. 可复用原则

### 10.1 抽取阈值

**同一 UI 片段出现 3 次或以上，才抽取为独立组件。**

过早抽象导致过度工程化。先内联实现，积累到3处再重构。

### 10.2 单一职责

每个组件只做一件事：

| 正确 | 错误 |
|------|------|
| `HolySiteCard` 只渲染圣地卡片 | `HolySiteCard` 同时处理收藏逻辑+评论弹窗+地图跳转 |
| `TripStatusBadge` 只显示状态标签 | `TripStatusBadge` 还负责触发状态转换 |

### 10.3 组合优于继承

```tsx
// 错误 — 继承扩展
class PremiumCard extends Card { ... }

// 正确 — 组合扩展
function PremiumCard({ children, badge, ...props }: IPremiumCardProps) {
  return (
    <Card {...props}>
      <Badge variant="premium">{badge}</Badge>
      {children}
    </Card>
  );
}
```

### 10.4 孤立文件禁止 [HH-V04]

新建组件必须有明确的调用者。禁止创建"备用"或"以后会用到"的孤立组件。

## 11. 性能优化

### 11.1 React.memo

仅用于**纯展示组件**（Props 不变则不重渲染）。业务组件通常不需要，过度使用反而增加比较开销。

```tsx
// 适合 memo — 纯展示，props 变化频率低
export const ReligionTag = React.memo(function ReligionTag({
  name, color, symbol,
}: IReligionTagProps) {
  return (
    <span style={{ backgroundColor: color }}>
      {symbol} {name}
    </span>
  );
});

// 不适合 memo — 含复杂逻辑或频繁更新的组件
```

### 11.2 useMemo / useCallback

**仅在有性能测量依据时使用**，不要提前优化。

```tsx
// 适合 useMemo — 高昂计算
const sortedSites = useMemo(
  () => sites.sort((a, b) => b.rating - a.rating),
  [sites],
);

// 适合 useCallback — 传给子组件的稳定回调
const handleSelect = useCallback((id: string) => {
  setSelectedId(id);
}, []); // 依赖项为空，函数引用稳定
```

### 11.3 图片懒加载

```tsx
// Web — Next.js Image 组件（自动懒加载）
import Image from 'next/image';
<Image src={imageUrl} alt={name} width={400} height={300} loading="lazy" />

// Web — 原生 img
<img src={imageUrl} alt={name} loading="lazy" decoding="async" />

// Mobile — Expo Image
import { Image } from 'expo-image';
<Image source={{ uri: imageUrl }} contentFit="cover" transition={300} />
```

### 11.4 列表虚拟化

长列表（超过 50 条）必须使用虚拟化：

- Web: `react-window` 或 `@tanstack/virtual`
- Mobile: `FlatList`（原生虚拟化）
- 小程序: Taro `VirtualList`
