## 1. 总则与设计原则

JOINUS.COM 定位全球宗教文化旅行平台，目标服务全球 7 大语系用户。国际化是平台全球化战略的技术基础，必须在架构层面提前设计，而非后期补救。

### 1.1 核心原则

| 原则 | 说明 |
|------|------|
| **i18n 优先** | 任何新页面/组件开发，必须从第一行代码起使用 i18n Key，禁止先写死文本再回头替换 |
| **Key 驱动** | 所有可见文本通过 Key 引用，不得出现硬编码字符串 |
| **命名空间隔离** | 不同功能模块使用独立命名空间，避免 Key 污染 |
| **回退链机制** | 当目标语言缺失时，自动回退：目标语言 → 英语 → 中文 |
| **内容与代码分离** | 翻译文件独立于代码，可由非工程师维护 |

### 1.2 当前阶段目标

- **Phase A (当前)**: 中文 + 英文双语全覆盖
- **Phase D (规划)**: 扩展至 7 语言（详见语言支持矩阵）

## 3. i18n 架构方案

### 3.1 各端技术方案

| 端 | 技术栈 | i18n 库 | 配置文件位置 |
|----|--------|---------|-------------|
| Web (Next.js 15) | next-intl | `next-intl` | `apps/web/src/lib/i18n/` |
| Admin (Vite) | 自定义 + react-i18next | `react-i18next` | `apps/admin/src/i18n/` |
| Mobile (Expo) | i18n-js | `i18n-js` | `apps/mobile/src/i18n/` |
| 小程序 (Taro) | 自定义方案 | 自定义 Hook | `apps/miniprogram/src/i18n/` |
| 共享包 | — | — | `packages/config/src/i18n/` |

### 3.2 Web 端 (next-intl) 配置示例

```typescript
// apps/web/src/lib/i18n/config.ts
export const locales = ['zh-CN', 'en'] as const;
export type Locale = typeof locales[number];
export const defaultLocale: Locale = 'zh-CN';

// 回退链配置
export const localeConfig = {
  'zh-CN': { fallback: 'en' },
  'en':    { fallback: 'zh-CN' },
};
```

```typescript
// apps/web/src/app/[locale]/page.tsx
import { useTranslations } from 'next-intl';

export default function HomePage() {
  const t = useTranslations('home');
  return <h1>{t('hero.title')}</h1>;
}
```

### 3.3 Mobile 端 (i18n-js) 配置示例

```typescript
// apps/mobile/src/i18n/index.ts
import { I18n } from 'i18n-js';
import zh from './locales/zh-CN/common.json';
import en from './locales/en/common.json';

export const i18n = new I18n({ 'zh-CN': zh, en });
i18n.locale = 'zh-CN';
i18n.enableFallback = true;
i18n.defaultLocale = 'zh-CN';

export const t = (key: string, options?: object) => i18n.t(key, options);
```

### 3.4 小程序端 (自定义 Hook) 配置示例

```typescript
// apps/miniprogram/src/i18n/useI18n.ts
import { useState } from 'react';
import zh from './locales/zh-CN/common.json';
import en from './locales/en/common.json';

const translations: Record<string, Record<string, unknown>> = { 'zh-CN': zh, en };

export function useI18n(locale: string = 'zh-CN') {
  const messages = translations[locale] ?? translations['zh-CN'];
  const t = (key: string): string => {
    const keys = key.split('.');
    let value: unknown = messages;
    for (const k of keys) {
      value = (value as Record<string, unknown>)?.[k];
    }
    return (typeof value === 'string' ? value : key);
  };
  return { t };
}
```

### 3.5 共享翻译常量

宗教名称、圣地名称等全平台共用内容，统一放置在 `packages/config/src/i18n/`，各端直接引用，避免重复维护。

```typescript
// packages/config/src/i18n/religions.ts
export const RELIGION_NAMES: Record<string, Record<string, string>> = {
  'zh-CN': {
    buddhism:  '佛教',
    taoism:    '道教',
    christianity: '基督教',
    // ...
  },
  en: {
    buddhism:  'Buddhism',
    taoism:    'Taoism',
    christianity: 'Christianity',
    // ...
  },
};
```

## 4. Key 命名规范

### 4.1 命名结构

```
{命名空间}.{页面/模块}.{区块}.{元素}
```

- 使用**点分隔**的层级命名
- 全部使用**小写字母 + 连字符**（kebab-case）
- **禁止使用中文作为 Key**
- **禁止使用数字开头的 Key**
- Key 应描述内容位置，而非内容本身（位置稳定，内容会变）

### 4.2 命名空间定义

| 命名空间 | 对应范围 | 文件名 |
|---------|---------|-------|
| `common` | 全局通用（按钮、提示、状态） | `common.json` |
| `home` | 首页 | `home.json` |
| `holy-site` | 圣地相关页面 | `holy-site.json` |
| `temple` | 祖庭相关页面 | `temple.json` |
| `trip` | 行程管理 | `trip.json` |
| `chat` | AI 聊天（小鸿） | `chat.json` |
| `journal` | 朝圣日志 | `journal.json` |
| `auth` | 登录/注册/个人中心 | `auth.json` |
| `order` | 订单管理 | `order.json` |
| `admin` | 管理后台（仅 Admin 端） | `admin.json` |
| `error` | 错误页面与提示 | `error.json` |

### 4.3 正确与错误示例


## 5. 翻译文件组织结构

### 5.1 目录结构

```
locales/
├── zh-CN/                    # 简体中文（源语言，开发者维护）
│   ├── common.json
│   ├── home.json
│   ├── holy-site.json
│   ├── temple.json
│   ├── trip.json
│   ├── chat.json
│   ├── journal.json
│   ├── auth.json
│   ├── order.json
│   └── error.json
├── en/                       # 英语（翻译语言）
│   ├── common.json
│   ├── home.json
│   ├── ...
├── ja/                       # 日语（Phase D）
├── ko/                       # 韩语（Phase D）
├── th/                       # 泰语（Phase D）
├── hi/                       # 印地语（Phase D）
└── ar/                       # 阿拉伯语（Phase D，RTL）
```

各端的 `locales/` 目录位于各自的 `src/i18n/locales/` 路径下。
共享翻译位于 `packages/config/src/i18n/locales/`。

### 5.2 文件格式规范

```json
{
  "section-name": {
    "element": "翻译文本",
    "element-with-var": "你好，{name}",
    "element-plural": "{count, plural, one {# 个} other {# 个}}"
  }
}
```

- 文件编码: **UTF-8**（无 BOM）
- 文件格式: **JSON**（不使用 YAML/TOML）
- 每个文件最大 Key 数量: **500 个**（超过则拆分命名空间）
- Key 按字母顺序排列，便于 diff 对比

### 5.3 复数规则

```json
// zh-CN/common.json
{
  "holy-site.count": "{count} 个圣地"
}

// en/common.json
{
  "holy-site.count": "{count, plural, one {# holy site} other {# holy sites}}"
}
```

## 6. 日期与时间规范

### 6.1 存储与传输标准

遵循 [R-23] 铁律：**所有时间值使用 ISO 8601 格式存储和 API 传输**。

```typescript
// ✅ API 传输/存储格式
"2026-03-29T08:00:00.000Z"    // UTC ISO 8601
"2026-03-29"                   // 纯日期

// ❌ 禁止在 API 中传输
"2026年3月29日"
"03/29/2026"
"29-03-2026"
```

### 6.2 显示格式（按 locale）

```typescript
// apps/web/src/lib/i18n/dateFormatter.ts
export function formatDate(
  isoString: string,
  locale: string,
  style: 'short' | 'medium' | 'long' = 'medium'
): string {
  const date = new Date(isoString);
  const options: Intl.DateTimeFormatOptions = {
    short:  { month: 'short', day: 'numeric' },
    medium: { year: 'numeric', month: 'long', day: 'numeric' },
    long:   { year: 'numeric', month: 'long', day: 'numeric', weekday: 'long' },
  }[style];
  return new Intl.DateTimeFormat(locale, options).format(date);
}

// 使用示例
formatDate('2026-03-29T08:00:00Z', 'zh-CN', 'medium')  // → "2026年3月29日"
formatDate('2026-03-29T08:00:00Z', 'en',    'medium')  // → "March 29, 2026"
formatDate('2026-03-29T08:00:00Z', 'ja',    'medium')  // → "2026年3月29日"
```

### 6.3 相对时间显示

```typescript
// 使用 Intl.RelativeTimeFormat
export function formatRelativeTime(isoString: string, locale: string): string {
  const diff = new Date(isoString).getTime() - Date.now();
  const rtf = new Intl.RelativeTimeFormat(locale, { numeric: 'auto' });
  // 智能选择单位：seconds/minutes/hours/days
  const seconds = Math.round(diff / 1000);
  if (Math.abs(seconds) < 60) return rtf.format(seconds, 'second');
  const minutes = Math.round(seconds / 60);
  if (Math.abs(minutes) < 60) return rtf.format(minutes, 'minute');
  const hours = Math.round(minutes / 60);
  if (Math.abs(hours) < 24) return rtf.format(hours, 'hour');
  return rtf.format(Math.round(hours / 24), 'day');
}
```

### 6.4 时区处理

圣地 GPS 数据包含 UTC 偏移，显示时应使用圣地本地时间：

```typescript
// 圣地详情页显示当地时间
export function formatLocalTime(
  utcString: string,
  utcOffset: number,  // 例: 8 表示 UTC+8
  locale: string
): string {
  const localMs = new Date(utcString).getTime() + utcOffset * 3600 * 1000;
  const localDate = new Date(localMs);
  return new Intl.DateTimeFormat(locale, {
    hour: '2-digit',
    minute: '2-digit',
    timeZoneName: 'short',
  }).format(localDate);
}
```

### 7.2 货币格式化

```typescript
// ✅ 货币格式化（预留多币种支持）
export function formatCurrency(
  amount: number,
  currency: string,  // ISO 4217: 'CNY' | 'USD' | 'JPY' | 'EUR' ...
  locale: string
): string {
  return new Intl.NumberFormat(locale, {
    style: 'currency',
    currency,
    minimumFractionDigits: currency === 'JPY' ? 0 : 2,
  }).format(amount);
}

// formatCurrency(1299, 'CNY', 'zh-CN') → "¥1,299.00"
// formatCurrency(1299, 'USD', 'en')    → "$1,299.00"
// formatCurrency(1299, 'JPY', 'ja')    → "¥1,299"
```

## 8. RTL 支持规范

### 8.1 CSS Logical Properties（优先使用）

阿拉伯语为 RTL 语言，布局需要镜像翻转。使用 CSS 逻辑属性可自动适应方向：

```css
/* ✅ 使用逻辑属性（RTL 自动适配） */
.card {
  margin-inline-start: 16px;   /* LTR: margin-left; RTL: margin-right */
  margin-inline-end:   16px;   /* LTR: margin-right; RTL: margin-left */
  padding-block:       12px;
  border-inline-start: 2px solid var(--color-gold);
}

/* ❌ 避免直接使用方向属性（RTL 下需手动覆盖） */
.card {
  margin-left: 16px;
  padding-top: 12px;
  border-left: 2px solid var(--color-gold);
}
```

### 8.2 HTML dir 属性

```tsx
// apps/web/src/app/[locale]/layout.tsx
const RTL_LOCALES = ['ar'];

export default function RootLayout({ children, params: { locale } }: Props) {
  const dir = RTL_LOCALES.includes(locale) ? 'rtl' : 'ltr';
  return (
    <html lang={locale} dir={dir}>
      <body>{children}</body>
    </html>
  );
}
```

### 8.3 图标与方向性图形

```tsx
// 有方向含义的图标在 RTL 下需镜像
// 例如: 箭头、返回按钮、进度条方向

const ArrowIcon = ({ locale }: { locale: string }) => {
  const isRTL = ['ar'].includes(locale);
  return (
    <svg
      style={{ transform: isRTL ? 'scaleX(-1)' : undefined }}
      // ...
    />
  );
};
```

### 8.4 TailwindCSS v4 RTL 支持

```html
<!-- 使用 Tailwind 的 rtl: 变体 -->
<div class="ms-4 rtl:me-4 text-left rtl:text-right">
  内容
</div>
```

> **当前阶段（Phase A）**: RTL 代码预留接口即可，不需要完整实现。禁止因此增加大量无用代码。

## 9. 翻译工作流

### 9.1 标准翻译流程

```
步骤 1: 开发者写功能
  └─ 直接使用 i18n Key，同步在 zh-CN/ 文件中新增对应中文翻译

步骤 2: 生成英文翻译
  └─ 开发者基于语义提供初稿英文翻译（可用 AI 辅助）
  └─ 存入 en/ 对应文件

步骤 3: 人工审核
  └─ 宗教文化专有术语必须由熟悉对应信仰的人工审核
  └─ 普通 UI 文本可由双语工程师审核

步骤 4: 补充缺失
  └─ 每次 PR 合并前运行 i18n 完整性检查，确保 zh-CN 与 en Key 数量一致

步骤 5: 上线验证
  └─ 切换到英文模式，验证页面无 Key 裸露（不应显示 "home.hero.title"）
```

### 9.2 i18n 完整性检查脚本

```bash
# scripts/check-i18n.sh
# 检查 zh-CN 与 en 的 Key 是否完全对齐
node scripts/i18n-check.js --source=zh-CN --target=en --path=apps/web/src/i18n/locales
```

```typescript
// scripts/i18n-check.ts
```

## 10. 图片与媒体国际化

### 10.1 图片中的文字

**禁止**在图片内嵌入文字（特别是 UI 说明性文字），因为图片内的文字无法被翻译。

```
// ✅ 正确做法: 图片 + 独立文字覆盖层
<div className="relative">
  <img src="/holy-site.jpg" alt={t('image.holy-site-alt')} />
  <div className="overlay-text">{t('image.caption')}</div>
</div>

// ❌ 错误做法: 在 Photoshop 中将中文文字烧录进图片
```

### 10.2 alt 文本国际化

所有图片的 `alt` 属性必须使用 i18n Key：

```tsx
// ✅ 正确
<img src={site.imageUrl} alt={t('holy-site.image-alt', { name: site.name })} />

// ❌ 错误
<img src={site.imageUrl} alt="圣地图片" />
<img src={site.imageUrl} alt="" />  // 装饰性图片除外，需 role="presentation"
```

### 10.3 语言切换 SEO

Web 端需要在 `<head>` 中声明多语言版本：

```html
<link rel="alternate" hreflang="zh-CN" href="https://joinus.com/zh-CN" />
<link rel="alternate" hreflang="en"    href="https://joinus.com/en" />
<link rel="alternate" hreflang="x-default" href="https://joinus.com" />
```

## 11. 铁律禁止项

以下行为**违反即阻断**：

```
[I18N-F01] 禁止硬编码任何面向用户的文本字符串
           ❌ return <h1>探索世界圣地</h1>
           ✅ return <h1>{t('home.hero.title')}</h1>

[I18N-F02] 禁止使用中文作为 i18n Key
           ❌ t('探索按钮')
           ✅ t('home.hero.cta-primary')

[I18N-F03] 禁止在图片内烧录文字（特别是 UI 引导文本）

[I18N-F04] 禁止跳过英文翻译直接上线（zh-CN 与 en 必须同步）

[I18N-F05] 禁止使用物理方向属性替代逻辑属性（新写的样式代码必须使用逻辑属性）
           ❌ margin-left  → ✅ margin-inline-start
           ❌ padding-right → ✅ padding-inline-end

[I18N-F06] 禁止在 API 传输中使用本地化日期格式
           ❌ "2026年3月29日"  → ✅ "2026-03-29T00:00:00.000Z"

[I18N-F07] 禁止在 alt 属性中硬编码文本（装饰性图片用 role="presentation" 代替）

[I18N-F08] 禁止将翻译逻辑散落在业务组件内
           ❌ const text = locale === 'zh-CN' ? '探索' : 'Explore'
           ✅ t('common.action.explore')
```

## 12. 验收标准

开发完成后，i18n 验收需通过以下全部检查项：

| 检查项 | 验收方法 | 通过标准 |
|--------|---------|---------|
| Key 完整性 | 运行 `check-i18n.sh` | zh-CN 与 en 的 Key 数量完全一致 |
| 无裸 Key | 切换英文模式浏览全页 | 页面不出现 `xxx.yyy.zzz` 格式文本 |
| 无硬编码文本 | `grep -rn "中文字" src/` | 零结果（宗教文化内容数据除外）|
| 日期格式 | 切换英文模式 | 日期显示为英文格式（Mar 29, 2026）|
| 图片 alt | 审查 JSX | 所有 `<img>` 有 i18n alt 或 `role="presentation"` |
| 数字格式 | 切换英文模式 | 大数字使用逗号分隔（1,000,000）|
| 回退机制 | 删除一个 Key 测试 | 回退到上级语言而非崩溃 |
| CSS 逻辑属性 | 代码审查 | 新增样式代码无 `margin-left` 等物理属性 |
