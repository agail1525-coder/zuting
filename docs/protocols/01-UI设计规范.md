## 1. 品牌色彩体系

> 核心设计方向: 以 Trip.com 蓝色系为主调(国际旅行信任感)，融合宗教文化金色(东方精神底蕴)。
> 亮色模式为 Web/Admin 默认，暗色模式为 Mobile/小程序 默认，两套均需完整支持。

### 1.1 主色调 — 品牌蓝

| 层级 | 色值 | Tailwind 近似 | 用途 |
|------|------|---------------|------|
| 主色 Primary | `#1A73E8` | `blue-600` | CTA按钮、主导航激活、链接 |
| 主色浅 Primary Light | `#4A90E2` | `blue-400` | 悬停态、浅背景 |
| 主色深 Primary Dark | `#1557B0` | `blue-700` | 按下态、强调 |
| 主色极浅 Primary 50 | `#E8F0FD` | `blue-50` | 选中背景、Tag背景 |

> 对标: Trip.com 主蓝 `#0085FF`，Google蓝 `#1A73E8`。JOINUS采用Google蓝，兼顾识别度与亲和力。

### 1.2 辅色 — 文化金

| 层级 | 色值 | 用途 |
|------|------|------|
| 金色 Gold | `#D4A855` | 宗教文化点缀、重要标签、评星、徽章 |
| 金色浅 Gold Light | `#F0CC7A` | 金色背景填充、高亮提示 |
| 金色深 Gold Dark | `#A8832A` | 金色文字、深色底上使用 |
| 金色极浅 Gold 50 | `#FDF8EC` | 金色区块背景 |

### 1.3 中性色

| 名称 | 色值 | 用途 |
|------|------|------|
| 正文黑 | `#1A1A2E` | 主要正文、标题 |
| 次要文字 | `#4A5568` | 辅助说明、标签 |
| 占位文字 | `#9CA3AF` | Placeholder、禁用态 |
| 分割线 | `#E5E7EB` | 卡片边框、分割线 |
| 浅背景 | `#F8FAFC` | 页面背景、卡片背景 |
| 纯白 | `#FFFFFF` | 卡片内容区、Modal |

### 1.4 功能色

| 类型 | 色值 | 用途 |
|------|------|------|
| 成功 Success | `#10B981` | 支付成功、状态完成 |
| 警告 Warning | `#F59E0B` | 提醒、待处理状态 |
| 错误 Error | `#EF4444` | 错误提示、危险操作 |
| 信息 Info | `#3B82F6` | 通知提示、说明 |

### 1.5 色彩使用铁律

```
[C-01] 主色 #1A73E8 仅用于CTA按钮、主导航、可交互链接，禁止用于纯装饰
[C-02] 金色 #D4A855 仅用于宗教文化元素、评星、徽章，禁止滥用
[C-03] 功能色不可互换使用，错误一定用红，成功一定用绿
[C-04] 文字与背景色对比度必须 ≥ 4.5:1 (WCAG AA)
[C-05] 禁止在白底上使用黄色或极浅色文字
```

## 4. 字体规范

### 4.1 字体族

| 场景 | 首选字体 | 降级字体 | 系统兜底 |
|------|----------|----------|---------|
| 中文正文 | 思源黑体 (Source Han Sans) | PingFang SC | system-ui, -apple-system |
| 中文标题 | 思源宋体 (Source Han Serif) | 华文细黑 | serif |
| 英文正文 | Inter | Helvetica Neue | sans-serif |
| 英文标题 | Inter Display | Georgia | serif |
| 数字/金额 | Inter (tabular-nums) | Helvetica Neue | monospace |
| 代码 | JetBrains Mono | Fira Code | monospace |

```css
/* 基础字体栈 */
body {
  font-family: Inter, 'Source Han Sans SC', 'PingFang SC', 'Helvetica Neue',
               -apple-system, BlinkMacSystemFont, system-ui, sans-serif;
}
```

### 4.2 字号层级

| 层级 | 字号 | 行高 | 字重 | Tailwind | 用途 |
|------|------|------|------|----------|------|

| H2 章节标题 | 36px / 2.25rem | 1.25 | 700 Bold | `text-4xl font-bold` | 首页区块标题 |
| H3 小节标题 | 28px / 1.75rem | 1.3 | 600 Semibold | `text-3xl font-semibold` | 卡片标题、弹窗标题 |
| H4 组件标题 | 22px / 1.375rem | 1.35 | 600 Semibold | `text-2xl font-semibold` | 列表项标题 |
| H5 标签标题 | 18px / 1.125rem | 1.4 | 500 Medium | `text-xl font-medium` | Tab标题、侧边栏 |
| H6 微标题 | 15px / 0.9375rem | 1.4 | 500 Medium | `text-base font-medium` | 表格头、辅助标题 |
| 正文大 Body L | 16px / 1rem | 1.6 | 400 Regular | `text-base` | 详情页正文 |
| 正文 Body M | 14px / 0.875rem | 1.6 | 400 Regular | `text-sm` | 卡片说明、列表正文 |
| 正文小 Body S | 13px / 0.8125rem | 1.5 | 400 Regular | `text-[13px]` | 辅助说明、元数据 |

| 极小 Mini | 11px / 0.6875rem | 1.4 | 400 Regular | `text-[11px]` | 角标、Badge数字 |

### 4.3 字体铁律

```
[F-01] 正文最小字号 12px，移动端正文最小 13px
[F-02] 行高最小 1.4，段落文字最小 1.6
[F-03] 金额/数字使用 font-variant-numeric: tabular-nums 等宽数字
[F-04] 禁止在正文中使用超过3种字重
[F-05] 中文标题慎用 italic，影响可读性
```

## 5. 间距系统

> 基于 4px 基础单位，所有间距必须是 4 的倍数。Tailwind 默认 4px=1unit 完全对应。

### 5.1 间距规模

| Token | 值(px) | Tailwind | 用途场景 |
|-------|--------|----------|---------|
| space-1 | 4px | `p-1 / m-1` | 图标内边距、极小间隔 |
| space-2 | 8px | `p-2 / m-2` | Tag内边距、紧凑列表行间距 |
| space-3 | 12px | `p-3 / m-3` | 按钮内边距(竖)、输入框内边距 |
| space-4 | 16px | `p-4 / m-4` | 卡片内边距(小)、列表项间距 |
| space-5 | 20px | `p-5 / m-5` | 按钮内边距(横)、表单行间距 |
| space-6 | 24px | `p-6 / m-6` | 卡片内边距(标准)、区块间距 |
| space-8 | 32px | `p-8 / m-8` | 页面内边距、区块内间距 |
| space-10 | 40px | `p-10 / m-10` | 大区块间距 |
| space-12 | 48px | `p-12 / m-12` | 页面Section间距 |
| space-16 | 64px | `p-16 / m-16` | Hero区域上下边距 |
| space-20 | 80px | `p-20 / m-20` | 大型Hero上边距 |
| space-24 | 96px | `p-24 / m-24` | 页面顶部留白(桌面端) |

### 5.2 页面布局间距规范

```
页面两侧边距:
  - Mobile: 16px (p-4)
  - Tablet: 24px (p-6)
  - Desktop: 32px (p-8)
  - Wide: auto(最大容器1280px居中)

内容区最大宽度:
  - 文章/详情: max-w-4xl (896px)
  - 列表/网格: max-w-7xl (1280px)
  - 全宽Hero: 无限制

卡片间距(grid gap):
  - 紧凑: 12px (gap-3)
  - 标准: 16px (gap-4)
  - 宽松: 24px (gap-6)

Section 间距:
  - Mobile: 32px (py-8)
  - Desktop: 64px (py-16)
```

## 6. 组件规范

### 6.1 按钮 Button

#### 尺寸规范

| 尺寸 | 高度 | 水平内边距 | 字号 | Tailwind |
|------|------|-----------|------|----------|
| XS | 28px | 12px | 12px | `h-7 px-3 text-xs` |
| SM | 32px | 16px | 13px | `h-8 px-4 text-sm` |
| MD (默认) | 40px | 20px | 14px | `h-10 px-5 text-sm` |
| LG | 48px | 24px | 16px | `h-12 px-6 text-base` |
| XL | 56px | 32px | 18px | `h-14 px-8 text-lg` |

#### 类型规范

```tsx
// 主按钮 Primary — CTA场景，每屏最多1个
<button className="bg-[#1A73E8] text-white hover:bg-[#1557B0] active:bg-[#1043A0]
                   rounded-lg font-medium transition-colors duration-200
                   disabled:opacity-40 disabled:cursor-not-allowed">
  立即预订
</button>

// 次要按钮 Secondary — 辅助操作
<button className="bg-white text-[#1A73E8] border border-[#1A73E8]
                   hover:bg-[#E8F0FD] active:bg-[#D0E4FA]
                   rounded-lg font-medium transition-colors duration-200">
  查看详情
</button>

// 危险按钮 Danger — 删除/取消等破坏性操作，必须有二次确认
<button className="bg-[#EF4444] text-white hover:bg-[#DC2626] active:bg-[#B91C1C]
                   rounded-lg font-medium transition-colors duration-200">
  删除行程
</button>

// Ghost按钮 Ghost — 极简场景，弱化视觉权重
<button className="bg-transparent text-[#4A5568] hover:bg-gray-100
                   rounded-lg font-medium transition-colors duration-200">
  返回
</button>

// 金色按钮 Gold — 宗教文化特色CTA，如"开启朝圣"
<button className="bg-[#D4A855] text-white hover:bg-[#A8832A]
                   rounded-lg font-medium transition-colors duration-200">
  开启朝圣之旅
</button>
```

#### 按钮铁律

```
[B-01] 主按钮每屏最多出现1个，引导最重要操作
[B-02] 危险按钮必须有二次确认弹窗
[B-03] 加载状态必须禁用按钮并显示spinner
[B-04] 按钮文字不超过8个中文字
[B-05] 禁止纯色背景+相近色文字(对比度不足)
```

### 6.4 导航 Navigation

```
顶部导航 (Web):
  - 高度: 64px (h-16)
  - 背景: 白色(亮色) / #0f172a(暗色)

  - 主导航链接: 14px Medium，激活态底部蓝色2px下划线
  - 右侧操作区: 语言切换 + 登录/头像
  - 滚动后: 添加 shadow-sm

底部标签栏 (Mobile):
  - 高度: 56px + safe-area-bottom
  - 背景: 白色(亮色) / #0f172a(暗色)
  - Tab项: 图标24px + 文字11px
  - 激活色: #1A73E8(蓝) 或宗教专属色
  - 顶部分割线: 1px #E5E7EB

面包屑 Breadcrumb:
  - 字号: 13px
  - 颜色: #9CA3AF → 当前页 #1A1A2E
  - 分隔符: " › " (U+203A)
  - 最大层级: 4层
```

## 10. 动效规范

> 参考 Airbnb 和 Trip.com 的动效标准: 流畅自然，辅助内容理解，禁止炫技式动画。

### 10.1 过渡时长

| 场景 | 时长 | Easing | 说明 |
|------|------|--------|------|
| 颜色/透明度变化 | 150ms | ease-out | 按钮hover、链接颜色 |
| 位置/尺寸变化 | 200ms | ease-out | 卡片hover放大 |
| 内容展开/收起 | 250ms | ease-out | 手风琴、下拉菜单 |
| Modal / 抽屉 | 250ms | ease-out | 弹窗进场 |
| 页面路由切换 | 300ms | ease-in-out | 页面级动画 |
| 骨架屏脉冲 | 1500ms | ease-in-out | 无限循环 |

### 10.2 标准动效类

```css
/* Tailwind 基础过渡类 */
.transition-base { @apply transition-all duration-200 ease-out; }
.transition-colors { @apply transition-colors duration-150 ease-out; }
.transition-transform { @apply transition-transform duration-200 ease-out; }

/* 卡片hover效果 */
.card-hover {
  @apply hover:shadow-xl hover:-translate-y-1
         transition-all duration-200 ease-out;
}

/* 图片hover缩放 */
.img-hover {
  @apply group-hover:scale-105 transition-transform duration-300 ease-out;
}
```

### 10.3 动效禁止事项

```
[A-01] 禁止无意义的旋转/闪烁动画
[A-02] 禁止超过 400ms 的用户操作响应动效
[A-03] 禁止连续/循环动画用于非骨架屏场景
[A-04] 用户点击后必须在 100ms 内有视觉反馈
[A-05] 禁用 animation 时(prefers-reduced-motion)必须降级为无动画
[A-06] 列表滚动不得有惯性以外的特效
```

```css
/* 无障碍: 减少动效偏好 */
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

## 12. 多端适配原则

### 12.1 Web (Next.js 15 + TailwindCSS v4)

```
核心原则:
  - 使用 App Router + Server Components 优化性能
  - Tailwind 原子类，禁止写内联 style(动态色值除外)
  - 暗色模式: darkMode: 'class'，通过 ThemeProvider 切换
  - 图片: 必须用 next/image，开启 WebP/AVIF 自动转换
  - 字体: next/font 自动优化，避免 FOUT

禁止事项:
  - 禁止在 Server Component 中使用 useState/useEffect
  - 禁止在页面级组件写超过 100 行的样式逻辑
  - 禁止用 @apply 嵌套超过3层
```

### 12.2 Mobile (React Native + Expo 52)

```
核心原则:
  - 使用 StyleSheet.create() 而非内联对象样式(性能)
  - 间距/字号使用 scale 函数适配不同屏幕密度
  - 颜色通过 useColorScheme() 动态切换，默认暗色
  - 图片使用 expo-image，支持 contentFit 和 blurhash 占位
  - 触摸区域最小 44×44pt (iOS HIG 标准)

禁止事项:
  - 禁止硬编码 px 尺寸，必须用 Dimensions.get 或 scale
  - 禁止在 FlatList renderItem 中创建内联函数
  - 禁止滥用 zIndex，超过3层 zIndex 需要评审
```

### 12.3 微信小程序 (Taro 4.x + React)

```
核心原则:
  - 单位使用 rpx (1rpx = 0.5px@750设计稿)
  - 基础组件优先使用 Taro 内置组件(跨端兼容)
  - 图片使用 Taro Image 组件，支持懒加载
  - 暗色适配通过 Taro 的 useTheme 或 CSS Variables

禁止事项:
  - 禁止使用 DOM API (document/window)
  - 禁止单个页面超过 16 个自定义组件
  - 禁止 setData 数据超过 1MB
```

### 12.4 Admin 后台 (Vite + Ant Design 5)

```
核心原则:
  - 严格使用 Ant Design 组件，禁止重复造轮子
  - 主题色通过 ConfigProvider token 统一配置:
    colorPrimary: '#1A73E8'
    colorWarning: '#D4A855' (金色用于宗教文化标记)
  - 表格必须支持排序/筛选/分页
  - 表单必须用 Form + rules 校验

禁止事项:
  - 禁止在 Admin 中引入 Tailwind (避免样式冲突)
  - 禁止覆盖 Ant Design 全局样式
  - 禁止在同一页面同时使用 Modal 和 Drawer(选其一)
```

### 12.5 跨端一致性要求

```
[CROSS-01] 视觉语言一致: 同一功能在4端使用相同颜色、图标语义
[CROSS-02] 数据展示一致: 日期格式 YYYY-MM-DD，货币 ¥#,###
[CROSS-03] 错误提示一致: 相同错误码显示相同文案
[CROSS-04] 加载状态一致: 骨架屏样式各端自适应，但交互逻辑统一
[CROSS-05] 空状态一致: 图示+说明文字+操作按钮三件套
```
