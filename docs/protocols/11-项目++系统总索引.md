## §0 系统概述

**项目++** 是一套**多层级质量审计与标准生成系统**，由10个专业化++技能组成，通过联动矩阵协同工作，覆盖从代码质量到业务逻辑、从用户体验到安全防护的全维度审查。

**核心理念**:
- 每个++技能是独立的 Inspector → Executor → Verifier 三角
- 全审++ 是顶层编排器，按需调用子++
- 所有++共享统一骨架 §0-§9
- 评分体系: ≥4.0 PASS / 3.0-3.9 PARTIAL / <3.0 FAIL

---

## §1 统一骨架结构 (§0-§9)

每个++技能文件必须包含以下9个标准Section:

```
§0 元数据 — name / version / triggers / 前置条件 / 后置动作 / 联动技能
§1 角色定义 — Inspector(发现) / Executor(修复) / Verifier(验证) 三角
§2 作用域 — 技能适用的模块/页面/端范围 + 排除项
§3 领域规则 — 该++专有的检查项 / 反模式 / 铁律 (XX-01~XX-N)
§4 执行流程 — Step 0→N 分步编排 (含工具调用)
§5 评分机制 — N维 × 5⭐ 评分 + 通过阈值 + 输出模板
§6 工具治理 — 允许使用的工具清单 + 检测命令速查
§7 生命周期 — L1触发 → L2执行 → L3输出 → L4反馈 → L5沉淀
§8 铁律清单 — 该++不可违反的硬性规则
§9 联动点 — 与其他++的接口协议 (输入/输出/触发条件)
```

## §5 执行协议

### 5.1 全审执行流 (hohoho)

```
Step 0: 读取 CLAUDE.md + 本索引 + 目标模块PRD
Step 1: RRR快速自检 → 发现P0阻断问题
Step 2: 业务++ → 验证数据/逻辑/PRD对齐
Step 3: 盲区++ → 扫描隐性缺陷(死按钮/伪空态/链路)
Step 4: 地狱++ → 深层安全(IDOR/Guard/事务/状态机)
Step 5: 体验++ → 用户感知(三态/响应式/i18n/性能)
Step 6: 飞轮++ → 闭环验证(事件链/副作用/数据一致)
Step 7: 五全++ → 跨端一致性检查
Step 8: 页面++ → PRD竞品对标(如需升级)
Step 9: 汇总 FA 评分 → 输出审计报告
```

### 5.2 快速模式 (X+/GOGOGO)

```
Step 0: RRR 3分钟速查
Step 1: 盲区++ 重点扫描
Step 2: 地狱++ 安全速查
Step 3: 修复 → tsc验证 → commit
```

### 5.3 项目++生成流 (新功能开发)

```
Step 0: 接收PRD或功能规格
Step 1: 深度调研竞品(SCP-02)
Step 2: 生成DOC-XX标准文档(§0-§12格式)
Step 3: 生成施工图(管线++)
Step 4: 生成验收清单(闭环++)
Step 5: 交付 → 开始编码
```

## §7 铁律清单 (系统级)

```
[PP-01] 新建++技能必须遵循§0-§9统一骨架
[PP-02] 每个++必须有N维×5⭐评分体系
[PP-03] 每个++必须有Inspector/Executor/Verifier三角
[PP-04] 每个++必须有L1-L5生命周期闭环
[PP-05] 全审++的调用矩阵让每层知道调用哪个++
[PP-06] 评分前缀全局唯一，禁止冲突
[PP-07] 新规则必须标注来源(经验沉淀/CEO指令/事故复盘)
[PP-08] ++技能只能升级(提升版本号)，不能降级
[PP-09] 每次事故复盘必须产出至少1条新规则写入对应++
[PP-10] 项目++生成的标准文档必须关联PRD编号
```
