## 1. 页面概述

### 1.1 页面定位

首页"为你推荐"是JOINUS.COM首页的个性化内容区块，位于现有"热门圣地"区块之后。该区块根据用户登录状态与浏览历史提供差异化内容，是提升用户探索深度与粘性的核心入口。

### 1.2 区块在首页中的位置

```
首页布局 (从上到下):
  [1] Hero Banner — 品牌主视觉 + 搜索入口
  [2] 宗教分类入口 — 12大信仰横向图标导航
  [3] 热门圣地 — 编辑精选 (固定内容)
  [4] ★ 为你推荐 ← 本PRD负责的新区块
  [5] 继续浏览 — 最近浏览的内容 (见F07)
  [6] 附近热门 — 基于地理位置 (需授权)
  [7] 底部导航/页脚
```

## 2. 竞品对标

### 2.1 Trip.com "猜你喜欢"

- **区块标题**: "猜你喜欢" + 副标题"根据你的浏览偏好"
- **卡片布局**: 大图横向滚动，每次显示 2.5 张卡片 (露出半张暗示可滑动)
- **"换一批"按钮**: 右上角，点击刷新全部推荐，带旋转动画
- **加载态**: 骨架屏，卡片形状与实际卡片一致
- **JOINUS借鉴点**: 标题设计 + "换一批" + 骨架屏

### 2.2 Booking.com 首页推荐

- **登录态差异**: 已登录显示"欢迎回来，根据你的搜索偏好" + 个性化内容
- **未登录态**: 显示"热门目的地" + 当前季节热门
- **位置标签**: 每张卡片右上角显示地区标签

### 2.3 Airbnb 首页

- **分类筛选器**: 推荐区块上方有"海滩/山区/城市"等快速筛选 Tab
- **无限滚动**: 推荐内容可以持续刷新加载
- **JOINUS借鉴**: 宗教类型筛选 Tab (佛教/道教/基督教...)

## 3. 用户故事

```
作为已登录用户 (有浏览历史):
  我打开首页时，看到"为你推荐"区块
  区块内容与我最近浏览的宗教类型相关
  我可以点击"换一批"刷新推荐，不需要刷新整个页面

作为已登录用户 (无浏览历史):
  我打开首页时，"为你推荐"区块显示当前平台热门内容
  提示"继续浏览后将获得个性化推荐"

作为未登录用户:
  我打开首页时，"为你推荐"区块显示热门圣地/祖庭
  区块标题为"热门推荐"(不显示"为你")
```

## 5. 交互设计

### 5.1 "换一批"按钮

```
触发: 用户点击"换一批 ↻"按钮
动作:
  1. 按钮图标开始旋转动画 (360° rotation, 500ms)
  2. 当前卡片淡出 (opacity 0, 200ms)
  3. 发起 GET /api/recommendations/homepage?refresh=true 请求
  4. 新卡片淡入 (opacity 1, 300ms)
  5. 按钮动画停止
规则:
  - 新一批不重复上一批内容 (后端维护 session 级排除列表)
  - 连续点击防抖: 500ms 内不重复触发
  - 若已无更多不重复内容，隐藏"换一批"按钮
```

### 5.2 宗教筛选 Tab

```
默认状态: "全部" Tab 激活
点击 Tab: 过滤当前推荐列表 (前端过滤，不重新请求)
Tab 列表: 全部 | 佛教 | 道教 | 基督教 | 伊斯兰教 | 印度教 | 更多
"更多" 展开: 显示全部 12 个宗教分类
```

### 5.3 卡片点击

```
点击卡片主体 → 跳转到对应详情页
点击收藏按钮 → 触发收藏/取消收藏 (见M14)
  已登录: 直接收藏，按钮变实心♥ + 短暂缩放动画
  未登录: 弹出登录引导弹窗
```

### 5.4 空状态处理

```
IF 推荐数为0 (极少见):
  显示: "暂无推荐，探索全部圣地 →" (链接到圣地列表页)
  不显示整个区块标题
```

## 6. 数据流

### 6.1 已登录用户

```
1. 首页加载 → 携带 Authorization Token 请求 /api/recommendations/homepage
2. 后端读取 user:views:{userId} (Redis) → 分析偏好宗教
3. 命中缓存 recommendations:homepage:{userId} → 直接返回 (TTL 30min)
4. 未命中缓存 → 执行算法V1 → 写入缓存 → 返回结果
5. 前端渲染骨架屏 → 接收数据 → 渐入显示
```

### 6.2 未登录用户

```
1. 首页加载 → 无 Token → 请求 /api/recommendations/popular
2. 后端命中 recommendations:popular:all (Redis, TTL 6h) → 直接返回
3. 前端区块标题显示"热门推荐"(非"为你推荐")
```

## 7. API 调用

```
GET /api/recommendations/homepage
Headers: Authorization: Bearer {token} (可选)
Params:
  limit: 10 (默认)
  refresh: false (换一批时为 true)

Response:
{
  "items": [...],  // 推荐卡片列表
  "personalized": true,  // 是否个性化
  "total": 10,
  "algorithm": "content-based-v1"
}
```

## 8. 性能要求

| 指标 | 要求 |
|------|------|
| 首屏内容渲染 (首页整体) | ≤ 2s (LCP) |
| 推荐区块骨架屏出现 | ≤ 100ms |
| 推荐数据加载完成 | ≤ 300ms (Redis命中) |
| "换一批"响应 | ≤ 500ms |
| 图片懒加载 | 仅加载视口内图片 |

- [ ] 卡片点击正确跳转到对应详情页
