# P03 收藏夹列表页 产品需求文档

> 版本: v1.0 | 日期: 2026-03-29 | 状态: 待开发 | 父模块: M14-收藏系统
> 路由: /collections (Web) | /collections (Mobile) | /pages/collections (小程序)

---

## 1. 需求背景与目标

收藏夹列表页是用户管理所有收藏夹的入口，对标 Airbnb Wishlist 列表页。用户在此创建主题收藏夹（如「华东佛教祖庭之旅」）、查看各收藏夹内容概览、以及执行分享/删除等管理操作。

**页面目标：**
- 用户能在 10 秒内找到目标收藏夹或创建新收藏夹
- 收藏夹卡片封面图直观呈现内容主题，降低认知负担
- 空状态页面能有效引导新用户完成首次收藏行为

---

## 2. 用户故事

### US-01 查看所有收藏夹

**作为** 拥有多个主题收藏夹的朝圣者，
**我希望** 在列表页看到所有收藏夹的封面图、名称和项目数量，
**以便** 快速定位我要查看的收藏夹。

**验收标准：**
- [ ] 网格卡片展示所有收藏夹（Web 3列，Mobile 2列，小程序列表）
- [ ] 每张卡片显示：封面图、名称、项目数、公开/私密标记
- [ ] 「默认收藏夹」排在第一位，带有系统标识角标
- [ ] 加载态显示骨架屏，错误态显示重试按钮

### US-02 创建新收藏夹

**作为** 想要新建主题收藏夹的用户，
**我希望** 点击「新建收藏夹」按钮后填写名称和设置，一步完成创建，
**以便** 快速开始收集内容。

**验收标准：**
- [ ] 「新建收藏夹」按钮位于页面顶部右侧（Web）或底部浮动按钮（Mobile）
- [ ] 弹窗含名称（必填）、描述（选填）、封面图上传（选填）、公开开关
- [ ] 名称不能为空，不超过 50 字，超出时实时提示剩余字数
- [ ] 创建成功后，列表自动刷新，新收藏夹卡片以动画插入
- [ ] 收藏夹数达 50 个时按钮置灰，Tooltip 提示「已达上限」

### US-03 管理收藏夹（编辑/删除/分享）

**作为** 想要整理收藏夹的用户，
**我希望** 通过长按或右键菜单对收藏夹进行编辑、删除、分享操作，
**以便** 保持收藏夹整洁有序。

**验收标准：**
- [ ] Web：鼠标悬停卡片时右上角显示「...」更多菜单按钮
- [ ] Mobile：长按卡片弹出操作菜单（编辑 / 分享 / 删除）
- [ ] 菜单选项：编辑（弹出编辑弹窗）、分享（仅公开夹可用）、删除
- [ ] 删除需二次确认弹窗，确认后移除，默认收藏夹删除按钮不可用并置灰
- [ ] 删除成功后，卡片以动画淡出，列表重排

## 4. 交互细节

### 4.1 收藏夹卡片交互

| 交互 | Web 触发方式 | Mobile 触发方式 | 结果 |
|------|-------------|-----------------|------|
| 进入详情 | 点击卡片主体 | 点击卡片主体 | 跳转收藏夹详情页 |
| 更多菜单 | 悬停后点击「···」 | 长按 300ms | 弹出操作菜单 |
| 拖拽排序 | 拖拽卡片（桌面） | 长按后拖拽 | 重新排序（前端本地，PATCH 保存） |
| 快速分享 | 菜单内「分享」 | 菜单内「分享」 | 复制链接 + Toast |

### 4.2 封面图逻辑

- 用户未上传封面图时：自动取收藏夹内第一个内容项的图片
- 收藏夹为空时：显示默认渐变占位图（使用信仰主题色渐变）
- 封面图宽高比固定为 4:3，超出部分裁切

### 4.3 空状态设计

**首次使用（无任何收藏夹）：**
```
        [朝圣图标]
    你还没有收藏夹
  开始收藏感兴趣的圣地
[浏览圣地]   [新建收藏夹]
```

**有默认收藏夹但为空：**
- 默认收藏夹卡片正常显示，内部详情页显示「尚未收藏任何内容，去浏览圣地吧」

## 5. 响应式布局规范

| 断点 | 布局 | 卡片尺寸 |
|------|------|----------|
| Web ≥ 1024px | 3 列网格，gap-6 | 宽度自适应 |
| Web 768-1023px | 2 列网格，gap-4 | 宽度自适应 |
| Web < 768px | 2 列网格，gap-3 | 宽度自适应 |
| Mobile (React Native) | 2 列 FlatList | 卡片宽 = (屏幕宽 - 48) / 2 |
| 小程序 (Taro) | 单列列表 | 全宽卡片，左图右文布局 |

## 6. API 调用

| 操作 | API | 触发时机 |
|------|-----|----------|
| 加载列表 | GET /api/collections | 页面挂载 |
| 创建收藏夹 | POST /api/collections | 创建弹窗确认 |
| 更新收藏夹 | PATCH /api/collections/:id | 编辑弹窗确认 |
| 删除收藏夹 | DELETE /api/collections/:id | 删除二次确认后 |
| 更新排序 | PATCH /api/collections/:id | 拖拽结束后（防抖 500ms） |

## 7. 验收标准

| 标准编号 | 描述 |
|----------|------|
| AC-01 | 页面首屏加载 < 1.5s (P95)，骨架屏在数据返回前全程展示 |
| AC-02 | 创建弹窗名称字段实时显示剩余字数（50字上限），超出时禁止提交 |
| AC-03 | 删除操作必须有二次确认，默认收藏夹的删除按钮置灰且不可点击 |
| AC-04 | 收藏夹封面图自动回落逻辑正确（用户图 → 第一项图 → 默认占位图） |
| AC-05 | Web 3列、Tablet 2列、Mobile 2列、小程序列表，4种布局均正常渲染 |
| AC-06 | 创建成功后新卡片动画插入列表，无需手动刷新 |
| AC-07 | 长按/悬停菜单对默认收藏夹隐藏「删除」选项 |
| AC-08 | 收藏夹数量 ≥ 50 时「新建」按钮置灰，Tooltip 提示达到上限 |
