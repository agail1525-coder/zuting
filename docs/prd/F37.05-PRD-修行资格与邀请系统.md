# F37.05-PRD-修行资格与邀请系统

> M37 圆满之路修行系统的**门神** | 三路径授权 + 五级角色 + 邀请圈

## 1. 目标
控制 M37 修行子系统的访问权限, 确保:
- 主站旅行网站零干扰
- 修行圈纯净度
- 内容敏感性可控
- 商业化基础 (稀缺 = 价值)

## 2. 五级角色体系

```
NONE          无资格 (默认所有用户)
  ↓ 申请通过 / 邀请码 / 付费
SEEKER        求道者 (申请待审中间态)
  ↓ admin 通过
PRACTITIONER  修行者 (基础权限, 可用七轴A1-A7观看)
  ↓ 积分/境界/推荐
MENTOR        导师 (可邀请他人, 每月5张邀请码)
  ↓ 实名认证 + 导师考核
MASTER        大师 (可开活佛直播堂)
```

## 3. 三路径获取资格

### 3.1 路径1: 管理员授权 (主流)
```
用户 → 申请页 → 提交 CultivationApplication
  ↓
Admin 审核 → APPROVED → User.cultivationAccess = true, role = PRACTITIONER
  ↓
触发邮件/站内信通知: "恭喜, 你的修行资格已开启"
```

### 3.2 路径2: 导师邀请
```
MENTOR+ 用户 → POST /cultivation/invite/generate
  → 系统生成邀请码 (CultivationInviteCode, 7天有效)
  → 生成专属链接 https://joinus.com/trips/cultivation/apply?code=xxx
  ↓
受邀人点击链接 → 自动预填申请表 + 标记来源
  → 跳过 admin 审核, 自动授权 PRACTITIONER
  ↓
邀请人获得奖励: 积分 + 邀请贡献值
```

### 3.3 路径3: 付费晋级
```
用户完成 M36 信仰力评估 (分数 ≥ 60)
  ↓
在申请页看到「付费晋级」按钮 → 跳转 /orders/checkout?sku=CULTIVATION_YEAR
  ↓
支付成功 → Webhook 自动授权 PRACTITIONER (有效期1年)
  ↓
到期前 30 天提醒续费, 过期自动降级 NONE (数据保留)
```

## 4. 数据模型

```prisma
enum CultivationRole {
  NONE
  SEEKER
  PRACTITIONER
  MENTOR
  MASTER
}

model User {
  // 现有字段保留
  cultivationAccess     Boolean         @default(false)
  cultivationRole       CultivationRole @default(NONE)
  cultivationGrantedBy  String?
  cultivationGrantedAt  DateTime?
  cultivationExpiresAt  DateTime?       // 付费通道年度有效期
}

model CultivationApplication {
  id          String   @id @default(cuid())
  userId      String
  motivation  String   @db.Text
  tradition   String?
  experience  String?  @db.Text
  source      String   @default("MANUAL") // MANUAL|INVITE|PAID
  inviteCodeId String?
  status      String   @default("PENDING") // PENDING|APPROVED|REJECTED|AUTO_APPROVED
  reviewedBy  String?
  reviewNote  String?  @db.Text
  submittedAt DateTime @default(now())
  reviewedAt  DateTime?
  user        User     @relation(fields: [userId], references: [id])
  @@index([status, submittedAt])
}

model CultivationInviteCode {
  id          String   @id @default(cuid())
  code        String   @unique @default(cuid())
  issuerId    String
  assigneeId  String?
  note        String?
  usedAt      DateTime?
  expiresAt   DateTime
  createdAt   DateTime @default(now())
  issuer      User     @relation("InviteIssuer", fields: [issuerId], references: [id])
  @@index([issuerId])
  @@index([code])
}

model CultivationAccessAudit {
  id          String   @id @default(cuid())
  userId      String
  action      String   // GRANT|REVOKE|UPGRADE|DOWNGRADE|EXPIRE
  fromRole    CultivationRole?
  toRole      CultivationRole?
  operatorId  String?  // admin 或系统
  reason      String?  @db.Text
  createdAt   DateTime @default(now())
  @@index([userId, createdAt])
}
```

## 5. API 契约

### 5.1 申请与查询
```
POST   /cultivation/apply
Body:  { motivation, tradition?, experience?, inviteCode? }
Auth:  JwtAuthGuard
Logic:
  1. 校验用户未 PRACTITIONER+ (已授权不能重复申请)
  2. 若 inviteCode 提供:
     - 校验有效性/未过期/未使用
     - auto-approve, 授权 PRACTITIONER, 标记 code 已用
  3. 否则入 PENDING 队列
Return: { applicationId, status }

GET    /cultivation/apply/mine
Auth:  JwtAuthGuard
Return: { application, currentRole }
```

### 5.2 管理员端点
```
GET    /admin/cultivation/applications
Query: status?, page, pageSize (Max 50)
Auth:  JwtAuthGuard + RolesGuard(SUPER_ADMIN)

POST   /admin/cultivation/applications/:id/approve
Body:  { note?, role: PRACTITIONER|MENTOR }
Logic: 更新 User 字段, 写审计, 发通知

POST   /admin/cultivation/applications/:id/reject
Body:  { reason }

POST   /admin/cultivation/grant
Body:  { userId, role, reason }
Logic: 绕过申请直接授权

POST   /admin/cultivation/revoke
Body:  { userId, reason }
Logic: 置 cultivationAccess = false, role = NONE, 写审计

GET    /admin/cultivation/users?role=PRACTITIONER
```

### 5.3 邀请
```
POST   /cultivation/invite/generate
Body:  { count: 1-5, note? }
Auth:  JwtAuthGuard + CultivationRoleGuard(MENTOR)
Logic:
  1. 校验本月已发 < 5
  2. 生成 count 张 InviteCode
Return: [{ code, shareUrl, expiresAt }]

POST   /cultivation/invite/redeem
Body:  { code }
Auth:  JwtAuthGuard
Logic: 同 /cultivation/apply 的 inviteCode 分支

GET    /cultivation/invite/mine
Return: { generated: [], used: [], remaining: 3 }
```

### 5.4 付费晋级 (与 M06 订单联动)
```
Webhook: order.paid event
  → 若 sku = CULTIVATION_YEAR
  → 授权 PRACTITIONER, cultivationExpiresAt = now + 1year
  → 写审计
```

## 6. 守卫实现

```typescript
// cultivation-access.guard.ts
@Injectable()
export class CultivationAccessGuard implements CanActivate {
  canActivate(ctx: ExecutionContext): boolean {
    const req = ctx.switchToHttp().getRequest();
    const user = req.user;
    if (!user) throw new UnauthorizedException();
    if (!user.cultivationAccess) {
      throw new ForbiddenException({
        code: 'CULTIVATION_REQUIRED',
        message: '需要修行资格',
        applyUrl: '/trips/cultivation/apply',
      });
    }
    // 过期检查
    if (user.cultivationExpiresAt && new Date() > user.cultivationExpiresAt) {
      throw new ForbiddenException({ code: 'CULTIVATION_EXPIRED' });
    }
    return true;
  }
}

// cultivation-role.guard.ts (角色升级门)
export const CultivationRole = (...roles: CultivationRole[]) =>
  SetMetadata('cultivationRoles', roles);
```

## 10. 安全铁律
- [R-63] 所有 /cultivation/* 路由 JwtAuthGuard 前置
- [R-64] Admin 列表 @Max(50)
- [R-68] /apply/mine 只能查自己的
- [Rate Limit] /cultivation/apply 10/天/用户 防刷
- [XSS] motivation/experience/reason DOMPurify 清洗
- [Invite] 邀请码单次使用 + 7天过期 + 每月 5 张上限
- [Audit] 授权变更强制写入 CultivationAccessAudit 无漏网
- [Expire CRON] 每日凌晨扫描 cultivationExpiresAt 过期用户, 自动 DOWNGRADE NONE
