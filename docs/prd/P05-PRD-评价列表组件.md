## 1. 背景与目标

### 1.1 背景

评价列表组件是 UGC 评价系统的核心展示载体，嵌入圣地详情页、祖庭详情页和行程详情页底部。当前各详情页缺乏可信的社区评价展示，用户在朝圣决策前无法参考他人真实体验，对比 TripAdvisor 和 Booking.com 存在明显功能差距。

### 1.2 目标

- [ ] 实现通用评价列表组件，可嵌入任意实体详情页（圣地/祖庭/行程）
- [ ] 展示评分汇总区（平均分 + 星级分布柱状图）和评价卡片列表
- [ ] 支持三种排序方式，加载更多分页，完整三态处理

### 1.3 非目标（本期不做的）

- 不包含评价提交功能（由 P06 负责）
- 不包含评价举报交互（延期到 Phase C）
- 不包含照片墙独立展示区（延期到 P2）

## 2. 竞品对标分析

### 2.1 竞品功能对比

| 竞品 | 功能描述 | 优点 | 缺点 | 借鉴点 |
|------|---------|------|------|-------|
| TripAdvisor | 评分摘要（总分+分布）+ 筛选标签 + 卡片列表 + "有用"按钮 | 层次清晰，分布图直观 | 页面较重，加载慢 | 星级分布百分比柱状图、"有用"计数展示 |
| Booking.com | 数字评分（0-10）+ 正面/负面标签 + 官方回复内嵌 | 正负分离直观，官方回复样式优雅 | 评分制与星级不直观 | 官方回复嵌入卡片底部的设计 |
| 大众点评 | 评分+热门标签词云+图片优先展示 | 图片吸引力强，标签提取方便浏览 | 文字评价沦为次要 | 图片网格在卡片内的展示比例 |

### 2.2 差距分析

JOINUS.COM 当前详情页无评价展示，缺少星级分布可视化和用户互动（有用投票），导致内容页面缺乏可信度背书和社区温度。

### 2.3 差异化优势

评价卡片附加宗教信仰标签（用户所属宗教），帮助同信仰朝圣者快速找到共鸣评价，形成垂直社区归属感。

## 3. 用户故事

### 3.1 目标用户

- 主要用户: 正在浏览圣地/祖庭详情的潜在朝圣者
- 次要用户: 已完成朝圣的用户（查看自己的评价状态）

| 编号 | 用户故事 | 优先级 |
|------|---------|-------|
| US-01 | 作为潜在朝圣者，我希望看到该圣地的平均评分和评价数量，以便快速判断朝圣价值 | P0 |
| US-02 | 作为潜在朝圣者，我希望看到1-5星各档次的数量分布，以便判断评价是否集中 | P0 |

| US-05 | 作为浏览者，我希望点击评价中的图片可以全屏预览，以便查看真实的圣地照片 | P1 |

## 4. 功能清单

### 4.1 核心功能（P0 — 必须有）

- [ ] RatingSummary 区块：平均分大数字展示、总评价数、1-5星分布柱状图+百分比
- [ ] ReviewList 评价卡片列表：头像+昵称+星级+时间+内容+图片网格
- [ ] 排序切换：最新（默认）/ 最有用 / 最高分（Tab 或 Select 组件）
- [ ] 加载更多：滚动到底部或点击"加载更多"按钮触发分页
- [ ] 三态处理：Loading 骨架屏 / Empty 引导 / Error 重试

### 4.2 增强功能（P1 — 应该有）

- [ ] 有用投票按钮：每条评价右下角显示"有用(N)"按钮，已投票变色
- [ ] 图片点击预览：评价图片网格支持点击全屏预览（Lightbox）
- [ ] 官方回复展示：评价卡片底部嵌入官方回复区块（带标记）
- [ ] 待审核状态提示：当前用户自己的 PENDING 评价在列表顶部显示，附"审核中"灰色标签

### 4.3 优化功能（P2 — 可以有）

- [ ] 无限滚动（替代"加载更多"按钮，Intersection Observer 实现）
- [ ] 评价文字折叠：超过4行时显示"展开全文"按钮
- [ ] 宗教信仰标签：评价者所属信仰显示在昵称旁

## 6. API 契约

### 6.1 使用的 API 端点

| 端点 | 方法 | 用途 |
|------|------|------|
| `/api/reviews` | GET | 获取评价列表（分页+排序） |
| `/api/reviews/stats` | GET | 获取评分汇总（平均分+分布） |
| `/api/reviews/:id/vote` | POST | 标记"有用" |
| `/api/reviews/:id/vote` | DELETE | 取消"有用" |

### 6.2 请求示例

```
GET /api/reviews?entityType=HOLY_SITE&entityId=clxxx&sort=newest&page=1&pageSize=10
GET /api/reviews/stats?entityType=HOLY_SITE&entityId=clxxx
```

详细请求/响应格式见 M15-PRD-UGC评价系统.md 第 6.2 节。

## 7. 数据模型

本组件为纯展示层，不新增数据模型。数据来源见 M15 第 7 节。

组件所需的 TypeScript 类型（在 @zuting/shared-types 中定义）：

```typescript
interface ReviewItem {
  id: string;
  userId: string;
  userNickname: string;
  userAvatar?: string;
  rating: number;           // 1-5
  title?: string;
  content: string;
  photos: string[];
  helpfulCount: number;
  isHelpful: boolean;       // 当前用户是否已投票（未登录为 false）
  status: 'PENDING' | 'APPROVED';  // 仅显示自己的 PENDING 和所有 APPROVED
  createdAt: string;
  replies: ReviewReplyItem[];
}

interface ReviewReplyItem {
  id: string;
  isOfficial: boolean;
  content: string;
  createdAt: string;
}

interface ReviewStats {
  avgRating: number;
  reviewCount: number;
  distribution: Record<'1' | '2' | '3' | '4' | '5', number>;
}
```

## 8. 技术约束

- **性能要求**: 组件首次渲染（含 API 请求）< 2s；骨架屏在 100ms 内出现
- **移动端**: 375px 下布局正常，图片网格最多3列，超出显示"+N"
- **兼容性**: Web 支持 Chrome 100+, Safari 15+
- **类型安全**: 禁止 any，ReviewItem 接口从 shared-types 导入
- **认证**: 有用投票需 JWT；列表和统计为公开接口
- **防抖**: 排序切换触发新请求时，100ms 防抖避免频繁请求
- **图片预览**: Web 端用 yet-another-react-lightbox 或自建 Modal；Mobile 端用 Expo ImageViewer

## 9. 验收标准

### 9.1 功能验收

| 编号 | 验收项 | 测试步骤 | 通过标准 |
|------|-------|---------|---------|
| AC-01 | 评分汇总正确显示 | 访问有 48 条评价（含 32 条5星）的圣地详情页 | RatingSummary 显示平均分 4.7、总数 48、5星柱最长且显示 67% |
| AC-02 | 评价列表默认"最新"排序 | 首次加载评价列表 | 第一条评价的创建时间最新，与第二条相比 createdAt 更大 |
| AC-03 | 切换"最有用"排序 | 点击"最有用"Tab | 列表重新请求，helpfulCount 最高的评价排在第一位 |
| AC-04 | 加载更多功能 | 10条评价加载完后，点击"加载更多" | 追加新的评价卡片，不刷新已显示内容，按钮变为"加载中..." |
| AC-05 | 无评价空状态 | 访问无任何已通过评价的圣地 | 显示"还没有评价，来做第一个评价者吧"和"写评价"按钮 |
| AC-06 | 加载中骨架屏 | 网速限制为 Slow 3G，访问详情页 | 首次加载时显示骨架屏，不出现白屏或布局错位 |
| AC-07 | 网络错误重试 | DevTools 断网后访问评价区 | 显示错误提示和"重试"按钮，点击重试后重新请求 |
| AC-08 | 图片预览 | 点击评价卡片内图片 | 全屏 Lightbox 打开，可左右切换，点击外部关闭 |
| AC-09 | 有用按钮已登录 | 登录状态下点击"有用"按钮 | helpfulCount+1，按钮样式变为已激活，不可再次点击 |
| AC-10 | 官方回复展示 | 某评价有官方回复 | 评价卡片底部出现"官方回复"区块，显示"官方"标记和回复内容 |
| AC-11 | 移动端 375px | Chrome DevTools iPhone SE 模拟 | 无横向滚动条，图片网格最多3列，有用按钮可点击（≥44px） |
| AC-12 | 评价文字超4行折叠 | 评价内容超过4行 | 默认折叠显示4行，底部显示"展开全文"，点击后展开全部 |
