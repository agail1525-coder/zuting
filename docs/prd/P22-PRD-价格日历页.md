# P22-PRD-价格日历页

> 版本: v1.0 | 日期: 2026-03-29 | 状态: 已批准 | 父模块: M24-PRD-价格工具
> 对标: Skyscanner Fare Calendar | Kayak Flight Calendar

---

## 1. 页面定位

**URL**: `/prices/calendar?entityType=ROUTE&entityId=xxx`

**入口**:
- 路线详情页 → "查看价格日历" 按钮
- 套餐详情页 → "价格日历" 标签页
- 首页促销区 → "寻找最低价" CTA

**目的**: 帮助用户直观看到一个月内每天的价格高低，选择最划算的出发日期。

---

## 2. 竞品对标：Skyscanner Fare Calendar

### 2.1 Skyscanner 核心设计

- **日历网格**: 月视图，7列对应周一至周日
- **价格展示**: 每个有数据的日期格显示起价（元）
- **颜色编码**:
  - 深绿: 当月最低价区间（最低20%价位）
  - 浅绿: 较低价区间（20%-40%价位）
  - 白/中性: 正常价区间（40%-60%价位）
  - 橙/黄: 较贵区间（60%-80%价位）
  - 红: 最贵区间（80%+价位）
  - 灰: 无数据或已过期日期
- **选中状态**: 点击日期高亮蓝色边框
- **月份导航**: 左右箭头切换，不允许切到过去的月份

### 2.2 JOINUS.COM 差异化

- 显示"朝圣季"标记（特殊节日/旺季日期）
- 与宗教节历联动（如藏历节日、佛诞节等）
- 会员专属价显示（会员价比普通价低高亮）

## 3. 用户故事

> 作为朝圣者，我想在一个月视图中看到每天的朝圣套餐价格，点击某天后看到当天可购买的套餐列表，以便选择最实惠的出发日期。

**验收场景:**
1. 进入页面时默认显示当月价格日历
2. 已过去的日期显示灰色，不可点击
3. 有价格的日期按颜色区分高低
4. 点击某个有价格的日期，底部弹出当天套餐/路线列表
5. 点击"下月"切换到下个月数据

## 5. 颜色编码规则

| 价格分位 | 颜色 | CSS 色值 | 说明 |
|----------|------|----------|------|
| 无数据/已过期 | 灰 | `#6B7280` | 不可交互 |
| 最低 (≤ P25) | 深绿 | `#16A34A` | 推荐出行 |
| 较低 (P25-P50) | 浅绿 | `#86EFAC` | 较划算 |
| 正常 (P50-P75) | 金色 | `#D4A855` | 一般 |
| 较贵 (P75-P90) | 橙 | `#F97316` | 偏贵 |
| 最贵 (>P90) | 红 | `#DC2626` | 高峰价格 |

**算法**: 每次加载日历时，基于当月所有有价格日期计算百分位数。

## 6. 交互规范

### 6.1 月份导航

- 不允许导航到当前月之前（已过去的月份不展示）
- 最多允许查看未来 3 个月
- 切换月份时显示加载骨架屏（Skeleton）

### 6.2 日期点击

- 无数据/已过期日期: 不响应点击
- 有数据日期: 点击后高亮，底部面板滑出展示套餐/路线列表
- 再次点击同一日期: 收起底部面板

### 6.3 空状态

- 该月无任何价格数据 → 显示 "暂无价格数据，请联系客服获取报价"
- API 加载中 → 日历格显示 Skeleton 动画
- API 报错 → 显示重试按钮

### 6.4 "本月最低价"快捷按钮

页面顶部显示快捷入口：点击后自动滚动到并高亮本月最低价日期。

## 7. API 调用

```
GET /api/prices/calendar
  ?entityType=ROUTE
  &entityId={routeId}
  &startDate={YYYY-MM-01}
  &endDate={YYYY-MM-last}
```

**调用时机**: 进入页面时立即调用，切换月份时重新调用。
