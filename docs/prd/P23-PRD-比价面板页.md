# P23-PRD-比价面板页

> 版本: v1.0 | 日期: 2026-03-29 | 状态: 已批准 | 父模块: M24-PRD-价格工具
> 对标: Kayak Compare Tool | Booking.com 对比选项

---

## 1. 页面定位

**URL**: `/prices/compare?entityType=ROUTE&entityIds=id1,id2,id3`

**入口**:
- 路线列表页 → 勾选2-4条路线 → "对比" 按钮
- 套餐列表页 → 勾选2-4个套餐 → "对比" 按钮
- 路线/套餐详情页 → "加入对比" 悬浮按钮

**目的**: 帮助用户在多个候选路线/套餐之间做出最优决策，以价格为核心，辅以评分、内容等维度。

---

## 2. 竞品对标：Kayak Compare

### 2.1 Kayak 核心设计原则

- **最多5列**: 一行标签列 + 最多4个对比项
- **粘性表头**: 滚动时对比项名称固定在顶部
- **价格突出**: 最低价高亮，用徽标标记"最佳价值"
- **迷你价格图**: 每列底部显示近30天价格走势折线图（80px高）
- **一键预订**: 每列有独立的CTA按钮

### 2.2 JOINUS.COM 适配

- 对比维度增加"朝圣特色"（如：是否含导游讲解、宗教仪式体验）
- 会员价独立展示
- 价格提醒入口嵌入每列底部

## 3. 用户故事

> 作为朝圣者，我已收藏了3条感兴趣的路线，我想把它们放在一个页面并排对比价格、天数、包含服务和用户评分，以便做出最终预订决策。

**验收场景:**
1. 从路线列表勾选3条路线，点击"对比"进入比价页
2. 看到3列并排展示，每列包含核心信息
3. 最低价列自动高亮绿色徽标"最低价"
4. 点击"价格趋势"展开该路线近30天价格折线图
5. 点击任一列"立即预订"跳转到对应详情页

## 4. 对比维度清单

| 维度 | 说明 | 数据来源 |
|------|------|----------|
| 封面图 | 路线/套餐主图 | Route.coverImage / TravelPackage.coverImage |
| 名称 | 路线/套餐名称 | title |
| 当前价格 | 最新价格快照 | PriceSnapshot.price |
| 会员价 | 会员折扣价 | TravelPackage.memberPrice（路线暂无） |
| 30天最低价 | 近30天最低 | PriceSnapshot 聚合 |
| 30天均价 | 近30天平均 | PriceSnapshot 聚合 |
| 价格趋势图 | 近30天折线迷你图 | /api/prices/trend |
| 天数/晚数 | 行程时长 | Route.duration / TravelPackage.duration |
| 包含内容 | 含住、含餐、含导游等 | Route.included / TravelPackage.includes |
| 用户评分 | 星级评分 | Route.rating |
| 评价数量 | 评价总数 | Route.reviewCount |
| 难度 | 行程难度 | Route.difficulty |
| 最佳季节 | 推荐出行月份 | Route.season |
| 操作 | 立即预订 / 加入收藏 / 设置价格提醒 | — |

## 6. 交互规范

### 6.1 选择对比项

- 路线/套餐列表页支持多选（最多4条）
- 选中后底部出现悬浮条: "已选 X 条 [开始对比]"
- 进入比价页后可通过 "×" 移除某一列
- 可通过 "+ 添加" 回到列表页继续选择

### 6.2 价格高亮逻辑

- 所有列中当前价格最低者 → 绿色背景 + "最低价"徽标
- 当前价格高于30天均价10%以上 → 橙色警告图标
- 当前价格低于30天均价10%以上 → 绿色"好时机"图标

### 6.3 迷你趋势图

- 高度固定 80px，宽度填充单元格
- 折线颜色: 金色 `#D4A855`
- 悬停显示具体日期和价格 Tooltip
- 使用 recharts/Chart.js 轻量实现

### 6.4 包含内容对比

- 后端返回 included 数组
- 常见项目: 住宿/早餐/三餐/导游/交通/门票/保险
- 有: 绿色 ✓    无: 灰色 ✗

### 6.5 空状态

- 少于2条路线: 提示 "至少需要2条才能对比，请返回添加"
- 超过4条: 自动截取前4条，提示 "最多同时对比4条"

## 7. API 调用

```
GET /api/prices/compare
  ?entityType=ROUTE
  &entityIds=id1,id2,id3

GET /api/prices/trend
  ?entityType=ROUTE
  &entityId={id}
  &days=30
```

**调用时机**:
- 进入页面时: 调用 /compare（一次拿所有对比数据）
- 展开某列趋势图时: 按需调用 /trend（懒加载）

## 8. 响应式布局

| 断点 | 布局 |
|------|------|
| Mobile (< 768px) | 水平滚动表格，左列固定，对比项可左右滑动 |
| Tablet (768-1024px) | 最多显示3列并排 |
| Desktop (> 1024px) | 最多显示4列并排，表格居中 |
