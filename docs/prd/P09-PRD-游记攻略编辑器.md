## 1. 页面概述

### 1.1 页面定位

游记攻略编辑器（`/community/editor`）是 JOINUS.COM 社区的核心内容生产工具，供用户撰写和发布朝圣游记与攻略。编辑器支持富文本排版、内联图片、圣地实体关联，提供草稿自动保存与预览功能，发布后内容出现在社区 Feed 中。

### 1.2 编辑器在内容生产链路中的位置

```
用户旅程:
  社区首页 FAB [写游记]
       │
       ▼
  游记攻略编辑器（本页）
       │
       ├── 保存草稿 → 草稿箱（个人中心）→ 再次编辑 → 发布
       │
       └── 直接发布 → 内容审核（AI预审）→ 通过 → 社区 Feed
```

### 1.3 页面 URL

```
Web:    /community/editor          （新建）
        /community/editor/:id      （编辑草稿/已发布）
Mobile: /community/editor
小程序: /pages/community/editor/index
```

## 2. 竞品对标分析

### 2.1 马蜂窝游记编辑器

**调研要点:**
- **分步骤向导**: 编辑器分"基本信息 → 正文 → 发布设置"三步，降低创作门槛
- **目的地绑定**: 写作前必须先选择目的地，为内容打上地理标签
- **图片拖拽排序**: 正文中图片可拖拽重新排序，操作直观
- **字数统计**: 右下角实时显示字数，提示"至少需要200字才能发布"
- **标题字数限制**: 标题最多50字，有实时字数倒计时
- **JOINUS借鉴点**: 目的地绑定（→ 圣地关联）+ 字数统计 + 分步向导结构

### 2.2 Medium 编辑器

**调研要点:**
- **极简写作界面**: 隐藏所有工具栏，专注内容，工具栏仅在选中文字时浮现
- **斜线命令（/commands）**: 输入`/`唤起块类型菜单（标题/图片/引用/分割线）
- **自动保存**: 每次键入后 2 秒触发自动保存，顶部显示"已保存"
- **大字号标题**: 标题区域用超大字号（36px+），与正文区分明显
- **JOINUS借鉴点**: 浮动工具栏 + 斜线命令 + 极简风格 + 自动保存提示

### 2.3 Notion 块编辑器

**调研要点:**
- **块（Block）概念**: 每个段落/图片/标题都是独立块，可拖拽重排
- **块菜单（+按钮）**: 每行左侧有`+`按钮，点击插入新块
- **多媒体嵌入**: 支持嵌入视频链接（YouTube/Bilibili）、地图
- **/@提及**: 输入`@`可提及人物/页面，JOINUS可扩展为提及圣地
- **JOINUS借鉴点**: 块拖拽 + `@提及`圣地实体关联

### 2.4 小红书笔记编辑器

**调研要点:**
- **封面图强制要求**: 必须上传至少1张图片作为封面，无图片无法发布
- **多图拼贴**: 最多9张图片，自动生成拼贴封面预览
- **话题标签**: `#话题`自动弹出话题建议，点击添加
- **提醒贴纸**: 可在图片上添加文字贴纸，增加视觉趣味
- **JOINUS借鉴点**: 封面图强制要求 + `#话题`联想

### 2.5 TipTap (ProseMirror 基础) 技术选型依据

| 特性 | TipTap | Quill | Draft.js | Slate |
|------|--------|-------|----------|-------|
| 框架支持 | React/Vue | 原生 | React | React |
| 可扩展性 | 极高（Extension API）| 中等 | 低 | 高 |
| 协同编辑 | 支持（Y.js）| 不支持 | 不支持 | 需额外配置 |
| TypeScript | 完整支持 | 部分 | 部分 | 完整支持 |
| 生态成熟度 | 高（9k+ stars）| 高 | 中 | 中 |
| JOINUS 选用理由 | Extension 可扩展圣地@提及；Y.js 支持未来协同；TypeScript 契合项目规范 |

## 3. 用户故事

```
作为首次创作的用户:
  我点击社区首页的"写游记"FAB 后，进入编辑器页面
  我看到简洁的编辑界面，顶部有"标题"占位符，下方是正文区域
  我输入标题和正文内容，系统每2秒自动保存一次草稿
  我点击"上传封面"添加一张朝圣照片作为封面
  我通过@提及功能将游记与"五台山"圣地关联
  我点击预览按钮，看到游记的最终呈现效果
  我觉得满意，点击"发布"完成创作
  系统提示"发布成功，内容审核后即可在社区看到"

作为继续写草稿的用户:
  我在个人中心看到上次的未完成草稿
  我点击"继续编辑"，草稿内容完整恢复，光标在上次编辑位置
  我完成剩余内容后，更新标签并发布

作为编辑已发布游记的用户:
  我在个人中心找到自己已发布的游记，点击"编辑"
  我修改部分内容后点击"更新"
  更新后游记重新进入审核状态，原游记保持显示直到新版本通过审核

作为需要插入图片的用户:
  我在正文中点击工具栏的图片图标，或输入/image
  弹出图片上传面板，我从相册选择照片
  图片上传进度显示，完成后内联显示在光标位置
  我可以点击图片调整对齐方式（居左/居中/居右）

作为需要关联圣地的用户:
  我在正文中输入@，弹出圣地/祖庭搜索框
  我输入"少林"，搜索结果显示"少林寺"等相关实体
  我选择"少林寺"，正文中插入蓝色可点击的"@少林寺"链接
  预览时读者点击"@少林寺"会跳转到少林寺详情页
```

## 6. 交互设计

### 6.1 自动保存机制

```
触发时机:
  - 用户停止输入 2 秒后
  - 手动点击 Ctrl+S / Cmd+S
  - 切换到其他 Tab 或 App（visibilitychange 事件）
  - 点击"预览"按钮之前

保存流程:
  1. 状态栏显示"保存中..."（旋转图标）
  2. 调用 PATCH /api/posts/:id（草稿状态）
  3. 成功 → 显示"草稿 · 已保存 刚刚"
  4. 失败 → 显示"保存失败，将在下次操作时重试"（不阻断编辑）

本地备份（防止网络故障数据丢失）:
  - 使用 localStorage 每次键入后实时缓存
  - key: draft_post_{userId}_{postId}
  - 页面加载时检查本地缓存与服务器草稿时间戳，取最新
```

### 6.2 发布校验规则

```
点击"发布"或"确认发布":
  校验 #1 — 标题: 不能为空，不超过50字
  校验 #2 — 封面图: 必须上传至少一张
  校验 #3 — 正文字数: 不少于 200 字（P1阶段降为100字）
  校验 #4 — 信仰标签: 至少选择一个

任一校验不通过:
  → 滚动到未通过的字段
  → 字段边框变红色
  → 显示具体错误提示文字

全部通过:
  → 弹出发布确认弹窗（5.5节）
```

### 6.3 图片上传交互

```
触发方式:
  - 点击工具栏图片图标
  - 输入 /image
  - 直接粘贴剪贴板中的图片（Ctrl+V）
  - 拖拽文件到编辑器

上传流程:
  1. 文件校验（格式/大小）
  2. 显示上传进度（内联进度条覆盖在图片占位框上）
  3. 上传至 /api/uploads/image
  4. 成功 → 替换占位框为真实图片（带淡入动画）
  5. 失败 → 占位框显示红色边框 + "上传失败，点击重试"

图片限制:
  - 格式: JPG / PNG / WebP / GIF
  - 单张最大: 10MB（超出弹Toast提示"图片超过10MB，请压缩后上传"）
  - 数量上限: 正文内图片最多 20 张

图片操作（点击图片后）:
  浮现内联工具栏: [居左] [居中] [居右] | [替换] [删除]
```

### 6.4 @提及实体关联

```
触发: 在正文中输入 @ 字符
响应:
  1. 光标处弹出搜索浮层
  2. 用户继续输入关键字时，实时搜索 /api/entities?q={keyword}&types=holy-site,temple
  3. 搜索结果最多显示 5 条，包含实体类型图标和信仰标签
  4. 用户选择后插入行内节点: @{实体名称}（蓝色，可点击）
  5. 按 Escape 关闭浮层，继续正常输入

在编辑器内: @提及以蓝色粗体显示，点击时弹出实体信息卡片
在预览/发布后: @提及渲染为可点击链接，跳转至实体详情页

实体关联同步: 选择@提及时，自动将该实体添加到页面底部"关联圣地"区域
```

### 7.7 数据模型扩展

```prisma
// Post 模型扩展（补充编辑器所需字段）
model Post {
  // ... 已有字段（见P08-PRD）

  contentJson     Json?             // TipTap JSON 格式，用于恢复编辑状态
  wordCount       Int       @default(0)
  visibility      PostVisibility @default(PUBLIC)
  reviewStatus    ReviewStatus   @default(PENDING)
  reviewedAt      DateTime?
  linkedEntities  PostEntity[]
}

enum PostVisibility {
  PUBLIC
  FOLLOWERS
  PRIVATE
}

enum ReviewStatus {
  PENDING    // 待审核
  APPROVED   // 审核通过
  REJECTED   // 审核拒绝
  SKIPPED    // 跳过审核（管理员操作）
}

model PostEntity {
  id          String  @id @default(cuid())
  postId      String
  post        Post    @relation(fields: [postId], references: [id])
  entityType  String  // 'holy-site' | 'temple'
  entityId    String
  entityName  String
  @@unique([postId, entityType, entityId])
}
```

## 11. 安全约束

```
内容安全:
  - 发布时调用 AI 内容审核接口（POST /api/moderation/check）
  - 审核通过后 reviewStatus → APPROVED，内容在 Feed 可见
  - 审核中（PENDING）：内容在作者个人主页可见，社区 Feed 不显示
  - 审核拒绝（REJECTED）：通知作者，内容不发布

XSS 防护:
  - TipTap HTML 输出必须经过服务端 DOMPurify sanitize
  - 禁止 <script>/<iframe> 等危险标签
  - 图片 src 只允许 cdn.joinus.com 域名（CSP 配置）

IDOR 防护:
  - PATCH/POST/:id/publish 必须校验 authorId === currentUserId
  - 草稿列表 API 只返回当前用户的草稿
```
