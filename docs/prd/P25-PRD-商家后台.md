## 二、用户故事

### 2.1 核心用户故事

```
作为一名导游商家，
我希望登录商家后台后能一眼看到今日的订单数、收入和评分，
这样我可以快速掌握经营状况。

验收标准:
  - Dashboard展示: 今日订单数/本月收入/平均评分/总订单数
  - 待处理事项: 新订单(未确认)/新评价(未回复)/审核通知
  - 收入趋势图: 近7天/30天柱状图
```

```
作为一名住宿提供者，
我希望能方便地添加和管理我的房源服务，
包括设置价格、上传图片、管理可用日期。

验收标准:
  - 服务列表: 显示所有已添加服务，含名称/价格/状态(上架/下架)
  - 添加服务: 表单填写名称/描述/价格/单位/容量/图片
  - 上下架: 一键切换isActive状态
  - 编辑/删除: 可修改已有服务信息
```

```
作为一名寺庙管理者，
我希望能查看和处理分配给我的订单，
及时确认或拒绝预约请求。

验收标准:
  - 订单列表: 按状态筛选(待确认/进行中/已完成)
  - 订单详情: 用户信息/服务内容/日期/金额
  - 操作: 确认/拒绝订单，附备注
  - 通知: 新订单实时推送(站内信+声音提示)
```

### 2.2 边缘场景

```
场景: 商家状态为SUSPENDED
期望: 登录后显示暂停通知+整改要求，仅可查看历史数据，不可新增/修改服务

场景: 商家首次进入后台(APPROVED状态)
期望: 引导流程——添加至少一个服务→确认上线→状态变为ACTIVE
```

## 三、页面结构

### 3.1 导航结构

```
商家后台 (路由: /merchant/)
├── Dashboard        (/merchant/dashboard)     — 经营概览
├── 服务管理          (/merchant/services)       — 添加/编辑/上下架
│   └── 添加/编辑服务  (/merchant/services/new)
├── 订单管理          (/merchant/orders)         — 查看/处理订单
│   └── 订单详情      (/merchant/orders/:id)
├── 评价管理          (/merchant/reviews)        — 查看/回复评价
├── 财务中心          (/merchant/finance)        — 收入/佣金/结算
└── 商家设置          (/merchant/settings)       — 修改商家信息
```

### 3.2 技术方案

商家后台作为Admin后台的扩展模块实现，通过角色(MERCHANT)区分：
- 管理员登录 → 看到完整Admin后台
- 商家登录 → 看到商家后台(仅自己的数据)
- 共享: 认证体系、UI组件库(Ant Design 5)、API客户端

## 五、API依赖

### 5.1 商家后台使用的API端点

| 页面 | API端点 | 说明 |
|------|--------|------|
| Dashboard | GET /api/merchants/my | 商家基本信息 |
| Dashboard | GET /api/merchants/:id/stats | 统计数据 |
| 服务管理 | GET /api/merchant-services?merchantId=xxx | 服务列表 |
| 服务管理 | POST /api/merchant-services | 添加服务 |
| 服务管理 | PATCH /api/merchant-services/:id | 编辑服务 |
| 服务管理 | DELETE /api/merchant-services/:id | 删除服务 |
| 服务管理 | PATCH /api/merchant-services/:id/toggle | 上下架 |
| 订单管理 | GET /api/orders?merchantId=xxx | 商家订单列表 |
| 订单管理 | POST /api/orders/:id/confirm | 确认订单 |
| 订单管理 | POST /api/orders/:id/reject | 拒绝订单 |
| 评价管理 | GET /api/reviews?merchantId=xxx | 商家评价列表 |
| 评价管理 | POST /api/reviews/:id/reply | 回复评价 |
| 财务中心 | GET /api/merchants/:id/finance | 财务明细 |
| 设置 | PATCH /api/merchants/my | 更新商家信息 |

| 方法 | 端点 | 权限 | 说明 |

## 六、组件设计

### 6.1 核心组件

| 组件 | 说明 | Props |
|------|------|-------|
| MerchantDashboard | 经营概览面板 | merchantId |
| StatCard | 统计卡片(数字+趋势) | title, value, trend, icon |
| RevenueChart | 收入趋势图(Recharts) | data, period |
| ServiceCard | 服务卡片(列表项) | service, onEdit, onToggle, onDelete |
| ServiceForm | 服务添加/编辑表单 | initialData?, onSubmit |
| OrderTable | 订单表格(Ant Design Table) | filters, onConfirm, onReject |
| TodoList | 待处理事项列表 | items |

### 6.2 状态管理

使用React Query管理服务端状态：
- `useMerchantInfo()` — 商家基本信息
- `useMerchantStats(merchantId)` — 运营统计
- `useMerchantServices(merchantId)` — 服务列表
- `useMerchantOrders(merchantId, filters)` — 订单列表
- `useMerchantReviews(merchantId)` — 评价列表

## 七、验收标准

| # | 验收项 | 标准 |
|---|--------|------|
| 1 | Dashboard加载 | 进入后台 < 2s 显示统计数据和趋势图 |
| 2 | 服务CRUD | 添加/编辑/删除/上下架全流程可用 |
| 3 | 图片上传 | 服务图片支持多选上传、预览、删除，单张 < 5MB |
| 4 | 订单处理 | 可查看/筛选/确认/拒绝订单 |
| 5 | 数据隔离 | 商家只能看到自己的数据，IDOR防护 |
| 6 | SUSPENDED状态 | 暂停商家进入后台显示整改通知，禁止操作 |
| 7 | 首次引导 | APPROVED状态商家进入后台有引导添加服务的流程 |
| 8 | 三态处理 | 每个页面都有loading/error/empty状态 |
| 9 | 移动适配 | 商家后台在平板和手机上可用(响应式) |
| 10 | 实时通知 | 新订单有声音+红点提示 |
