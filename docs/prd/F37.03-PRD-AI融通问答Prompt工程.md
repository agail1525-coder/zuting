# F37.03-PRD-AI融通问答Prompt工程

> A4 智慧融通问答的核心工程 | 12祖师人设 + 并行调用 + 流式融合

## 1. 目标
让小鸿 (Qwen3.5-35B) 同时以 12 位祖师身份回答同一问题, 每个答案必须:
- 符合该祖师的语言风格
- 引用该文化的原典
- 控制在 150-300 字
- 避免跨文化混淆

## 2. 12 祖师人设 (System Prompt 模板)

1. 不超过 300 字

4. 不使用现代流行语

## 3. 并行调用策略

```typescript
async function wisdomQuery(question: string) {
  const traditions = getUserBlendTraditions(); // 主修+融修, 默认12全开
  const prompts = traditions.map(t => buildPrompt(t, question));

  // 并行流式调用, 最多 15s
  const streams = await Promise.allSettled(
    prompts.map(p =>
      withTimeout(xiaohongClient.stream(p, { maxTokens: 400 }), 15000)
    )
  );

  // 降级: 超时的文化使用缓存或跳过
  return streams.map((s, i) => ({
    tradition: traditions[i],
    status: s.status,
    answer: s.status === 'fulfilled' ? s.value : null,
  }));
}
```

## 4. 融合 Prompt

```
用户问题: ${question}
用户选择了以下 ${n} 位祖师的回答:
${chosenAnswers.map(a => `【${a.master}】${a.text}`).join('\n\n')}

请以求道者第一人称, 将上述智慧融合成一段属于"我"的答案.
要求:
1. 不超过 200 字
2. 保留每位祖师的核心洞见
3. 使用"我"而非引用
4. 结尾给出一个具体行动
```

## 5. 缓存策略
- Key: `wisdom:${hash(question)}:${traditionsHash}`
- TTL: 7 天
- 命中率目标: ≥ 40% (节省 LLM 成本)

## 6. 安全
- 输入: moderation 敏感词扫描
- 输出: 二次敏感词扫描 + 宗教和谐检查
- 记录: 所有 query 入 WisdomQuery 表审计
