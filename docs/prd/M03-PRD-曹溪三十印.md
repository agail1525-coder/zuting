# M03 曹溪三十印 产品需求文档

> 版本: v1.0 | 日期: 2026-03-25 | 状态: 已发布

---

## 1. 需求背景与目标

### 1.1 业务背景

曹溪愿命三十印（简称"三十印"）是祖庭旅行平台的灵性核心模块。三十印源自曹溪宗法脉，将修行历程系统化为 30 个阶次（印），分属五个系列（初印系、中印系、印果印、成道印、归源印），每印配有偈颂（poem）、精要（essence）、修法（practice）和愿文（vow）。

三十印不仅是平台的内容展示模块，更是未来修行打卡（Practice）功能的基础。用户可以逐印修行，记录日常功课，形成个人修行轨迹。这一模块承载着平台"帮助 100 万人走祖庭"愿景中"修行实践"的核心价值。

**五系概览：**

| 系列 | 印号 | 色系 | HEX | 寓意 |
|------|------|------|-----|------|
| 初印系 | 1-10 | 青色 (cyan) | #0EA5E9 | 修行发心，初入门径 |
| 中印系 | 11-20 | 靛色 (indigo) | #6366F1 | 深入修持，智慧开显 |
| 印果印 | 21-23 | 紫色 (purple) | #A855F7 | 修行结果，证量显现 |
| 成道印 | 24-26 | 红色 (red) | #EF4444 | 成就道业，转化圆满 |
| 归源印 | 27-30 | 琥珀 (amber) | #F59E0B | 回归本源，功德圆满 |

### 1.2 需求目标（量化）

| 指标 | 目标值 | 度量方式 |
|------|--------|----------|
| 30 印内容完整率 | 30/30 印全部上线，每印含偈颂/精要/修法/愿文 | 字段非空率 |
| 印详情页停留时长 | 平均 > 60s | 埋点分析 |
| 五系 Tab 使用率 | 每系 Tab 点击率 > 10% | 埋点分析 |
| 修行打卡日活（v2） | > 500 DAU | 打卡记录统计 |
| 连续打卡留存（v2） | 7 日留存 > 30% | 用户行为分析 |

### 1.3 需求范围

**v1 包含：**
- 30 印列表展示（五系 Tab 切换）
- 印详情页（偈颂/精要/修法/愿文完整展示）
- Web / App / 小程序 / Admin 四端
- 管理后台 CRUD
- 五系色彩主题

**v2 规划（本期不含）：**
- 修行打卡功能（Practice 模型）
- 打卡日历与修行进度
- 个人修行统计
- 打卡提醒推送
- 社区修行排行榜

## 2. 用户故事

### US-01 浏览三十印体系

**作为** 一名修行爱好者，
**我希望** 按五个系列浏览三十印的列表，每印有编号、名称和系列色标识，
**以便** 我能了解三十印的整体框架和修行脉络。

**验收标准：**
- [ ] 列表页显示 5 个系列 Tab：初印系/中印系/印果印/成道印/归源印
- [ ] 每个 Tab 显示对应系列的印列表
- [ ] 每张印卡片显示：编号、名称、系列色边框/背景
- [ ] 默认展示"全部"或"初印系"
- [ ] Tab 切换时有平滑过渡

### US-02 深入学习某印

**作为** 一名正在修行的用户，
**我希望** 在印详情页完整阅读偈颂、精要、修法和愿文，
**以便** 我能深入理解和实践该印的修行内涵。

**验收标准：**
- [ ] 详情页分四个区块展示：偈颂、精要、修法、愿文
- [ ] 偈颂区域使用大号居中排版（类似诗词格式）
- [ ] 精要使用重点标注样式
- [ ] 修法使用步骤化排版
- [ ] 愿文使用庄重排版（引用块风格）
- [ ] 页面主题色跟随该印所属系列色

### US-03 按系列探索修行路径

**作为** 一名系统修行者，
**我希望** 按系列顺序（初印→归源）逐步浏览各印，页面有上一印/下一印导航，
**以便** 我能按修行次第有序推进。

**验收标准：**
- [ ] 详情页底部显示"上一印"/"下一印"导航按钮
- [ ] 第 1 印无"上一印"，第 30 印无"下一印"
- [ ] 跨系列时（如第 10 印 → 第 11 印），系列色自动切换
- [ ] 进度指示器显示当前位置（如 "7/30"）

### US-04 修行打卡（v2）

**作为** 一名日常修行的用户，
**我希望** 在印详情页点击"今日打卡"，记录修行时长和心得笔记，
**以便** 我能持续跟踪自己的修行进度。

**验收标准（v2）：**
- [ ] 印详情页底部"今日打卡"按钮
- [ ] 打卡弹窗：选择修行时长（分钟）+ 笔记输入
- [ ] 打卡成功显示动画（印章盖印效果）
- [ ] 个人中心可查看打卡日历和统计
- [ ] 连续打卡有徽章奖励

### US-05 管理员维护三十印

**作为** 管理员，
**我希望** 在后台编辑每印的偈颂、精要、修法、愿文内容，
**以便** 我能维护和优化三十印的内容质量。

**验收标准：**
- [ ] 表格展示 30 印，含编号/名称/系列/操作
- [ ] 编辑表单包含偈颂/精要/修法/愿文四个 TextArea
- [ ] 系列字段使用 Select 下拉，选项为五系 enum
- [ ] 颜色字段根据系列自动填充默认色
- [ ] 支持新增自定义印（如用户自定义印，v2）

### 3.2 业务规则清单

| 规则编号 | 描述 | 触发条件 | 处理逻辑 | 异常处理 |
|----------|------|----------|----------|----------|
| R-01 | 印编号唯一 | 新增印 | id/编号字段 unique 校验 | 409 Conflict |
| R-02 | 系列枚举校验 | 新增/编辑印 | series 必须为 CHUYIN/ZHONGYIN/YINGUOYIN/CHENGDAOYIN/GUIYUANYIN 之一 | 400 Bad Request |
| R-03 | 系列色自动映射 | 新增印时选择系列 | 根据系列自动填充默认色 HEX | 可手动覆盖 |
| R-04 | 印序号范围 | — | 初印系 1-10, 中印系 11-20, 印果印 21-23, 成道印 24-26, 归源印 27-30 | 不合规警告 |
| R-05 | 偈颂必填 | 新增印 | poem 字段不能为空 | 400 Bad Request |
| R-06 | 每日打卡去重（v2） | 同一用户同一印同一天 | 同日仅允许一次打卡 | 提示"今日已打卡" |
| R-07 | 打卡时长范围（v2） | 提交打卡 | duration 1~480 分钟 | 超出范围提示 |
| R-08 | 列表排序 | GET /seals | 默认按编号升序排列 | — |
| R-09 | 系列筛选 | GET /seals?series=CHUYIN | 只返回该系列的印 | 无效系列返回空 |

## 5. 数据模型

### 5.1 实体关系

```
Seal (1) ──────< (N) Practice (v2)
   │
   └── series: SealSeries (enum)

User (1) ──────< (N) Practice (v2)
```

v1 阶段 Seal 为独立实体，无外键关联。v2 引入 Practice 后形成 User ↔ Seal 的多对多（通过 Practice 中间表）。

### 5.2 核心字段

**Seal 印表：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | String (cuid) | 是 | 主键 |
| name | String | 是 | 印名称，如"初印·发心" |
| nameEn | String | 否 | 英文名称 |
| series | SealSeries (enum) | 是 | 所属系列: CHUYIN/ZHONGYIN/YINGUOYIN/CHENGDAOYIN/GUIYUANYIN |
| order | Int | 是 | 排序编号 1-30 |
| poem | String | 是 | 偈颂（诗文） |
| poemEn | String | 否 | 英文偈颂 |
| essence | String | 否 | 精要（核心含义阐述） |
| essenceEn | String | 否 | 英文精要 |
| practice | String | 否 | 修法（实修方法指导） |
| practiceEn | String | 否 | 英文修法 |
| vow | String | 否 | 愿文（发愿文字） |
| vowEn | String | 否 | 英文愿文 |
| color | String | 是 | 系列色 HEX，如"#0EA5E9" |
| createdAt | DateTime | 是 | 创建时间 |
| updatedAt | DateTime | 是 | 更新时间 |

**SealSeries 枚举：**

| 值 | 中文 | 印号范围 | 色值 |
|----|------|----------|------|
| CHUYIN | 初印系 | 1-10 | #0EA5E9 |
| ZHONGYIN | 中印系 | 11-20 | #6366F1 |
| YINGUOYIN | 印果印 | 21-23 | #A855F7 |
| CHENGDAOYIN | 成道印 | 24-26 | #EF4444 |
| GUIYUANYIN | 归源印 | 27-30 | #F59E0B |

**Practice 修行打卡表（v2）：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | String (cuid) | 是 | 主键 |
| userId | String (FK) | 是 | 用户 ID |
| sealId | String (FK) | 是 | 印 ID |
| date | DateTime | 是 | 打卡日期（精确到日） |
| duration | Int | 是 | 修行时长（分钟） |
| note | String | 否 | 心得笔记 |
| createdAt | DateTime | 是 | 创建时间 |

## 6. API 接口

| 接口 | Method | 路径 | 入参 | 出参 | 说明 |
|------|--------|------|------|------|------|
| 印列表 | GET | /api/seals | ?series (enum) | Seal[] | 支持系列筛选，按 order 排序 |
| 印详情 | GET | /api/seals/:id | id: string | Seal | 完整印数据 |
| 新增印 | POST | /api/seals | SealCreateDto | Seal | 管理员权限 |
| 编辑印 | PATCH | /api/seals/:id | SealUpdateDto | Seal | 管理员权限 |
| 删除印 | DELETE | /api/seals/:id | id: string | { success: true } | 管理员权限 |
| 打卡 (v2) | POST | /api/practices | PracticeCreateDto | Practice | 需登录 |
| 打卡列表 (v2) | GET | /api/practices | ?userId, ?sealId, ?dateFrom, ?dateTo | Practice[] | 需登录 |
| 打卡统计 (v2) | GET | /api/practices/stats | ?userId | PracticeStats | 总时长/天数/连续 |

```json
  "name": "string (required)",

  "order": "number (required, 1-30)",
  "poem": "string (required)",
  "essence": "string",
  "practice": "string",
  "vow": "string",
  "color": "string (hex, required)"
}
```

**PracticeCreateDto（v2）：**
```json
{
  "sealId": "string (required)",
  "date": "string (ISO date, required)",
  "duration": "number (required, 1-480)",
  "note": "string"
}
```

- **加载中：** 30 个骨架屏卡片

- **加载失败：** 错误提示 + 重试
- **筛选无结果：** 不适用（数据固定）

```
│  │  正文: 阐述核心含义                       │     │

│  ┌─── 修法 (practice) ──────────────────────┐     │

│                                                    │
│  ┌─── 愿文 (vow) ───────────────────────────┐     │

```

**字段表：**

| 字段 | 来源 | 显示格式 |
|------|------|----------|
| 编号 | seal.order | "#N" 大号显示 |
| 名称 | seal.name / nameEn | 横幅标题 |
| 系列 | seal.series | 系列名标签 |
| 进度 | seal.order / 30 | 进度条 + 文字 "N/30" |
| 偈颂 | seal.poem | 大号居中，每行一句，系列色 |
| 精要 | seal.essence | 正文排版，浅色背景卡片 |
| 修法 | seal.practice | 步骤排版，深色背景 |
| 愿文 | seal.vow | 引用块样式，系列色左边框 |

**交互表：**

| 交互 | 触发 | 行为 |
|------|------|------|
| 上一印 | click | 跳转 /seals/[prevId]，第 1 印隐藏 |
| 下一印 | click | 跳转 /seals/[nextId]，第 30 印隐藏 |
| 键盘左箭头 | keydown ← | 同上一印 |
| 键盘右箭头 | keydown → | 同下一印 |
| 进度条 | — | 纯展示，不可交互 |

**状态：**
- **加载中：** 横幅骨架屏 + 内容区骨架屏
- **ID 无效：** 跳转 404
- **偈颂/精要/修法/愿文为空：** 对应区块显示"内容待补充"灰色提示

- Tab 横向滚动（ScrollView horizontal）
- 下拉刷新支持（虽然数据较固定）

#### 页面 A-02: 印详情 `/seals/[id]`

**与 Web 差异：**
- 单栏全宽布局
- 偈颂区域支持双指缩放字体大小
- 无键盘快捷键
- 支持左右滑动切换上下印（SwipeGesture，v2）
- 底部预留"今日打卡"按钮位置（v2 启用）

#### 页面 A-03: 打卡日历（v2）

**入口：** 修行 Tab 顶部"我的修行"按钮

**布局：** 日历组件（标记打卡日） + 统计卡片（总天数/总时长/连续天数） + 最近打卡列表

### 7.3 小程序端（Taro）

#### 页面 MP-01: 三十印列表 `/seals`

**布局：** 与 App 类似，横向 Tab + 竖向卡片列表

**与 App 差异：**
- Tab 使用 Taro ScrollView horizontal
- 列表使用 Taro virtual-list 优化长列表性能
- 无下拉刷新（数据固定）
- 卡片点击跳转 `/seal-detail?id=xxx`

#### 页面 MP-02: 印详情 `/seal-detail`

**布局：** 与 App 类似，ScrollView 纵向展示四区块

**与 App 差异：**
- 参数通过 query string 传递
- 支持微信分享印内容（onShareAppMessage）：标题为印名，描述为偈颂前 40 字
- 无手势切换，仅底部按钮切换上下印
- 不支持打卡功能（v2 仅 App 端）
