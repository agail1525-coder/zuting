## 1. 背景与目标

### 1.1 背景

提交评价页是 UGC 评价系统的写入入口，让用户将朝圣体验转化为可分享的内容。良好的评价写作体验能显著提升 UGC 生产率——对标 TripAdvisor 数据，引导式星级选择 + 内容提示可使评价完成率提升约 40%。Web 端以弹窗形式呈现，Mobile/小程序端以全屏独立页面呈现。

### 1.2 目标

- [ ] 实现星级选择（1-5星）、文字评价（必填）、图片上传（可选）的完整表单
- [ ] 支持提交、编辑（PENDING 状态）两种场景
- [ ] 行程完成后触发评价邀请，提供便捷评价入口

### 1.3 非目标（本期不做的）

- 不包含多维度评分（灵性体验、文化深度等子维度，延期到 P2）
- 不包含评价模板/快捷标签（延期到 P2）
- 不包含视频上传（延期到 Phase D）

## 2. 竞品对标分析

### 2.1 竞品功能对比

| 竞品 | 功能描述 | 优点 | 缺点 | 借鉴点 |
|------|---------|------|------|-------|
| TripAdvisor | 星级选择+评价标题+正文+照片上传，各星级有对应文字描述（很差/差/一般/好/极好） | 星级文字引导降低用户困惑，步骤清晰 | 表单字段较多，有填写疲劳 | 星级对应文字描述、照片上传内嵌表单 |
| Booking.com | 入住后邮件/App 触发，正负面评论分开填写，滑块式评分 | 强制触发时机好，完成率高 | 正负分开增加认知负担 | 行程完成后自动推送评价邀请 |
| 大众点评 | 移动端全屏，星级+文字+标签推荐+图片，字数进度条激励 | 字数进度条有效激励用户多写 | 标签推荐容易被滥用 | 字数进度条（输入字数/建议最低字数） |

### 2.2 差距分析

JOINUS.COM 当前完全缺失写评价功能，用户无法表达朝圣体验，平台内容生态无法启动，对比竞品存在根本性功能缺口。

### 2.3 差异化优势

在提交成功后附加"分享到朝圣日志"快捷入口，将评价内容同步至朝圣日志（JournalEntry），打通评价与日志两个内容模块，形成独特的朝圣记录生态。

## 4. 功能清单

### 4.1 核心功能（P0 — 必须有）

- [ ] 星级选择器：5颗星交互（点击/Touch 选择），每颗星对应文字描述
- [ ] 评价内容输入框：必填，最少 10 字，实时字数统计
- [ ] 图片上传区：最多9张，显示缩略图网格，支持删除，详见 F04-PRD
- [ ] 提交按钮：所有必填项满足时激活，提交中显示 loading 状态
- [ ] 提交成功反馈：Toast "评价已提交，等待审核" → 关闭弹窗/返回详情页
- [ ] 表单校验：评分未选提示"请选择评分"，内容过短提示"评价内容至少10个字"

### 4.2 增强功能（P1 — 应该有）

- [ ] 标题输入框：可选填，最多 50 字
- [ ] 字数激励进度条：内容字数/推荐字数（50字）的进度条，到达时显示"👍 评价很详细"
- [ ] 编辑模式：已有 PENDING 评价时，表单预填原内容，提交走 PATCH 接口
- [ ] 行程完成触发：行程状态变为 COMPLETED → 通知推送"评价此次行程的圣地"
- [ ] 离开确认：用户有未提交内容时，返回/关闭触发"确认放弃评价？"弹窗

### 4.3 优化功能（P2 — 可以有）

- [ ] 多维度评分：灵性体验/文化深度/环境设施 三个子维度
- [ ] 快捷标签：推荐标签（"风景震撼"/"灵气充沛"/"交通便利"等），点击追加到内容
- [ ] 分享到日志：提交成功后快捷入口同步至朝圣日志

### 5.3 星级交互细节

| 星级值 | 对应文字 | 文字颜色 |
|--------|---------|---------|
| 1 | 很差 | #EF4444（红色） |
| 2 | 较差 | #F97316（橙色） |
| 3 | 一般 | #EAB308（黄色） |
| 4 | 满意 | #22C55E（绿色） |
| 5 | 极好 | #D4A855（金色，品牌色） |

### 5.4 关键交互流程

```
1. 用户点击"写评价"按钮
   → 检查登录状态
   → 未登录: 跳转登录页（保留回跳路径）
   → 已登录: 检查是否已有评价
      → 已有 APPROVED 评价: Toast "您已评价过此地" + 展示已有评价
      → 已有 PENDING 评价: 进入编辑模式（预填内容）
      → 无评价: 进入新建模式

2. 用户填写表单
   → 实时校验星级（未选则提交按钮禁用）
   → 实时统计字数

3. 用户点击"提交评价"
   → 前端校验（评分、字数）
   → 调用 POST /api/reviews（或 PATCH）
   → 成功: Toast "评价已提交，等待审核" + 关闭弹窗/返回
   → 失败: Toast 显示错误信息（重复评价/服务器错误等）

4. 用户点击"取消"或关闭
   → 若表单有内容，弹出"确认放弃评价？"二次确认
   → 确认放弃: 关闭，内容不保存
   → 取消: 返回表单继续编辑
```

### 5.5 三态设计

| 状态 | 表现 |
|------|------|
| Loading（提交中） | 提交按钮变为 Spinner + "提交中..."，禁止重复点击 |
| Empty（已有已通过评价） | 不显示写评价入口，或显示"您已评价，查看我的评价"链接 |
| Error（提交失败） | 按钮恢复可用，Toast 显示具体错误（如"网络错误，请重试"） |

## 6. API 契约

### 6.1 使用的 API 端点

| 端点 | 方法 | 用途 |
|------|------|------|
| `/api/reviews` | POST | 新建评价（需 JWT） |
| `/api/reviews/:id` | PATCH | 编辑评价（需 JWT，限 PENDING 状态） |
| `/api/uploads/review-photos` | POST | 上传评价图片（见 F04-PRD） |

### 6.2 新建评价请求体

```json
{
  "entityType": "HOLY_SITE",
  "entityId": "clxxx",
  "rating": 5,
  "title": "震撼心灵的朝圣之旅",
  "content": "这次朝圣之旅让我深刻感受到禅宗文化的博大精深...",
  "photos": [
    "https://cdn.joinus.com/review-photos/uuid1.jpg",
    "https://cdn.joinus.com/review-photos/uuid2.jpg"
  ]
}
```

### 6.3 编辑评价请求体

```json
{
  "rating": 4,
  "title": "更新后的标题",
  "content": "修正后的评价内容...",
  "photos": ["https://cdn.joinus.com/review-photos/uuid1.jpg"]
}
```

### 6.4 错误码处理

| HTTP 状态码 | 错误码 | 前端处理 |
|------------|-------|---------|
| 400 | INVALID_RATING | 显示"请选择1-5星评分" |
| 400 | CONTENT_TOO_SHORT | 显示"评价内容至少10个字" |
| 401 | UNAUTHORIZED | 跳转登录 |
| 403 | REVIEW_APPROVED | 显示"已审核通过的评价不可修改" |
| 409 | DUPLICATE_REVIEW | 显示"您已评价过此地" |
| 500 | INTERNAL_ERROR | 显示"提交失败，请稍后重试" |

## 7. 数据模型

本页面不新增数据模型，使用 M15 中定义的 Review 模型。

```typescript
// 表单状态类型
interface ReviewFormData {
  rating: number;      // 1-5，0 表示未选
  title: string;
  content: string;
  photos: string[];    // 已上传图片的 URL 列表
}

// 提交评价入口的上下文
interface WriteReviewContext {
  entityType: 'HOLY_SITE' | 'TEMPLE' | 'TRIP';
  entityId: string;
  entityName: string;
  existingReview?: {  // 已有 PENDING 评价时预填
    id: string;
    rating: number;
    title?: string;
    content: string;
    photos: string[];
  };
}
```

## 8. 技术约束

- **性能要求**: 弹窗/页面打开 < 300ms；图片上传后预览 < 1s
- **移动端**: 375px 下键盘弹出时表单可上滚，星级选择触控区域 ≥ 44px
- **兼容性**: Web 支持 Chrome 100+, Safari 15+；小程序支持微信基础库 3.0+
- **类型安全**: 禁止 any，ReviewFormData 从 shared-types 导入
- **认证**: 提交接口需 JWT，进入表单前检查登录状态
- **重复提交防护**: 提交中按钮禁用，防止用户重复点击
- **状态持久化**: 若用户意外退出（非主动取消），表单内容不自动保存（保持简单）
