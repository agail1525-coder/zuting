# 全球祖庭旅行平台 — 产品概述与信息架构

> 版本: v2.0 | 日期: 2026-03-25 | 状态: 实施中

---

## 1. 产品愿景

**帮助100万人走祖庭，建立全球宗教文化和平使者网络。**

祖庭旅行平台是全球首个跨宗教文化朝圣旅行一站式服务平台，覆盖12大信仰体系、60处圣地、27座祖庭，提供从文化探索、行程规划、在线支付、朝圣导航到修行记录的完整闭环。

## 2. 产品定位

| 维度 | 定位 |
|------|------|
| 核心用户 | 宗教文化爱好者、朝圣者、修行者、文化旅行者 |
| 核心价值 | 跨宗教文化认知 + 朝圣行程一站式服务 + 修行成长体系 |
| 商业模式 | 朝圣行程服务费(B2C) + 导游平台佣金(C2C) + 文化IP衍生(长期) |
| 竞品差异 | 不是单一宗教App，而是**跨信仰对话**平台；不是纯旅游OTA，而是**文化深度体验**平台 |

## 3. 用户画像

### 3.1 核心用户角色

| 角色 | 画像 | 核心需求 | 使用端 |
|------|------|----------|--------|
| **朝圣者 (PILGRIM)** | 25-55岁，有宗教信仰或文化兴趣，年旅行1-3次 | 了解圣地文化→规划行程→完成朝圣→记录感悟 | Web + App + 小程序 |
| **导游 (GUIDE)** | 30-50岁，宗教文化学者/当地向导，精通某信仰体系 | 接单带团→展示专业资质→获取收入→积累口碑 | App |
| **文化大使 (AMBASSADOR)** | 知名宗教人士/文化KOL，影响力传播者 | 分享文化见解→组织主题活动→建立社区影响力 | Web + App |
| **管理员 (ADMIN)** | 运营团队，负责内容审核/订单管理/数据分析 | 内容CRUD→订单处理→数据看板→用户管理 | Admin后台 |

## 4. 产品架构

### 4.1 多端矩阵

| 平台 | 技术栈 | 核心场景 | 页面数 |
|------|--------|----------|--------|
| **Web官网** | Next.js 15 + React 19 + TailwindCSS v4 | SEO引流 + 深度内容浏览 + PC端行程管理 | 22页 |
| **Mobile App** | React Native + Expo 52 | 朝圣中导航 + 离线日志 + 推送通知 | 13屏 |

## 附录

### 设计规范引用
- 品牌色: 金色 #D4A855 | 深殿 #0f172a → #020617
- 每个宗教有独特色系和符号
- 五系印各有系列色 (青/蓝/紫/红/金)
- 暗色主题，金色点缀，庄严而现代

### 技术约束
- NestJS使用SWC编译器(非tsx)
- Prisma enum不能直接传Swagger装饰器
- 小程序需微信审核合规
- 多端共享 @zuting/shared-types + @zuting/config
