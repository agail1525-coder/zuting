## 五、状态设计

```
Loading:
  → 骨架屏（3个卡片骨架，保持布局稳定）

有数据:
  → 正常卡片列表 + 分页/无限滚动

空状态（有筛选条件）:
  ┌─────────────────────────────────────┐
  │         [空行李箱插画]              │
  │   当前筛选条件下没有符合的套餐      │
  │        [清除筛选条件]               │
  └─────────────────────────────────────┘

空状态（无套餐）:
  → "套餐即将上线，敬请期待"

Error状态:
  → 错误提示 + 重新加载按钮
```

## 六、API调用

| 操作 | API端点 | 触发时机 |
|------|---------|---------|
| 获取套餐列表 | GET /api/packages | 页面加载、筛选变更、排序变更 |
| 获取下一页 | GET /api/packages?page=N | 滚动到底部 |
| 收藏/取消收藏 | POST /api/wishlist/toggle | 点击心形图标（P2） |

**筛选参数映射:**
```
Tab[全部]            → 不带packageType参数
Tab[经典朝圣]        → packageType=CLASSIC_PILGRIMAGE
Tab[深度体验]        → packageType=DEEP_EXPERIENCE
Tab[尊享VIP]         → packageType=VIP_ALL_INCLUSIVE
Tab[自由行]          → packageType=FREE_STYLE
Tab[团队朝圣]        → packageType=GROUP_PILGRIMAGE

时长[1-3天]         → minDuration=1&maxDuration=3
时长[4-7天]         → minDuration=4&maxDuration=7
时长[8天+]          → minDuration=8

排序[热门]          → sort=POPULAR
排序[价格↑]         → sort=PRICE_ASC
排序[价格↓]         → sort=PRICE_DESC
排序[时长]          → sort=DURATION_ASC
```

## 七、验收标准

- [ ] 页面首屏加载时间 < 2s，骨架屏立即显示
- [ ] 5个类型Tab切换均正确过滤数据
- [ ] 价格筛选输入后去抖300ms后触发查询
- [ ] 时长复选框多选生效，清除筛选一键重置
- [ ] 排序切换后列表重排，当前排序有选中态
- [ ] 套餐卡片显示：封面图/名称/关联圣地/时长/includes图标/价格
- [ ] Lv3+登录用户的会员价正确显示
- [ ] 未登录用户不显示会员价，卡片底部有"登录享会员价"引导
- [ ] 无结果时显示空状态，有清除筛选操作
- [ ] 无限滚动触底自动加载，不出现重复数据
- [ ] 点击"立即预订"跳转对应套餐详情页

## 八、技术约束

```
[TC-P18-01] Web端使用 useInfiniteQuery 实现无限滚动，Intersection Observer触发
[TC-P18-02] Mobile端使用 FlatList + onEndReached 实现无限滚动
[TC-P18-03] 筛选状态存 URL queryString（Web）或 AsyncStorage（Mobile），支持分享/刷新保持
[TC-P18-04] 价格显示: basePrice / 100，保留0位小数，千分位逗号分隔
[TC-P18-05] 封面图使用 Next.js Image 组件优化（Web），懒加载
[TC-P18-06] 会员价判断: 当前用户level >= 3 且套餐有memberPrice时展示
```
