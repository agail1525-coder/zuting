# P29-PRD-圣地导览页 (Holy Site Tour Page)

> Sprint E2 | 版本: v1.0 | 日期: 2026-03-29
> 依赖: M31-PRD-多媒体导览

---

## 1. 概述

在圣地详情页新增"导览"Tab，提供视频播放、360度全景查看、音频讲解三种沉浸式体验。用户可以在出发前通过多媒体内容深度了解圣地的文化环境。

### 1.1 目标用户
- 准备朝圣的旅行者: 出发前预览目的地
- 在线浏览的文化爱好者: 远程体验圣地氛围
- 行动不便的信众: 通过多媒体实现"云朝圣"

### 1.2 页面入口
- 圣地详情页 → "导览" Tab
- 祖庭详情页 → "导览" Tab (复用同一组件)

## 4. 交互设计

### 4.1 视频播放器
- HTML5 Video原生播放
- 自定义控件: 播放/暂停、进度条、音量、全屏、倍速
- 缩略图列表点击切换视频源
- 移动端: 横屏自动全屏提示
- Loading态: 骨架屏+加载动画
- Error态: 播放失败提示+重试按钮

### 4.2 360度全景查看器
- 方案A (推荐): Pannellum.js — 轻量级开源全景库
- 方案B (备选): CSS 3D Transform + 触摸事件
- 交互: 鼠标拖拽/手指滑动旋转视角，滚轮/捏合缩放
- 按钮: 全屏、自动旋转、重置视角、陀螺仪(移动端)
- 支持热点标注(后续Phase)

### 4.3 音频播放条
- 每条音频独立播放条
- 控件: 播放/暂停按钮、进度条(可拖拽)、当前时间/总时长
- 同时只能播放一条音频(切换时暂停上一条)
- 后台播放支持(移动端)
- 下载按钮(离线收听，后续Phase)

## 5. API调用

```typescript
// 获取圣地的所有导览媒体
GET /api/media?entityType=HOLY_SITE&entityId={siteId}

// 响应按mediaType分组展示:
// - videos: mediaType === 'VIDEO'
// - panoramas: mediaType === 'PANORAMA'
// - audios: mediaType === 'AUDIO'
```

### 5.1 三态处理
- **Loading**: 骨架屏，视频区域显示灰色占位
- **Empty**: "暂无导览内容，敬请期待" + 插画
- **Error**: "加载失败，请稍后重试" + 重试按钮

## 6. 响应式设计

| 断点 | 布局 |
|------|------|
| Desktop (>1024px) | 视频16:9大图，全景600px高，音频列表 |
| Tablet (768-1024px) | 视频16:9，全景400px高 |
| Mobile (<768px) | 视频全宽16:9，全景300px高，音频紧凑布局 |

### 6.1 Mobile APP (React Native)
- 视频: `expo-av` Video组件
- 全景: `react-native-panorama-view` 或 WebView + Pannellum
- 音频: `expo-av` Audio组件

### 6.2 微信小程序 (Taro)
- 视频: `<Video>` 原生组件
- 全景: WebView嵌入全景页
- 音频: `Taro.createInnerAudioContext()`
