## 1. 背景与动机

CEO 在 /about 页截图上圈出 Vision 与 Core Values 之间的空位,指示:

> "这里加入非宗教组织仅旅行平台声明,你全面搜索相关政策然后再声明。"

### 1.1 为什么必须有这个声明

| 风险点 | 若无声明的后果 |
|-------|---------------|
| 内容涉及 12 大文化传统 (含佛/道/基督/伊斯兰等宗教文化) | 监管可能认定为「互联网宗教信息服务」,需 `《互联网宗教信息服务许可证》` |
| 平台介绍寺院/宫观/教堂/神社等场所 | 易被误解为代理组织宗教活动 |
| 售卖「岭南道禅路线」等含文化传统关键词产品 | 工商/网信/宗教局多头审查风险 |
| AI 助手"小鸿"回答文化传统问题 | UGC + AIGC 双重内容审核压力 |
| 用户 UGC 攻略可能含祈福/受戒等描述 | 需明确平台边界,区分用户行为与平台立场 |

### 1.2 前置合规基础 (已在路线层面落地)

- **岭南线合规徽章 Set**: `非宗教 · 非寺观 · 纯文化游学` 白名单 (见 `project_lingnan_dao_chan_route.md`)
- **Route 文案合规铁律**: 禁功利/修行/人设/宗教词,合规声明仅前端徽章呈现 (见 `feedback_route_content_compliance.md`)
- **12 文化游学路线模板化**: 全部纯文化、徽章 Set 统一 (见 `project_cultural_routes_12_deployed.md`)

> **缺口**: 路线层有徽章,但**平台层面的整体声明**未系统化,本次 F-ZZ08 填补此空白。

## 2. 法规检索过程

### 2.1 检索渠道

- ✅ `WebSearch` 查询: `互联网宗教信息服务管理办法 2026 最新 文化旅行平台 合规 非宗教组织声明`
- ✅ 官方来源校验: gov.cn · moj.gov.cn · sara.gov.cn (国家宗教事务局)
- ✅ 权威解读: 澎湃新闻 · 北大法宝 · freebuf

### 2.2 命中的核心法规 (共 7 部)

| # | 法规名称 | 发布/施行时间 | 对本平台影响 |
|---|---------|--------------|-------------|
| 1 | 《中华人民共和国宗教事务条例》(国务院令第 686 号) | 2017-09 修订 · 2018-02-01 施行 | **第 41 条**: 非宗教团体/院校/活动场所不得组织、举行宗教活动;**第 44 条**: 禁止以宗教名义进行商业宣传;**第 54 条**: 非宗教机构不得组织公民参加境内外宗教培训 |
| 2 | 《互联网宗教信息服务管理办法》(国家宗教事务局等五部门) | 2021-12-20 公布 · 2022-03-01 施行 | 未取得《互联网宗教信息服务许可证》的平台:不得传教、不得讲经讲道、不得直播/录播宗教仪式 (礼拜/烧香/受戒/诵经/祈祷/弥撒/洗礼 等) |
| 3 | 《宗教教职人员网络行为规范》 | 2025-09-16 发布 (国家宗教事务局) | 平台不招募/展示宗教教职人员身份代言 |
| 4 | 《中华人民共和国旅游法》 | 2013-10-01 施行 | 旅行社业务许可、出境游资质要求 |
| 5 | 《中华人民共和国网络安全法》 | 2017-06-01 施行 | ICP/备案/实名/日志留存 |
| 6 | 《中华人民共和国个人信息保护法》 | 2021-11-01 施行 | 用户个人信息处理合规 |
| 7 | 《互联网信息服务管理办法》 | 2000 / 持续修订 | ICP 备案要求 |

### 2.3 关键监管线 (must-have 边界)

**禁止项 (平台不做):**

- ❌ 传教 · 讲经 · 讲道
- ❌ 直播/录播宗教仪式 (礼拜/烧香/受戒/诵经/祈祷/弥撒/洗礼)
- ❌ 接受/转达/代收宗教捐献与功德款
- ❌ 组织公民参加境内外宗教培训
- ❌ 以宗教名义进行商业宣传
- ❌ 招募/安排宗教教职人员代言 (2025-09 新规)

**允许项 (平台可做):**

- ✅ 提供世俗文化旅行 · 文化游学 · 文化交流产品
- ✅ 介绍文化场所的建筑/历史/艺术/民俗背景 (客观信息)
- ✅ 规划行程 · 预订住宿/交通 · 撮合合规供应商
- ✅ 展示公开学术资料与文化遗产数字化内容
- ✅ 协助用户遵守目的地法律/场所规定/风俗

## 3. 设计决策

### 3.1 位置决策

**候选位置评估:**

| 选项 | 优点 | 缺点 | 裁决 |
|-----|------|------|------|
| 页脚 Footer 小字 | 不打扰主体 | 权重太低,监管友好度不足 | ❌ |
| Hero 正下方 | 权重最高 | 破坏首屏品牌感 | ❌ |
| Vision 之前 | 优先级明确 | 先声明再愿景,顺序违反叙事 | ❌ |
| **Vision 之后 · Core Values 之前** | 叙事顺: 我们是什么 → **法律定位** → 我们做什么 | 需与主体视觉区隔 | ✅ |
| ICP 备案区旁 | 合规扎堆 | 传统做法,但埋得太深 | ❌ |

> **选定**: Vision (平台愿景) → **Compliance (平台性质声明)** → Core Values (核心理念)
> **理由**: 叙事顺先讲"为什么存在",再声明"法律边界",最后展开"价值主张",既诚实又不失品牌节奏。

### 3.2 视觉风格

**对齐已有合规徽章语言:**

- **色系**: 暖金 #F5D898 + 琥珀 `amber-200` 边框,与岭南线路线合规徽章 `非宗教 · 非寺观` 同色系
- **结构**: `bg-gradient-to-br from-amber-50 via-white to-amber-50/40` + `border-2 border-amber-200`
- **对比度**: 刻意不走 Trust Badges 的绿色 (避免"夸赞式"观感), 暖金给"郑重告知"感
- **层次**: Header (标签+标题) + Body (主句 + 两列白/黑名单 + 法规依据 + 参访提示 + 投诉邮箱)

### 3.3 内容结构 (五段式)

```
1. 标签带:  ⚖️  Compliance · 合规声明
2. 主标题:  平台性质声明 · 非宗教组织,仅文化旅行平台
3. 核心句:  Joinus 是依法 ICP 备案的文化旅行互联网平台,
           不属于/不附属/不代表任何宗教组织,亦不持《互联网宗教信息服务许可证》
4. 双列对照:
   ✕ 平台不做 (6 条) ←→ ✓ 平台所做 (5 条)
5. 支撑区:
   - 法规依据 (7 部法律法规列举)
   - 用户参访提示 (3 条)
   - 合规投诉邮箱 compliance@joinus.com + 更新日期
```

### 3.4 文案精细化 (逐条来源)

| 文案条目 | 法规来源 |
|---------|---------|
| "不传教、不讲经、不开展宗教教育与培训" | 《互联网宗教信息服务管理办法》第 15 条 |
| "不直播、不录播宗教仪式 (礼拜/烧香/受戒/诵经/祈祷/弥撒/洗礼)" | 《互联网宗教信息服务管理办法》第 17 条 (原文列举) |
| "不接受、不转达、不代收宗教捐献" | 《互联网宗教信息服务管理办法》第 17 条 |
| "不组织公民参加境内外宗教培训" | 《宗教事务条例》第 54 条 |
| "不以宗教名义进行商业宣传" | 《宗教事务条例》第 44 条 |
| "不招募/安排宗教教职人员讲经或代言" | 《宗教教职人员网络行为规范》(2025-09) |

## 5. 踩坑与教训

### 5.1 Git push 受 SSL 握手失败 (本次环境)

**现象**: `fatal: unable to access 'https://github.com/...': schannel: failed to receive handshake, SSL/TLS connection failed`

**根因**: GitHub 网络层抖动 (GFW/本地网络波动),`curl -sI https://github.com` 无返回,而 baidu 200 OK,非代码问题。

**处理**:

- 尝试 `http.sslBackend=openssl` 降级 → 报 `TLS unexpected eof`,同样网络层问题
- 本地 commit 已就绪 (`2ac56c6`),等待网络恢复后 `git push origin master` 即可
- 生产已通过 SSH 直连部署完成,声明对外已生效

**教训 (新铁律候选)**:

```
[COM-01] 合规声明上线必须先"生产部署 + 本地 commit",
         GitHub 同步因网络层抖动允许延后,不阻断对外合规呈现
```

### 5.2 文案必须字面精确

合规声明不得用近义词改写。比如:

- ✅ "不直播或者录播宗教仪式" — 与《管理办法》第 17 条原文对齐
- ❌ "不进行宗教活动直播" — 漏了"录播",可能被指覆盖不全

**教训**: 涉及禁止项必须逐字对应法条,不做文学性改写。

### 5.3 位置必须在"讲完我们是什么"之后

**初版直觉**: 放在页脚或 ICP 备案旁边 (传统"法律条款"位置)。

**CEO 截图纠偏**: 明确圈出 Vision 与 Core Values 之间。

**顿悟**: 合规声明不是免责条款,是平台自我定位的一部分。它的位置应该紧随"愿景",在"价值观"之前,让用户"先知我们合法、再信我们有价值"。

### 5.4 视觉不走"安心绿"走"郑重金"

Trust Badges (加密/审核/多语言/AI) 用绿色, 给人"稳妥"感;合规声明若也用绿色,会被视作"卖点",反而失去严肃性。

**决策**: 暖金 + 琥珀边框,给"正式告知"的分量感,同时与路线合规徽章语言打通。
