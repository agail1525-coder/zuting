## 1. 页面概述

### 1.1 页面定位

社区首页（`/community`）是 JOINUS.COM 的 UGC 内容发现中心，是用户探索他人游记攻略、参与问答互动、浏览圣地照片的核心入口。

该页面承载三大职责：
1. **内容发现** — 瀑布流展示社区游记攻略，支持多维度排序与标签筛选
2. **知识问答** — 呈现热门问答，降低新朝圣者决策门槛
3. **视觉灵感** — 精选照片轮播，激发用户朝圣欲望

### 1.2 社区首页在全站信息架构中的位置

```
全站导航:
  首页 | 圣地 | 祖庭 | 地图 | 行程 | ★ 社区 ← 本页 | 个人中心
                                           │
                        ┌──────────────────┼──────────────────┐
                        │                  │                  │
                    游记列表           问答广场           精选照片
                   (本页核心)         (本页侧边栏)       (本页轮播)
```

### 1.3 页面 URL

```
Web:        /community
Mobile:     /(tabs)/community
小程序:     /pages/community/index
```

## 2. 竞品对标分析

### 2.1 TripAdvisor — 旅行者论坛主页

**调研要点:**
- **分区导航**: 顶部有"论坛/游记/照片/问答"分区 Tab，内容层级清晰
- **热度排序**: 每个帖子显示"X 人回复 · Y 浏览"等热度数据
- **地区论坛**: 左侧列出"亚洲/欧洲/美洲"等地区分类，方便定向浏览
- **标题优先**: 论坛卡片以文字标题为主，无图片预览（适合深度文字讨论）
- **JOINUS借鉴点**: 热度数据显示 + 宗教/地区双维度分类导航

### 2.2 Trip.com 攻略社区

**调研要点:**
- **大图瀑布流**: 双列瀑布流，每张卡片有封面大图 + 标题 + 作者头像 + 点赞数
- **顶部 Tab**: 推荐/最新/热门/关注 四个 Tab，点击切换 Feed 内容
- **标签筛选芯片**: Tab 下方有横向滚动的目的地/主题标签芯片
- **浮动写作按钮**: 右下角橙色 FAB，"写游记"，用户随时可创作
- **JOINUS借鉴点**: 双列大图瀑布流 + Tab + 标签芯片 + FAB 设计

### 2.3 马蜂窝 — 游记 Feed

**调研要点:**
- **单列/双列切换**: 右上角切换布局，满足不同阅读偏好
- **卡片信息密度**: 封面图 + 标题（2行）+ 作者头像+名称 + 点赞数 + 目的地标签
- **"热门"角标**: 热门内容右上角显示红色"热"字角标
- **下拉刷新**: 下拉触发内容刷新，顶部显示"X 条新内容"提示条
- **无限滚动**: 页面底部自动加载更多，无翻页按钮
- **JOINUS借鉴点**: 热门角标 + 下拉刷新提示条 + 无限滚动

### 2.4 小红书 — 发现页

**调研要点:**
- **纯瀑布流**: 双列不等高瀑布流（Masonry Layout），视觉张力强
- **视频/图文混排**: 视频内容自动播放封面预览
- **快速互动**: 长按卡片弹出快速收藏/分享菜单
- **JOINUS借鉴点**: 不等高瀑布流排版 + 快速操作菜单

### 2.5 JOINUS 差异化护城河

| 竞品共有 | JOINUS 独有 |
|----------|-------------|
| 游记瀑布流 | 宗教信仰标签系统（12大信仰）|
| 作者信息 + 点赞 | 朝圣者认证徽章（已完成行程）|
| 标签筛选 | 圣地/祖庭实体关联（点击直跳详情页）|
| 问答社区 | 跨信仰和平对话 AI 审核 |
| 精选照片 | 按宗教系列展示圣地灵性景观 |

## 3. 用户故事

```
作为游客（未登录）:
  我浏览社区首页，看到推荐的游记攻略列表
  我可以按信仰、地区标签过滤内容
  我点击游记卡片可以查看游记详情
  我看到"写游记"按钮时，点击后被引导去登录/注册

作为注册用户（已登录）:
  我打开社区首页，默认看到"推荐"Tab 的个性化内容
  我切换"最新"Tab，看到最近发布的游记按时间排序
  我切换"关注"Tab，看到我关注的作者发布的内容
  我点击右下角 FAB "写游记"，进入游记编辑器页面
  我点击某个宗教标签芯片，Feed 自动过滤为该信仰相关内容
  我在侧边栏看到热门问答，点击感兴趣的问题查看回答
  我在精选照片轮播中受到灵感启发，点击图片看到对应圣地详情

作为内容创作者:
  我发布游记后，能在"最新"Tab 立即看到自己的内容
  我可以看到自己游记的浏览数、点赞数、评论数
  我的朝圣者认证徽章在作者头像旁显示，增加内容可信度

作为平台管理员:
  社区内容经过 AI 预审后，不当内容不显示在 Feed 中
  我可以在管理后台精选/置顶特定游记
```

## 6. 交互设计

### 6.1 Tab 切换行为

```
切换 Tab:
  1. Tab 下划线动画滑动到目标位置（200ms ease）
  2. 内容区域淡出淡入切换（100ms）
  3. 若目标 Tab 数据已缓存（5分钟内）→ 直接渲染缓存数据
  4. 若缓存过期 → 显示骨架屏 → 请求 API → 渲染

关注 Tab 特殊逻辑:
  - 未登录 → 点击"关注" Tab 时弹出登录引导弹窗
  - 已登录但无关注 → 显示空状态引导（"去发现有趣的朝圣者"）
```

### 6.2 标签筛选芯片行为

```
默认状态: "全部" 芯片激活（蓝色填充）
点击信仰芯片:
  1. 芯片变为激活状态（金色填充）
  2. "全部" 芯片取消激活
  3. Feed 内容过滤为该信仰相关游记
  4. URL 更新: /community?religion=buddhism（支持分享/刷新保持状态）

点击地区芯片（下拉选择）:
  1. 弹出地区选择面板（亚洲/欧洲/中东/美洲/非洲）
  2. 选定后与信仰芯片叠加过滤

清除筛选: 点击已激活芯片 → 取消筛选，恢复"全部"
```

### 6.3 无限滚动

```
触发条件: 用户滚动到距页面底部 200px 时
加载状态: 底部显示3个半透明骨架卡片
请求参数: cursor（游标分页，避免分页插入导致内容重复）
错误处理: 加载失败显示"加载失败，点击重试"按钮
终止条件: 后端返回 hasMore: false 时显示"— 已显示全部内容 —"
```

### 6.4 卡片点赞交互

```
点击心形图标:
  已登录:
    1. 立即乐观更新（点赞数+1，图标变红色实心♥）
    2. 后台发送 POST /api/posts/:id/like
    3. 若请求失败 → 回滚到原状态 + Toast "操作失败，请重试"
  未登录:
    → 弹出登录引导弹窗（"登录后才能点赞"）

取消点赞:
  1. 立即乐观更新（点赞数-1，图标变空心♡）
  2. 后台发送 DELETE /api/posts/:id/like
```

### 6.5 FAB 按钮行为

```
位置: 页面右下角固定，距底部 88px（Mobile 需避开底部 Tab Bar）
样式: 金色圆形按钮，直径 56px，内含 ✏ 图标 + "写游记"文字（展开态）
展开/收起: 用户向上滚动时展开显示文字，向下滚动时收起只显示图标
点击:
  已登录 → 跳转到 /community/editor（游记编辑器）
  未登录 → 弹出登录引导弹窗
```

### 7.5 数据模型（新增字段）

```prisma
// 在现有 Post 模型上扩展
model Post {
  id              String    @id @default(cuid())
  title           String
  coverImage      String?
  content         String    @db.Text
  excerpt         String?   @db.VarChar(200)
  status          PostStatus @default(DRAFT)
  authorId        String
  author          User      @relation(fields: [authorId], references: [id])
  likeCount       Int       @default(0)
  commentCount    Int       @default(0)
  viewCount       Int       @default(0)
  readMinutes     Int?
  isHot           Boolean   @default(false)
  publishedAt     DateTime?
  createdAt       DateTime  @default(now())
  updatedAt       DateTime  @updatedAt
  tags            PostTag[]
  likes           PostLike[]
}

enum PostStatus {
  DRAFT
  PUBLISHED
  HIDDEN
  DELETED
}

model PostTag {
  id         String  @id @default(cuid())
  postId     String
  post       Post    @relation(fields: [postId], references: [id])
  type       String  // 'religion' | 'entity' | 'region' | 'custom'
  slug       String
  label      String
  entityType String? // 'holy-site' | 'temple'
  entityId   String?
}

model PostLike {
  userId    String
  postId    String
  createdAt DateTime @default(now())
  @@id([userId, postId])
}
```

## 8. 性能要求

| 指标 | 要求 | 说明 |
|------|------|------|
| 首屏 LCP | ≤ 2.5s | 首屏 Feed 20 条卡片渲染完成 |
| 骨架屏出现 | ≤ 100ms | Tab 切换或首次加载立即显示 |
| Feed 数据加载 | ≤ 500ms | Redis 缓存命中（推荐 Tab）|
| 图片加载 | 懒加载 + WebP | 仅加载视口内图片，格式用 WebP |
| 瀑布流渲染 | 虚拟化（100条+）| 超过 100 条后使用虚拟滚动 |
| Tab 切换响应 | ≤ 200ms（缓存命中）| 缓存 TTL：推荐5分钟，最新2分钟 |
| 无限滚动加载 | ≤ 300ms | 触发到内容呈现 |

## 9. 异常态处理

| 场景 | 处理方式 |
|------|---------|
| 网络断开 | Toast "网络连接失败" + 保留上次加载的内容 |
| Feed 为空（推荐/最新）| 插画 + "社区还很新，成为第一个发布游记的人" + CTA按钮 |
| Feed 为空（关注）| 插画 + "关注有趣的朝圣者，获取他们的最新动态" + [去发现] 按钮 |
| 图片加载失败 | 显示宗教主题占位图（按游记标签对应信仰） |
| 点赞失败 | 乐观更新回滚 + Toast 提示 |
| 加载更多失败 | 底部"加载失败，点击重试"按钮（不影响已加载内容） |

## 10. 验收标准

### 功能验收

- [ ] 三个 Tab（推荐/最新/关注）可正常切换，内容不同
- [ ] 关注 Tab 未登录时弹出登录引导弹窗
- [ ] 关注 Tab 无关注用户时显示正确空状态
- [ ] 标签芯片点击后 Feed 内容被正确过滤
- [ ] 多个标签芯片叠加筛选正确工作
- [ ] 无限滚动触发加载更多，游标分页无重复内容
- [ ] 加载到最后一页时显示"已显示全部内容"
- [ ] 点赞操作乐观更新，成功后数字正确，失败后回滚
- [ ] FAB 按钮已登录跳转编辑器，未登录弹登录引导

- [ ] 首屏 LCP ≤ 2.5s（Lighthouse 测量）
