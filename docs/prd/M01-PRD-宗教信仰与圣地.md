# M01 宗教信仰与圣地 产品需求文档

> 版本: v1.0 | 日期: 2026-03-25 | 状态: 已发布

---

## 1. 需求背景与目标

### 1.1 业务背景

祖庭旅行平台的核心内容基础是全球 12 大信仰体系及其 60 处圣地。宗教信仰模块作为平台的内容根基，承载着用户浏览、探索、筛选宗教文化信息的第一入口。圣地模块则将抽象的信仰落地为具体的地理坐标，为后续的行程规划、朝圣日志等功能提供空间锚点。

平台覆盖的 12 大信仰包括：佛教、道教、基督教、伊斯兰教、印度教、犹太教、儒教、锡克教、神道教、藏传佛教、原住民灵性、巴哈伊教。每个信仰关联 5 处代表性圣地，共计 60 处，分布于全球各大洲，均配有精确 GPS 坐标与 UTC 时区偏移。

### 1.2 需求目标（量化）

| 指标 | 目标值 | 度量方式 |
|------|--------|----------|
| 信仰内容完整率 | 12/12 信仰全部上线 | 后台数据统计 |
| 圣地数据完整率 | 60/60 圣地含 GPS、图片、描述 | 字段非空率 |
| 首页信仰卡片加载时间 | < 1.5s (P95) | APM 监控 |
| 圣地地图渲染时间 | < 2s (P95) | 前端性能埋点 |
| 用户浏览深度 | 平均 > 3 页/会话 | 埋点分析 |
| 管理后台 CRUD 操作成功率 | > 99.5% | 后端日志 |

### 1.3 需求范围

**包含：**
- 12 大信仰的展示、详情、筛选
- 60 处圣地的列表、地图、详情
- Web / App / 小程序 / 管理后台四端适配
- 管理后台 CRUD（增删改查）
- 中英双语内容支持

**不包含：**
- 用户自定义信仰/圣地（v2）
- 圣地评论/评分（v2）
- 圣地实时天气/人流（v3）
- 圣地 AR 导览（v3）

## 2. 用户故事

### US-01 浏览全球信仰体系

**作为** 一名对宗教文化感兴趣的用户，
**我希望** 在首页看到 12 大信仰的卡片列表，每个信仰有独特的符号和颜色标识，
**以便** 我能快速了解平台覆盖的信仰范围并选择感兴趣的深入了解。

**验收标准：**
- [ ] 信仰列表页展示 12 张卡片，每张含名称（中/英）、符号、主题色、简短描述
- [ ] 点击卡片跳转到信仰详情页
- [ ] 支持中英文切换，内容同步变化
- [ ] 加载态显示骨架屏，错误态显示重试提示

### US-02 探索信仰关联内容

**作为** 一名正在研究某信仰的用户，
**我希望** 在信仰详情页看到该信仰关联的圣地、祖庭、祖师、祖训，
**以便** 我能全面了解该信仰的文化脉络和地理分布。

**验收标准：**
- [ ] 详情页顶部显示信仰名称、符号、颜色主题横幅
- [ ] 下方分区展示：关联圣地（卡片列表）、关联祖庭、关联祖师、关联祖训
- [ ] 每个关联项可点击跳转到对应详情页
- [ ] 空数据时显示"暂无数据"占位

### US-03 在地图上发现圣地

**作为** 一名计划朝圣旅行的用户，
**我希望** 在世界地图上看到 60 处圣地的标记点，点击可查看简介，
**以便** 我能直观了解圣地的地理位置并规划路线。

**验收标准：**
- [ ] 地图使用 Leaflet + CartoDB 暗色瓦片
- [ ] 60 个圣地标记使用对应信仰的主题色
- [ ] 点击标记弹出 popup：圣地名称、所属信仰、国家、缩略图
- [ ] popup 中"查看详情"按钮跳转圣地详情页
- [ ] 支持列表视图/地图视图切换

### US-04 查看圣地详细信息

**作为** 一名即将出发的朝圣者，
**我希望** 在圣地详情页看到详细描述、高清图片、精确位置和相关信仰信息，
**以便** 我能充分了解目的地并做好出行准备。

**验收标准：**
- [ ] 详情页显示：名称（中/英）、国家、高清图片、详细描述
- [ ] 嵌入小地图展示精确位置（lat/lng）
- [ ] 显示所属信仰（可点击跳转）
- [ ] 显示 UTC 时区偏移信息
- [ ] 底部"加入行程"按钮（跳转行程模块）

### US-05 管理员维护信仰与圣地数据

**作为** 平台管理员，
**我希望** 在后台管理界面对信仰和圣地数据进行增删改查操作，
**以便** 我能及时维护和更新平台内容。

**验收标准：**
- [ ] 信仰管理：表格展示所有信仰，支持新增/编辑/删除
- [ ] 圣地管理：表格展示所有圣地，支持按信仰筛选
- [ ] 圣地编辑表单包含地图选点功能（点击地图获取 lat/lng）
- [ ] 删除操作需二次确认
- [ ] 操作成功/失败有明确提示

### 3.2 业务规则清单

| 规则编号 | 描述 | 触发条件 | 处理逻辑 | 异常处理 |
|----------|------|----------|----------|----------|
| R-01 | 信仰 slug 唯一性 | 新增/编辑信仰 | slug 字段做 unique 校验 | 返回 409 Conflict |
| R-02 | 圣地必须关联信仰 | 新增圣地 | religionId 为必填外键 | 返回 400 Bad Request |
| R-03 | GPS 坐标范围校验 | 新增/编辑圣地 | latitude: -90~90, longitude: -180~180 | 返回 400 参数错误 |
| R-04 | UTC 偏移范围校验 | 新增/编辑圣地 | utcOffset: -12~14 | 返回 400 参数错误 |
| R-05 | 删除信仰级联检查 | 删除信仰 | 检查是否有关联圣地/祖庭/祖师/祖训 | 有关联则禁止删除，提示先移除关联 |
| R-06 | 图片 URL 格式校验 | 编辑圣地 imageUrl | 校验 URL 格式（http/https 开头） | 返回 400 参数错误 |
| R-07 | 列表分页默认值 | 请求列表 API | 默认 page=1, limit=20 | — |
| R-08 | 信仰筛选 | 圣地列表带 religionId 参数 | 返回该信仰下的圣地 | religionId 无效返回空数组 |

## 5. 数据模型

### 5.1 实体关系

```
Religion (1) ──────< (N) HolySite
Religion (1) ──────< (N) Temple
Religion (1) ──────< (N) Patriarch
Religion (1) ──────< (N) Teaching
```

Religion 是核心实体，与圣地、祖庭、祖师、祖训均为一对多关系。

### 5.2 核心字段

**Religion 信仰表：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | String (cuid) | 是 | 主键 |
| name | String | 是 | 中文名称，如"佛教" |
| nameEn | String | 是 | 英文名称，如"Buddhism" |
| slug | String (unique) | 是 | URL 标识，如"buddhism" |
| symbol | String | 是 | 信仰符号字符，如"☸" |
| color | String | 是 | 主题色 HEX，如"#F59E0B" |
| description | String | 否 | 中文描述 |
| descriptionEn | String | 否 | 英文描述 |
| createdAt | DateTime | 是 | 创建时间（自动） |
| updatedAt | DateTime | 是 | 更新时间（自动） |

**HolySite 圣地表：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | String (cuid) | 是 | 主键 |
| name | String | 是 | 中文名称，如"菩提伽耶" |
| nameEn | String | 是 | 英文名称，如"Bodh Gaya" |
| country | String | 是 | 所在国家，如"印度" |
| latitude | Float | 是 | 纬度，范围 -90 ~ 90 |
| longitude | Float | 是 | 经度，范围 -180 ~ 180 |
| utcOffset | Float | 是 | UTC 时区偏移，如 5.5（印度） |
| description | String | 否 | 中文描述 |
| descriptionEn | String | 否 | 英文描述 |
| imageUrl | String | 否 | 主图 URL |
| soundEffect | String | 否 | 音效文件 URL |
| religionId | String (FK) | 是 | 关联信仰 ID |
| createdAt | DateTime | 是 | 创建时间 |
| updatedAt | DateTime | 是 | 更新时间 |

## 6. API 接口

| 接口 | Method | 路径 | 入参 | 出参 | 说明 |
|------|--------|------|------|------|------|
| 信仰列表 | GET | /api/religions | — | Religion[] | 返回全部 12 个信仰 |
| 信仰详情 | GET | /api/religions/:slug | slug: string | Religion & { holySites, temples, patriarchs, teachings } | 按 slug 获取，含关联数据 |
| 新增信仰 | POST | /api/religions | ReligionCreateDto | Religion | 管理员权限 |
| 编辑信仰 | PATCH | /api/religions/:id | ReligionUpdateDto | Religion | 管理员权限 |
| 删除信仰 | DELETE | /api/religions/:id | id: string | { success: true } | 管理员权限，级联检查 |
| 圣地列表 | GET | /api/holy-sites | ?religionId, ?page, ?limit | { data: HolySite[], total } | 支持信仰筛选与分页 |
| 圣地详情 | GET | /api/holy-sites/:id | id: string | HolySite & { religion } | 含关联信仰信息 |
| 新增圣地 | POST | /api/holy-sites | HolySiteCreateDto | HolySite | 管理员权限 |
| 编辑圣地 | PATCH | /api/holy-sites/:id | HolySiteUpdateDto | HolySite | 管理员权限 |
| 删除圣地 | DELETE | /api/holy-sites/:id | id: string | { success: true } | 管理员权限 |

**ReligionCreateDto：**
```json
{
  "name": "string (required)",
  "nameEn": "string (required)",
  "slug": "string (required, unique)",
  "symbol": "string (required)",
  "color": "string (required, hex)",
  "description": "string",
  "descriptionEn": "string"
}
```

**HolySiteCreateDto：**
```json
{
  "name": "string (required)",
  "nameEn": "string (required)",
  "country": "string (required)",
  "latitude": "number (required, -90~90)",
  "longitude": "number (required, -180~180)",
  "utcOffset": "number (required, -12~14)",
  "description": "string",
  "descriptionEn": "string",
  "imageUrl": "string (url)",
  "soundEffect": "string (url)",
  "religionId": "string (required, FK)"
}
```

## 7. 多端页面规格

### 7.1 Web 端

#### 页面 W-01: 信仰列表页 `/religions`

**入口：** 顶部导航栏"信仰"菜单 / 首页"探索信仰"按钮

**布局描述：**
```
┌────────────────────────────────────────────┐
│  顶部导航栏 (深色, logo + 菜单 + 语言切换) │
├────────────────────────────────────────────┤
│  页面标题: "十二大信仰" / "12 Faiths"      │
│  副标题: 简短介绍文字                       │
├────────────────────────────────────────────┤
│  ┌────┐ ┌────┐ ┌────┐ ┌────┐             │
│  │佛教│ │道教│ │基督│ │伊斯│             │
│  │ ☸  │ │ ☯  │ │ ✝  │ │ ☪  │  ... x12   │
│  └────┘ └────┘ └────┘ └────┘             │
│  (4列网格, 每卡片: 符号+颜色背景+名称+描述) │
├────────────────────────────────────────────┤
│  页脚                                      │
└────────────────────────────────────────────┘
```

**字段表：**

| 字段 | 来源 | 显示格式 | 备注 |
|------|------|----------|------|
| 信仰符号 | religion.symbol | 大号居中显示 | 背景色为 religion.color |
| 信仰名称 | religion.name / nameEn | 根据语言切换 | 金色文字 #D4A855 |
| 简短描述 | religion.description | 截断 80 字 | 灰色文字 |

**交互表：**

| 交互 | 触发 | 行为 | 备注 |
|------|------|------|------|
| 点击卡片 | click | 跳转 /religions/[slug] | 带过渡动画 |
| hover 卡片 | mouseenter | 卡片上浮 + 边框发光（信仰色） | 仅 Web |
| 语言切换 | 切换 i18n | 名称/描述切换中英文 | — |

**状态：**
- **加载中：** 12 个骨架屏卡片（shimmer 动画）
- **加载成功：** 正常展示 12 张卡片
- **加载失败：** 居中错误提示 + 重试按钮
- **空数据：** 不适用（种子数据保证 12 条）

### 7.2 App 端（Expo React Native）

#### 页面 A-01: 圣地 Tab `/holy-sites`

**入口：** 底部 Tab 栏"圣地"

**布局：** 顶部搜索栏 + 信仰筛选横向滚动标签 + 竖向 FlatList 圣地卡片

**与 Web 差异：**
- 无地图视图切换（地图仅在 Web 端），App 端圣地列表为纯卡片列表
- 信仰筛选使用横向滚动标签（ScrollView horizontal）而非下拉框
- 卡片为单列全宽（非网格）
- 下拉刷新 + 上拉加载更多（无分页器）

#### 页面 A-02: 圣地详情 `/holy-sites/[id]`

**布局：** ScrollView，顶部大图 → 名称/信仰标签 → 描述 → 底部固定"加入行程"按钮

**与 Web 差异：**
- 无嵌入小地图（简化），仅显示 GPS 坐标文字
- 底部按钮使用固定定位（sticky bottom）
- 支持左滑返回手势

#### 页面 A-03: 信仰详情 `/religions/[slug]`

**布局：** ScrollView，顶部信仰色横幅 → 描述 → 关联圣地横向滚动 → 关联祖庭/祖师

**与 Web 差异：**
- 关联内容不使用 Tab 切换，改为纵向 Section 排列
- 每个 Section 最多显示 5 项 + "查看更多"
