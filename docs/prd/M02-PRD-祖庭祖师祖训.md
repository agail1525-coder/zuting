# M02 祖庭祖师祖训 产品需求文档

> 版本: v1.0 | 日期: 2026-03-25 | 状态: 已发布

---

## 1. 需求背景与目标

### 1.1 业务背景

祖庭（Temple）、祖师（Patriarch）、祖训（Teaching）共同构成平台的"传承遗产"内容体系。祖庭是信仰传承的物理载体，祖师是信仰精神的人格化身，祖训是信仰智慧的文字凝结。三者相互关联，围绕 Religion 主实体组织，为用户提供深入了解各信仰文化传承脉络的内容通道。

当前数据规模：27 座祖庭、28 位祖师、39 条祖训，覆盖 12 大信仰体系。

### 1.2 需求目标（量化）

| 指标 | 目标值 | 度量方式 |
|------|--------|----------|
| 内容完整率 | 27 祖庭 + 28 祖师 + 39 祖训全部上线 | 后台数据统计 |
| 详情页停留时长 | 祖师/祖训详情页平均 > 45s | 埋点分析 |
| 关联跳转率 | 信仰详情页 → 祖庭/祖师/祖训点击率 > 15% | 埋点分析 |
| 管理后台操作响应 | CRUD 操作 < 500ms (P95) | API 监控 |
| 数据字段完整度 | 必填字段非空率 100% | 数据校验脚本 |

### 1.3 需求范围

**包含：**
- 祖庭列表与详情展示（含国家、创建年代、GPS）
- 祖师列表与详情展示（含生卒年、头衔、核心教导）
- 祖训列表与详情展示（含原文、来源、中文翻译）
- 以上三类在 Web / App / 小程序 / Admin 四端的适配
- 管理后台 CRUD 操作
- 中英双语支持

**不包含：**
- 祖庭 VR 导览（v3）
- 祖师关系图谱（v2）
- 祖训每日推送（v2）
- 用户收藏/笔记功能（v2）

## 2. 用户故事

### US-01 浏览信仰祖庭

**作为** 一名想要了解佛教历史建筑的用户，
**我希望** 看到佛教相关的祖庭列表，包含名称、创建年代、所在国家和图片，
**以便** 我能选择想要深入了解或实地参访的祖庭。

**验收标准：**
- [ ] 祖庭列表页展示所有祖庭卡片
- [ ] 支持按信仰筛选
- [ ] 每张卡片显示：名称（中/英）、国家、创建年代、封面图
- [ ] 点击跳转祖庭详情页

### US-02 了解祖师生平

**作为** 一名对宗教历史人物感兴趣的用户，
**我希望** 在祖师详情页看到其生卒年份、头衔、完整传记和核心教导，
**以便** 我能深入了解这位宗教先贤的生平与思想。

**验收标准：**
- [ ] 详情页展示祖师头像/肖像图
- [ ] 显示生卒年份（如公元前 563 - 公元前 483）
- [ ] 显示头衔/尊称
- [ ] 展示传记全文（支持长文排版）
- [ ] 突出展示核心教导（coreTeaching）
- [ ] 底部关联：所属信仰（可点击跳转）

### US-03 阅读祖训原文

**作为** 一名修行实践者，
**我希望** 阅读各信仰的经典祖训原文及其翻译，
**以便** 我能从源头汲取信仰智慧。

**验收标准：**
- [ ] 祖训详情页以醒目排版展示原文（originalText）
- [ ] 显示来源出处（sourceText）
- [ ] 显示中文翻译（translationCn）
- [ ] 显示所属信仰（可点击）
- [ ] 原文支持大号字体展示，适合阅读冥想

### US-04 从信仰详情跳转

**作为** 一名正在浏览某信仰详情的用户，
**我希望** 能直接从信仰详情页的"祖庭"/"祖师"/"祖训"区域点击进入对应详情，
**以便** 我能顺畅地在关联内容间跳转。

**验收标准：**
- [ ] 信仰详情页的祖庭/祖师/祖训卡片可点击
- [ ] 跳转到对应详情页后，有"返回"按钮回到信仰详情
- [ ] 面包屑导航正确显示层级

### US-05 管理员维护遗产数据

**作为** 管理员，
**我希望** 在后台分别管理祖庭、祖师、祖训数据，
**以便** 我能独立维护这三类内容。

**验收标准：**
- [ ] 三个独立管理页面：TemplesPage, PatriarchsPage, TeachingsPage
- [ ] 每个页面支持增删改查
- [ ] 新增/编辑时必须选择所属信仰
- [ ] 表格支持按信仰筛选和关键词搜索

### 3.2 业务规则清单

| 规则编号 | 描述 | 触发条件 | 处理逻辑 | 异常处理 |
|----------|------|----------|----------|----------|
| R-01 | 祖庭必须关联信仰 | 新增祖庭 | religionId 必填 | 400 Bad Request |
| R-02 | 祖师必须关联信仰 | 新增祖师 | religionId 必填 | 400 Bad Request |

| R-08 | 信仰筛选 | 列表 API 带 religionId | 只返回该信仰下的数据 | 无效 ID 返回空数组 |
| R-09 | 年份格式 | 祖师 birthYear/deathYear | 整数，负数表示公元前（如 -563 = BC 563） | 400 格式错误 |

## 5. 数据模型

### 5.1 实体关系

```
Religion (1) ──────< (N) Temple
Religion (1) ──────< (N) Patriarch
Religion (1) ──────< (N) Teaching

Temple     ──── 独立实体，通过 religionId 关联信仰
Patriarch  ──── 独立实体，通过 religionId 关联信仰
Teaching   ──── 独立实体，通过 religionId 关联信仰
```

三者之间目前无直接关联，均通过 Religion 间接联系。v2 可考虑增加 Patriarch ↔ Temple（祖师驻锡祖庭）、Patriarch ↔ Teaching（祖师所授祖训）关系。

### 5.2 核心字段

**Temple 祖庭表：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | String (cuid) | 是 | 主键 |
| name | String | 是 | 中文名称，如"少林寺" |
| nameEn | String | 是 | 英文名称，如"Shaolin Temple" |
| country | String | 是 | 所在国家 |
| foundingDate | String | 否 | 创建年代描述，如"公元495年" |
| description | String | 否 | 中文描述 |
| descriptionEn | String | 否 | 英文描述 |
| imageUrl | String | 否 | 封面图片 URL |
| latitude | Float | 否 | 纬度 |
| longitude | Float | 否 | 经度 |
| religionId | String (FK) | 是 | 关联信仰 ID |
| createdAt | DateTime | 是 | 创建时间 |
| updatedAt | DateTime | 是 | 更新时间 |

**Patriarch 祖师表：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | String (cuid) | 是 | 主键 |
| name | String | 是 | 中文姓名，如"释迦牟尼" |
| nameEn | String | 是 | 英文姓名，如"Shakyamuni Buddha" |
| birthYear | Int | 否 | 出生年份（负数=公元前） |
| deathYear | Int | 否 | 去世年份（负数=公元前） |
| title | String | 否 | 头衔/尊称，如"佛陀" |
| titleEn | String | 否 | 英文头衔 |
| biography | String | 否 | 传记（长文本） |
| biographyEn | String | 否 | 英文传记 |
| coreTeaching | String | 否 | 核心教导摘要 |
| coreTeachingEn | String | 否 | 英文核心教导 |
| imageUrl | String | 否 | 肖像/头像图片 URL |
| religionId | String (FK) | 是 | 关联信仰 ID |
| createdAt | DateTime | 是 | 创建时间 |
| updatedAt | DateTime | 是 | 更新时间 |

**Teaching 祖训表：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | String (cuid) | 是 | 主键 |
| name | String | 是 | 祖训标题/编号 |
| originalText | String | 是 | 原文内容 |
| sourceText | String | 否 | 来源出处，如"《金刚经》" |
| translationCn | String | 否 | 中文白话翻译 |
| translationEn | String | 否 | 英文翻译 |
| religionId | String (FK) | 是 | 关联信仰 ID |
| createdAt | DateTime | 是 | 创建时间 |
| updatedAt | DateTime | 是 | 更新时间 |

### 7.2 App 端（Expo React Native）

#### 页面 A-01: 祖庭详情 `/temples/[id]`

**入口：** 信仰详情页关联祖庭卡片点击

**布局：** ScrollView — Hero 图 → 名称/国家/年代 → 描述 → 信仰链接

**与 Web 差异：**
- 无独立祖庭列表页（从信仰详情进入）
- 无嵌入地图，仅文字显示坐标
- 支持左滑返回

#### 页面 A-02: 祖师详情 `/patriarchs/[id]`

**入口：** 信仰详情页关联祖师卡片点击

**布局：** ScrollView — 顶部头像（居中圆形） → 姓名/头衔/生卒年 → 核心教导高亮卡片 → 传记全文

**与 Web 差异：**
- 单栏布局（非左右分栏）
- 传记长文本支持展开/收起（默认显示前 300 字）

#### 页面 A-03: 祖训详情（从信仰详情进入）

**布局：** 全屏卡片风格 — 大号原文居中 → 来源 → 翻译（可下滑查看）

**与 Web 差异：**
- 更强调阅读沉浸感，原文占据大部分屏幕
- 无同信仰推荐区域
