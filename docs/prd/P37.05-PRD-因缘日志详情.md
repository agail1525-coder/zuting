# P37.05-PRD-因缘日志详情

> 圆满之路 A5 轴 | 路由 /trips/cultivation/karma-journal/:id | AI 自动标注境界 + 因果链

## 1. 页面定位
升级版日志 — 普通日志只记发生了什么, 因缘日志 **由 AI 自动标注**:
- 境界层级 (七境界哪一层)
- 因果链 (这件事的"因"是什么, "果"会是什么)
- 修行建议 (当前该做什么)

## 2. 与 M07 (朝圣日志) 的关系
- M07 保留, 作为"旅行足迹记录"
- M37 新增因缘日志, 作为"心灵修行记录"
- 可互相关联: 旅行日志可一键转为因缘日志 (追加 AI 分析)

## 3. 数据流
```
用户记录事件 → POST /karma/event
  → Service 调小鸿 NLP pipeline (F37.04)
    → 返回 { realmTag, causeTag, effectTag, advice }
  → 保存到 KarmaEvent
  → 若 realmTag 与当前境界不符, 触发 "境界重评" 建议
```

## 4. 布局
```
┌───────────────────────────────────┐
│ 2026-04-10 · 晴 · 深圳            │
├───────────────────────────────────┤
│ 标题: 合伙人提出退股              │
│                                     │
│ [正文] ... 用户输入 ...           │
│                                     │
│ 🤖 小鸿标注:                      │
│   境界: 证道期 (考验定力)          │
│   因: 三年前未建立清晰股权协议    │
│   果: 若处理得当, 此为得牛之机    │
│   建议: 晨课修"无相禅", 参P37.04  │
│                                     │
│ [重新标注] [转朝圣日志] [分享]    │
└───────────────────────────────────┘
```

## 5. API
- `POST /trips/cultivation/karma/event` — 创建 + 自动 AI 标注
- `GET /trips/cultivation/karma/event/:id` — 详情
- `GET /trips/cultivation/karma/timeline?year=2026` — 时间轴
- `PATCH /trips/cultivation/karma/event/:id` — 编辑 (重新 AI 标注)
- `DELETE /trips/cultivation/karma/event/:id`

## 6. 隐私
- 默认 PRIVATE
- 可选 FRIENDS (师徒圈) / PUBLIC (入社区)
- AI 标注不上链外部, 仅本地 LLM

## 7. 验收
- [ ] AI 标注 < 5s
- [ ] 标注可编辑覆盖
- [ ] 时间轴按月聚合
- [ ] IDOR 防护 (必须 userId 匹配)
- [ ] body 入库前 DOMPurify
- [ ] 软删除, 可恢复
