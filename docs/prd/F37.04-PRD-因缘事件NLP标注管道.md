# F37.04-PRD-因缘事件NLP标注管道

> A5 因缘日志的核心引擎 | 用户记录 → AI 标注境界 + 因果链 + 建议

## 1. 目标
用户自由文本记录一件事 → 小鸿自动输出结构化标注:
```json
{
  "realmTag": "ATTAINING",
  "causeTag": "三年前未建立清晰股权协议",
  "effectTag": "若处理得当可转为得牛之机",
  "advice": "晨课修'无相禅', 参融通问答",
  "confidence": 0.82
}
```

## 2. Pipeline 步骤

```
原始文本
  ↓ [Step 1] 清洗: DOMPurify + 去除 emoji/控制字符
  ↓ [Step 2] 情感分析: positive/neutral/negative + 强度
  ↓ [Step 3] 实体抽取: 人/地/物/事件
  ↓ [Step 4] 境界分类: LLM 单次调用, 输出 Realm 枚举
  ↓ [Step 5] 因果抽取: LLM 输出 cause/effect 短句
  ↓ [Step 6] 建议生成: 基于用户主修文化 + 境界 + 事件
  ↓ [Step 7] 置信度: LLM self-consistency (3次采样投票)
```

## 3. Prompt 模板

```
你是【小鸿】, 佳绩之旅圆满之路 AI 助手.

用户主修: ${tradition}
用户当前境界: ${currentRealm}

用户记录的事件:
【${title}】
${body}

请输出 JSON:
{
  "realmTag": "AWAKENING|CLARIFYING|SEEING|ATTAINING|INTEGRATING|RETURNING|GIVING_BACK",
  "causeTag": "20字以内的因",
  "effectTag": "20字以内的果",
  "advice": "50字以内的修行建议, 基于 ${tradition} 智慧",
  "confidence": 0.0-1.0
}

只输出 JSON, 不要其他文字.
```

## 4. 降级策略

| 失败点 | 降级 |
|--------|------|
| LLM 超时 5s | 返回 `{ realmTag: null, advice: "稍后小鸿将为你分析" }` + 异步重试队列 |
| JSON 解析失败 | 取第一次完整 JSON 匹配, 否则 null |
| 敏感内容 | moderation 拦截, 不入库标注, 原文可存 |
| 置信度 < 0.5 | realmTag = null, 前端显示"境界待定" |

## 5. 异步队列
- 失败事件入 BullMQ `karma-nlp-retry`, 最多重试 3 次
- 重试成功后通过 WebSocket 推送用户 "因缘已分析完成"

## 6. 批量重标注
- 用户切换主修文化时, 提供"重新分析最近 30 条"按钮
- 境界重算 CRON 同步触发重标注

## 7. API
- 内部服务: `karmaNlpService.annotate(event)` 不对外
- 调用点: POST /karma/event 创建时 + PATCH 编辑时

## 8. 监控
- Prometheus 指标:
  - `karma_nlp_duration_seconds` histogram
  - `karma_nlp_success_total` counter
  - `karma_nlp_fallback_total` counter
  - `karma_nlp_confidence_avg` gauge

## 9. 验收
- [ ] 平均响应 < 5s
- [ ] 成功率 ≥ 90%
- [ ] 重试队列工作正常
- [ ] 人工评测 realmTag 准确率 ≥ 75%
- [ ] 敏感内容 100% 拦截
- [ ] 用户可一键重新标注
