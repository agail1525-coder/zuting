### 4.2 移动端适配要点

```
- Hero卡片: 宽度100%，圆角16px，深色背景带光晕效果
- 快捷操作: 3列均分，图标64px，触摸区域≥44px
- 进度条: 高度8px，当前等级色渐变填充
- 特权网格: 3列，已解锁亮色，未解锁灰色+锁图标
- 等级对比: 横向滚动表格，当前等级列高亮背景
- 积分明细: 左侧彩色圆点（绿/红），右侧时间右对齐
```

## 五、交互设计

### 5.1 状态机

```
页面加载:
  → 骨架屏Loading
  → 有会员数据: 渲染完整页面
  → API Error: 显示错误提示+重试按钮
  → 未登录: 跳转登录页，登录后返回

签到按钮:
  未签到: [签到领取 +5分] — 主色按钮，可点击
  已签到: [今日已签到 ✓] — 灰色按钮，禁用
  签到中: [签到中...] — Loading状态

积分兑换:
  点击"积分兑换" → 打开兑换弹窗/跳转兑换页
  兑换成功 → Toast提示 + 积分数字刷新动画
```

### 5.2 动效规范

```
- Hero卡片: 进入时从下方slide-up + fade-in，时长300ms
- 积分数字: 签到成功后数字滚动动画，+N积分气泡浮起
- 进度条: 页面加载时从左到右填充动画，时长600ms
- 等级Tab切换: 滑动切换，高亮指示器跟随
```

## 六、API调用

| 操作 | API端点 | 时机 |
|------|---------|------|
| 获取会员信息 | GET /api/membership/me | 页面挂载 |
| 获取积分明细 | GET /api/membership/transactions?pageSize=5 | 页面挂载 |
| 每日签到 | POST /api/membership/checkin | 点击签到按钮 |
| 签到日历 | GET /api/membership/checkin-calendar | 签到区域展开 |

## 七、验收标准

- [ ] Hero卡片展示等级名称、徽章颜色、可用积分、进度条
- [ ] 进度条百分比与API数据progressPercent一致
- [ ] 已达最高Lv5时进度条满格，显示"已达最高等级"
- [ ] 签到按钮状态：未签到可点击，已签到禁用并显示确认文案
- [ ] 签到成功后积分数字更新，无需刷新页面（乐观更新）
- [ ] 特权网格：已解锁vs未解锁视觉区别明显
- [ ] 等级对比表当前等级列有高亮背景
- [ ] 积分明细EARN显示绿色+号，SPEND显示红色-号
- [ ] 点击"查看全部"跳转积分流水完整列表页
- [ ] Loading骨架屏与真实布局一致
- [ ] 未登录跳转登录，登录后回到本页
