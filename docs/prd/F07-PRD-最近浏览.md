## 1. 功能概述

### 1.1 功能定位

"最近浏览"记录用户访问过的实体详情页，并在首页和个人中心提供快速回访入口。该功能同时为推荐算法提供用户偏好数据，是推荐系统的数据基础。

### 1.2 两个展示入口

| 入口 | 位置 | 内容 | 上限 |
|------|------|------|------|
| 首页"继续浏览"区块 | 首页 (位于"为你推荐"之后) | 最近10条 | 10条 |
| 个人中心"最近浏览"页 | 个人中心 → 最近浏览 | 全部历史 | 50条 |

---

## 2. 竞品对标

### 2.1 Trip.com "最近浏览"

- **位置**: 首页横向滚动区块 + 个人中心列表页
- **去重规则**: 同一酒店重复访问只保留最新一条
- **清空功能**: 个人中心页面顶部"清空历史"按钮，二次确认

### 2.2 Booking.com "Recently viewed"

- **未登录处理**: 使用 localStorage 保存，刷新/关闭浏览器后保留
- **登录同步**: 登录后自动将 localStorage 数据同步到服务器
- **时间显示**: "2小时前" / "昨天" / "3天前" 相对时间格式

### 2.3 Airbnb

- **展示方式**: 首页横向滚动，最多显示 5 张，右侧"查看全部"
- **图片质量**: 使用高质量缩略图，与详情页主图相同

## 3. 用户故事

```
作为已登录用户:
  我浏览过少林寺详情页后，第二天回到首页
  "继续浏览"区块显示少林寺，我可以一键继续浏览
  我可以在个人中心查看并清空我的全部浏览历史

作为未登录用户:
  我浏览了几个圣地后关闭了浏览器
  重新打开后首页"继续浏览"区块仍显示我之前看的内容
  (localStorage 持久化)

作为推荐系统(内部):
  用户每次访问详情页，系统静默记录行为
  分析最近浏览的宗教类型分布 → 驱动首页个性化推荐
```

## 4. 记录逻辑

### 4.1 触发时机

```
触发条件: 用户打开任意实体详情页
  - 圣地详情页 (/holy-sites/[id])
  - 祖庭详情页 (/temples/[id])
  - 祖师详情页 (/patriarchs/[id])

触发方式: 静默 (不阻塞页面加载，不显示任何 UI 反馈)
执行时机: 页面 onMount 后延迟 1s 执行 (避免影响 LCP)
```

### 4.2 去重规则

```
同一实体 24h 内重复访问 → 只更新 viewedAt，不新增记录
超过 24h 后再次访问 → 视为新记录，更新到列表顶部

实现: POST /api/user/view-history 后端 upsert 逻辑
  WHERE userId = X AND entityId = Y AND viewedAt > (now - 24h)
  → EXISTS: UPDATE viewedAt = now()
  → NOT EXISTS: INSERT new record
```

### 4.3 存储策略

| 用户类型 | 存储方式 | 上限 | 持久化 |
|---------|---------|------|--------|
| 已登录 | Redis List + DB (双写) | 50条 | 永久 |
| 未登录 | localStorage | 20条 | 浏览器本地 |

**登录后同步**: 用户登录时，将 localStorage 中的历史合并到服务端 (去重后按时间排序)

## 6. 清空功能

### 6.1 交互流程

```
1. 用户点击"清空"按钮
2. 弹出确认弹窗: "确认清空全部浏览历史？此操作不可撤销"
3. [取消] → 关闭弹窗
4. [确认清空] → 发起 DELETE /api/user/view-history
5. 列表清空，显示空状态: "暂无浏览记录"
6. 首页"继续浏览"区块消失
```

### 6.2 空状态

```
图示: 📋 (圆形图标)
标题: 暂无浏览记录
副标题: 探索圣地后，这里会显示你的浏览足迹
按钮: [去探索圣地] → /holy-sites
```

## 7. API 调用

### 7.1 记录浏览 (静默调用)

```
POST /api/user/view-history
Headers: Authorization: Bearer {token} (可选)
Body:
{
  "entityType": "HOLY_SITE",
  "entityId": "clxxx",
  "sessionId": "sess_xxx"  // 未登录时使用
}
Response: 201 Created (无 body)
```

### 7.2 获取浏览历史

```
GET /api/user/view-history?limit=10
Headers: Authorization: Bearer {token}
Response:
{
  "items": [
    {
      "entityType": "HOLY_SITE",
      "entityId": "clxxx",
      "name": "少林寺",
      "religion": "BUDDHISM",
      "religionName": "佛教",
      "region": "河南，中国",
      "rating": 4.9,
      "imageUrl": "https://...",
      "viewedAt": "2026-03-29T10:30:00Z",
      "viewedAtRelative": "2小时前"
    }
  ],
  "total": 28
}
```

### 7.3 清空历史

```
DELETE /api/user/view-history
Headers: Authorization: Bearer {token}
Response: 200 OK
```

## 9. 验收标准

- [ ] 打开圣地/祖庭/祖师详情页后，浏览历史正确记录
- [ ] 同一实体 24h 内重复访问只保留一条记录 (更新时间)
- [ ] 首页"继续浏览"区块显示最近 10 条历史
- [ ] 无浏览历史时首页区块不渲染
- [ ] 个人中心"最近浏览"页最多显示 50 条
- [ ] 时间显示格式正确 (相对时间)
- [ ] 一键清空功能有二次确认，清空后列表为空状态
- [ ] 未登录用户浏览历史存储于 localStorage
- [ ] 登录后 localStorage 历史自动同步到服务端
- [ ] 浏览记录 POST 为非阻塞，不影响详情页 LCP
