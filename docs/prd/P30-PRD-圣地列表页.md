# P30-PRD-圣地列表页 (Holy Sites Listing Page)

> CEO++页面++ | 版本: v2.0 | 日期: 2026-04-01
> 对标: Booking.com目的地列表 + TripAdvisor景点列表 + Airbnb体验列表

---

## 1. 概述

圣地列表页是用户探索全球60个宗教圣地的核心入口。当前版本(v1)功能骨架完整但与竞品差距显著：缺少真实评分数据、分类Hero视觉、收藏交互、价格信息、大洲筛选等。本次升级对标三大竞品，将页面从"能用"提升到"想用"。

### 1.1 现状评估 (v1 → v2)

| 维度 | v1现状 | v2目标 | 竞品参考 |
|------|--------|--------|---------|
| 数据真实性 | hashStr假评分+假访客 | 真实Review聚合评分 | Booking/TripAdvisor |
| 视觉层次 | 平铺网格,无Hero | 信仰分类Hero卡片+推荐区 | Airbnb/Trip.com |
| 卡片丰富度 | 名称+国家+描述 | +评分+评价数+价格+收藏 | Booking详情卡 |
| 筛选能力 | 信仰+国家+搜索+排序 | +大洲+评分范围+价格+门票 | Kayak/Booking |
| 转化设计 | 仅底部CTA | 卡片内"查看路线"+收藏 | Airbnb/Trip.com |
| i18n | 15+处硬编码中文 | 全量i18n | 全部竞品 |

## 5. API变更

### 5.1 后端: HolySite列表enrichment

```typescript
// GET /api/holy-sites?page=1&limit=20&religionId=xxx
// 响应增加字段:
{
  items: [{
    ...existingFields,
    reviewStats: { averageRating: 4.5, reviewCount: 23 },
    collectionCount: 15, // 被收藏次数
  }],
  total, page, limit
}
```

实现方式: 在HolySiteService.findAll中，对每批结果聚合Review统计和Collection计数。

### 5.2 前端: fetchHolySites 响应类型扩展

```typescript
interface HolySiteWithStats extends HolySite {
  reviewStats?: { averageRating: number; reviewCount: number };
  collectionCount?: number;
}
```
