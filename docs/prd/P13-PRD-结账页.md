> 版本: v1.0 | 创建: 2026-03-29 | 状态: 待评审
> 页面编号: P13 | 所属模块: M06-订单与支付 | Sprint: C1

---

## 1. 页面定位

结账页是用户将行程意向转化为实际订单的关键转化节点，也是促销和优惠券兑现的唯一入口。设计原则：**最小化摩擦，最大化信任感**。

### 1.1 竞品对标

| 竞品 | 核心设计亮点 | 借鉴点 |
|------|-------------|--------|
| Booking.com | 两栏布局：左侧摘要+右侧价格/支付，顶部进度条 | 信息层次清晰，用户无需滚动即可看到价格 |
| Agoda | 价格明细折叠展开，优惠券输入框显眼，支付方式大图标 | 优惠券 UX、多种支付方式并列 |
| 飞猪/携程 | 国内支付习惯：微信/支付宝优先，倒计时锁单 | 微信支付优先级、锁单倒计时 |
| Airbnb | 价格明细完全透明（每项费用解释），右侧浮动卡片 | 信任感建设，费用透明化 |

## 2. 用户故事

```
US-01: 作为用户，我希望在结账页看到行程的完整摘要（日期/人数/圣地），以便确认购买内容
US-02: 作为用户，我希望看到完整的价格明细（原价/折扣/优惠券/税费/总价），不被隐性费用惊吓
US-03: 作为用户，我希望能在结账页选择并应用优惠券，看到即时更新的折扣效果
US-04: 作为用户，我希望看到当前适用的促销标签，如无需手动输入即可自动应用
US-05: 作为用户，我希望选择微信/支付宝/Stripe等支付方式后流畅完成支付
US-06: 作为移动端用户，我希望结账页在手机上也能流畅使用，支付按钮始终在屏幕底部可见
US-07: 作为用户，我希望提交订单前有倒计时提示（如"此价格保留15分钟"），增加决策紧迫感
```

## 3. 功能清单

### P0（必须）

- [ ] 行程摘要展示：标题/封面图/日期/人数/圣地列表（最多3条，超出折叠）
- [ ] 价格明细展示：原价 / 促销折扣 / 优惠券折扣 / 平台服务费 / 税费 / 应付总计
- [ ] 优惠券选择器：弹出可用优惠券列表，支持手动输入码
- [ ] 促销标签自动展示：若有适用促销，自动展示折扣信息（无需手动输入）
- [ ] 促销与优惠券互斥：已选促销时禁止再选优惠券，反之亦然；UI 层面有明确提示
- [ ] 支付方式选择：微信支付 / 支付宝 / Stripe（信用卡）
- [ ] 提交订单按钮：显示最终金额，Loading 状态防重复提交
- [ ] 订单创建成功后跳转支付结果页（P14）

### P1（重要）

- [ ] 价格保留倒计时（15分钟锁单）
- [ ] 支付宝 / 微信唤起（移动端 deep link / H5 跳转）
- [ ] 联系信息确认：姓名/手机（预填用户信息，允许修改）
- [ ] 特殊需求备注输入框（可选）
- [ ] 订单摘要折叠/展开（移动端默认折叠）

### P2（增强）

- [ ] 分期付款选项（Stripe Installments）
- [ ] 企业开票信息填写入口
- [ ] 礼品订单（赠送他人朝圣行程）

```
POST /api/orders
Authorization: Bearer {token}
```

**Request Body:**
```json
{
  "tripId": "trip_xxx",
  "paymentMethod": "WECHAT_PAY | ALIPAY | STRIPE",
  "promotionId": "promo_xxx",
  "couponId": null,
  "contactName": "张三",
  "contactPhone": "13800008888",
```

**Response 201:**
```json
{
  "orderId": "order_yyy",
  "paymentUrl": "weixin://wxpay/...",
  "qrCodeUrl": "https://...",      // PC端二维码
  "expiredAt": "2026-04-30T10:15:00Z"  // 支付二维码有效期

```

## 6. 交互细节

### 6.1 促销与优惠券互斥逻辑

```
状态机:
  NONE → 选择促销 → 已选促销（优惠券入口置灰 + tooltip: "已使用促销，不可叠加优惠券"）
  NONE → 选择优惠券 → 已选优惠券（促销标签变为灰色 + tooltip: "已使用优惠券，不可叠加促销"）
  已选促销 → 取消促销 → NONE（优惠券恢复可选）
  已选优惠券 → 取消优惠券 → NONE（促销恢复可选）
```

### 6.2 支付方式处理

| 支付方式 | PC 端 | 移动端 |
|---------|-------|-------|
| 微信支付 | 展示二维码弹层，轮询支付状态 | deep link 唤起微信APP |
| 支付宝 | 展示二维码 或 跳转新标签页 | H5 跳转支付宝 |
| Stripe  | Stripe Elements 嵌入式表单 | Stripe Mobile SDK |

### 6.3 锁单倒计时

- 进入结账页后，后端生成15分钟价格锁定（Redis TTL）
- 前端展示倒计时，归零后提示"价格已变动，请刷新"
- 刷新后重新获取最新价格

## 7. 状态与错误处理

| 状态 | 展示 |
|------|------|
| Loading（提交中） | 按钮 disabled + spinner，防重复提交 |
| 网络错误 | Toast: "网络异常，请重试" |
| 促销已失效 | Banner 提示，自动移除促销重算价格 |
| 库存不足（配额耗尽） | Toast: "该促销已售罄"，移除促销 |
| 支付超时 | 跳转 P14 失败页，提示重新支付 |
| 金额不一致（409） | Toast: "价格已变动，正在刷新..." 自动 reload |

## 8. 技术约束

```
[HH-V01] 必须接入真实 API，禁止 mock 数据结账
[R-63]   提交订单接口必须有 JwtAuthGuard，禁止匿名下单
[R-08]   不允许越过行程详情页直接进入结账页（需 tripId 参数）
[B7]     所有金额展示使用 (value ?? 0).toFixed(2)，防 NaN
Web路由:  /checkout?tripId=xxx
Mobile:   /checkout/[tripId]
```
