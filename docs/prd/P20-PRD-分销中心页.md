## 一、竞品对标分析

### 1.1 竞品页面对标

| 竞品 | 页面名称 | 核心设计 | 参考要素 |
|------|---------|---------|---------|
| 花生日记 | 推广中心 | 邀请码大图+团队分级展示+收益统计 | 整体布局参考 |
| 云集 | 分销中心 | 数据看板+下线列表+收益明细 | 数据展示方式 |

- **Trip.com**: 旅行场景话术，"带好友一起朝圣，共同获得积分"更有感召力

## 二、页面定位

### 2.1 目标用户

- 已有积分想了解分销功能的老用户
- 通过好友邀请了解到分销功能的新用户
- 已有下线希望查看收益的活跃分销者

### 2.2 页面目标

1. 让用户快速获取并分享邀请码/链接/海报
2. 清晰展示分销成果(团队规模+累计收益)
3. 查看团队成员和返佣明细
4. 激励用户持续邀请(可视化进度+奖励感知)

### 2.3 三问测试

- Q1 目标用户: 有分享意愿或已有下线的朝圣旅行用户
- Q2 帮做什么决策: 决定是否分享邀请码，查看分销收益
- Q3 去掉会变差: 会导致分销功能无法使用，用户无法获得分销返佣

## 六、API调用规划

### 6.1 页面初始化 (并行请求)

```
进入页面时并行调用:
1. GET /api/referral/my-code       → 邀请码信息
2. GET /api/referral/stats         → 数据统计(含趋势图数据)
3. GET /api/referral/my-team?level=1&page=1&pageSize=20  → 一级下线(默认Tab)
4. GET /api/referral/my-rewards?page=1&pageSize=10       → 返佣明细(前10条)

用户积分余额: 从全局用户状态获取，无需额外请求
```

### 6.2 交互触发请求

```
切换团队Tab(二级) → GET /api/referral/my-team?level=2&page=1&pageSize=20
团队列表上拉      → GET /api/referral/my-team?level={active}&page={next}&pageSize=20
明细列表上拉      → GET /api/referral/my-rewards?page={next}&pageSize=20
筛选变更          → GET /api/referral/my-rewards?status={s}&level={l}&page=1
下拉刷新          → 重新发起所有初始化请求
```

## 七、数据展示规范

### 7.1 数字格式

| 数据 | 格式 | 示例 |

```

```
PENDING   → "待结算" 橙色标签

## 九、技术约束

### 9.1 移动端 (Expo React Native)

- 海报生成: 使用 `react-native-view-shot` 截图Canvas组件
- 分享: 使用 `expo-sharing` + `expo-file-system`
- 趋势图: 使用 `victory-native` 或 `react-native-gifted-charts`
- 剪贴板: 使用 `expo-clipboard`
- 图表动画: 进入页面时趋势折线有绘制动画(1s)

### 9.2 小程序 (Taro)

- 海报生成: 使用 Canvas 2D API 绘制
- 分享: 使用 wx.shareAppMessage + wx.shareTimeline
- 趋势图: 使用 echarts-for-weapp 或 wx-charts

### 9.3 Web端 (Next.js)

- 海报生成: 使用 `html2canvas` 渲染DOM为图片
- 分享: Web Share API + 降级为复制链接弹窗
- 趋势图: 使用 `recharts`

### 9.4 性能约束

- 页面初始化时间 < 1s (4个并行请求)
- 趋势图渲染时间 < 500ms
- 海报生成时间 < 2s，超时显示错误提示
- 团队列表支持虚拟滚动(超过50条时)
