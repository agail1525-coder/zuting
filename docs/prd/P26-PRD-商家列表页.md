## 二、用户故事

### 2.1 核心用户故事

```
作为一名计划朝圣九华山的用户，
我希望能浏览九华山附近的导游和住宿商家，
通过评分和评价选择靠谱的服务提供者。

验收标准:
  - 商家列表默认展示全部ACTIVE商家
  - 可按类型(寺庙/导游/住宿/交通)筛选
  - 可按关键词搜索商家名称
  - 可按评分/订单量/距离排序
  - 每个商家卡片显示: 名称/类型/评分/订单数/简介/LOGO
```

```
作为一名想找佛教文化导游的用户，
我希望能查看导游的详细信息、服务列表和历史评价，
以便做出预订决策。

验收标准:
  - 点击商家卡片进入详情页
  - 详情页展示: 商家介绍/服务列表/评价列表/位置地图
  - 服务列表显示: 名称/价格/容量/图片
  - 评价区域显示: 评分分布/评价列表/评价图片
  - 有明确的"预订"或"联系商家"按钮
```

```
作为一名使用手机的用户，
我希望商家列表在移动端也能流畅浏览，
支持下拉刷新和无限滚动加载。

验收标准:
  - 移动端商家卡片纵向排列，图片占比更大
  - 支持下拉刷新(Pull to Refresh)
  - 触底自动加载下一页(Infinite Scroll)
  - 筛选器折叠为底部Sheet
```

## 四、路由设计

### 4.1 Web端路由

| 路由 | 页面 | 说明 |
|------|------|------|
| /merchants | 商家列表 | 支持query参数: ?type=GUIDE&keyword=&sortBy=rating |
| /merchants/:id | 商家详情 | 展示商家信息+服务+评价 |

### 4.2 Mobile端路由

| 路由 | 页面 |
|------|------|
| /merchants | 商家列表(Tab筛选) |
| /merchants/[id] | 商家详情 |

### 4.3 小程序路由

| 路由 | 页面 |
|------|------|
| /pages/merchants/index | 商家列表 |
| /pages/merchants/detail | 商家详情(?id=xxx) |

## 五、API依赖

| 功能 | API端点 | 说明 |
|------|--------|------|
| 商家列表 | GET /api/merchants?type=&keyword=&sortBy=&page=&pageSize= | 仅ACTIVE商家 |
| 商家详情 | GET /api/merchants/:id | 含服务列表+评价统计 |
| 商家服务 | GET /api/merchant-services?merchantId=xxx&isActive=true | 上架中的服务 |
| 商家评价 | GET /api/reviews?merchantId=xxx&page=&pageSize= | 分页评价列表 |

## 六、组件设计

### 6.1 核心组件

| 组件 | 说明 | Props |
|------|------|-------|
| MerchantList | 商家列表容器 | filters, sortBy |
| MerchantCard | 商家卡片(列表项) | merchant, onClick |
| MerchantTypeFilter | 类型筛选Tab | activeType, onChange |
| MerchantSortSelect | 排序下拉 | value, onChange |
| MerchantDetail | 商家详情页容器 | merchantId |
| MerchantHeader | 详情页头部(LOGO+名称+评分) | merchant |
| MerchantServiceList | 服务列表 | services, onBook |
| MerchantServiceCard | 服务卡片 | service, onBook |
| MerchantReviewSection | 评价区域 | merchantId |
| MerchantLocationMap | 位置地图(Leaflet) | lat, lng, name |
| RatingDistribution | 评分分布柱状图 | distribution |

### 6.2 状态管理

```typescript
// React Query hooks
useMerchants(filters)        // 商家列表 + 分页
useMerchantDetail(id)        // 商家详情
useMerchantServices(id)      // 商家服务列表
useMerchantReviews(id)       // 商家评价列表
```

### 6.3 共享组件复用

| 已有组件 | 复用场景 |
|---------|---------|
| StarRating | 评分展示 |
| ReviewCard | 评价卡片 |
| MapView (Leaflet) | 商家位置地图 |
| Pagination | 分页控件 |
| EmptyState | 空状态(无商家/无评价) |
| LoadingSkeleton | 骨架屏 |

## 七、SEO与性能

| 维度 | 策略 |
|------|------|
| SSR | 商家列表和详情页使用Next.js SSR，利于搜索引擎收录 |
| Meta | 动态生成title/description: "{商家名} - JOINUS.COM合作伙伴" |
| 图片 | LOGO使用next/image懒加载+WebP格式，服务图片使用缩略图 |
| 分页 | 首屏20条，滚动加载(Web)/无限滚动(Mobile) |
| 缓存 | 商家列表staleTime: 5min，详情staleTime: 2min |

## 八、验收标准

| # | 验收项 | 标准 |
|---|--------|------|
| 1 | 列表加载 | 首屏 < 1.5s，仅展示ACTIVE商家 |
| 2 | 类型筛选 | 点击类型Tab即时过滤，URL同步query参数 |
| 3 | 关键词搜索 | 输入防抖300ms，搜索名称+描述 |
| 4 | 排序 | 支持评分/订单量/最新排序 |
| 5 | 商家详情 | 展示完整信息: 介绍/服务/地图/评价 |
| 6 | 服务预订 | 点击"立即预订"跳转到对应订单创建流程 |
| 7 | 评价展示 | 显示评分分布+评价列表+商家回复 |
| 8 | 地图 | 详情页展示商家位置(Leaflet暗色瓦片) |
| 9 | 空状态 | 无结果时展示引导文案("暂无该类型商家") |
| 10 | 响应式 | Web宽屏卡片横排 / Mobile纵向卡片 / 小程序适配 |
| 11 | 三端一致 | Web/Mobile/小程序功能和数据一致 |
| 12 | SEO | 列表和详情页SSR渲染，有正确的meta信息 |
