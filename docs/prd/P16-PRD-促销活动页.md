## 1. 页面定位

促销活动页是平台促销活动的集中展示大厅，对标 Priceline / Agoda 的 Deals 页面。通过视觉冲击和稀缺感设计，最大化用户的点击和转化行为。不同于优惠券中心（管理个人权益），本页面聚焦**公开促销活动的发现与转化**。

### 1.1 竞品对标

| 竞品 | 页面核心设计 | 借鉴点 |
|------|------------|--------|
| Priceline Express Deals | 倒计时 + 盲订高折扣 + 稀缺感强烈 | 倒计时组件设计、价格锚点、折扣百分比显眼 |
| Agoda Flash Sale | 已售百分比进度条 + 每日闪购时间表 | 社会证明（已售X%）、多时段安排 |
| Booking.com Deals | 红色折扣标签 + 条件筛选 | 筛选器设计、折扣标签视觉规范 |
| 飞猪特卖 | 大图卡片 + 限量角标 + 价格对比 | 国内用户熟悉的视觉风格 |
| 携程特惠 | Banner轮播 + 按类型分Tab | Tab分类结构参考 |

## 2. 用户故事

```
US-01: 作为用户，我希望在活动页快速看到当前所有促销活动，判断是否值得出手
US-02: 作为用户，闪购活动中我希望看到倒计时和剩余数量，感受紧迫感
US-03: 作为用户，我希望看到原价和折后价的对比，理解折扣力度
US-04: 作为早鸟用户，我希望知道提前多少天订能享受早鸟价，以及截止日期
US-05: 作为用户，我希望按类型筛选（闪购/限时/早鸟），快速找到感兴趣的活动
US-06: 作为用户，我希望点击活动卡片后跳转到对应行程详情页直接下单
```

## 5. API 契约

### 5.1 获取促销活动列表（前台）

```
GET /api/promotions/active
```

**Query Params:**
```
type?:       'TIME_LIMITED' | 'EARLY_BIRD' | 'FLASH_SALE'
entityType?: 'TRIP' | 'ROUTE'
page?:       number (default: 1)
pageSize?:   number (default: 12, max: 50)
sortBy?:     'endAt_asc' | 'discount_desc' | 'createdAt_desc'
```

**Response 200:**
```json
{
  "items": [
    {
      "id": "promo_xxx",
      "name": "五一朝圣闪购",
      "type": "FLASH_SALE",
      "discountType": "PERCENT",
      "discountValue": 20,
      "startAt": "2026-04-30T00:00:00Z",
      "endAt": "2026-05-01T23:59:59Z",
      "totalQuota": 100,
      "usedQuota": 81,
      "remainingQuota": 19,
      "soldPercent": 81,
      "coverImage": "https://cdn.joinus.com/promotions/xxx.jpg",
      "isAvailable": true,
      "secondsLeft": 5025,      // 距结束秒数，前端用于倒计时
      "entities": [             // 绑定的行程摘要
        {
          "id": "trip_abc",
          "title": "丝绸之路朝圣7日游",
          "coverImage": "https://...",
          "originalPrice": 5980.00,
          "discountedPrice": 4784.00
        }
      ]
    }
  ],
  "total": 6,
  "page": 1,
  "pageSize": 12
}
```

### 5.2 获取单个促销活动详情

```
GET /api/promotions/:id
```

**Response 200:** 同上单项，附加 `description`, `minAmount`, `maxDiscount`

## 6. 倒计时组件规范

### 6.1 实现方式

```typescript
// 基于 secondsLeft 字段实现前端倒计时（服务端时间，避免客户端时区问题）
// 每秒 -1，归零时:
//   1. 触发 refetch 重新获取活动状态
//   2. 若服务端返回 isAvailable: false → 卡片变为"已结束"灰色状态

interface CountdownProps {
  secondsLeft: number;
  onExpired: () => void;  // 归零回调
}
```

### 6.2 展示格式

```
> 24小时: 显示 "还剩 X 天 X 小时"
≤ 24小时: 显示 "HH : MM : SS"（实时倒计时）
已结束:   显示 "活动已结束"（灰色）
```

## 7. 状态与交互

| 状态 | 卡片展示 |
|------|---------|
| 活动进行中 + 有余量 | 正常展示，倒计时运转 |
| 活动进行中 + 售罄 | 进度条满100% + "已售罄"标签 + 按钮禁用 |
| 活动已结束（endAt 过期）| 卡片置灰 + "活动已结束"覆盖层 |
| 活动未开始（startAt 未到）| "即将开始" + 预热倒计时（P2） |
| 网络加载中 | 骨架屏（Skeleton） |
| 无促销活动 | 空状态："暂无进行中的活动，敬请期待" |

## 8. 技术约束

```
Web路由:   /promotions 或 /deals
Mobile:    /(tabs)/explore/promotions 或 独立页

公开接口，无需登录（点击"立即抢购"时若未登录跳转登录页）

[R-64]   pageSize @Max(50)
[B8]     (Array.isArray(data?.items) ? data.items : []).map()
倒计时:  基于 secondsLeft，使用 setInterval(1000) 递减
         组件卸载时必须 clearInterval，防内存泄漏

图片懒加载: Web 用 next/image，Mobile 用 expo-image
骨架屏:    list loading 时展示 4-8 个骨架卡片
```

## 9. 验收标准

- [ ] 四 Tab（全部/闪购/限时/早鸟）切换正常，各 Tab 数据独立筛选
- [ ] 闪购卡片：倒计时每秒更新（格式 HH:MM:SS），归零后变为"已结束"
- [ ] 已售百分比进度条：数值正确（usedQuota/totalQuota*100），视觉对应
- [ ] 仅剩数量提示：remainingQuota < 20 时展示"仅剩 X 个"（橙色高亮）
- [ ] 原价划线展示，折后价大号加粗
- [ ] 折扣百分比徽章（如"8折"）展示正确（100 - discountValue）
- [ ] 限时折扣卡片：展示剩余天数（> 24h 显示天数，≤ 24h 显示时分秒）
- [ ] 早鸟价卡片：展示提前天数要求和截止日期
- [ ] 售罄状态：进度条满 + 按钮禁用 + 文案变"已售罄"
- [ ] 已结束活动：卡片置灰，不展示倒计时
- [ ] 点击卡片/按钮：正确跳转行程详情页，URL 携带 promotionId
- [ ] 分页/加载更多功能正常
- [ ] 骨架屏在数据加载时展示
- [ ] 移动端布局在各屏幕尺寸下正常，图片不变形
