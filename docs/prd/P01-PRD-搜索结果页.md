## 1. 背景与目标

### 1.1 背景

搜索结果页（路由：`/search?q={keyword}&type={type}&religion={slug}`）是用户完成搜索行为后的核心落地页。此页面需要在有限屏幕空间内高效呈现多类型实体结果，同时提供灵活的筛选和排序能力，帮助用户在搜索结果中快速定位目标内容。参考 Booking.com 双栏布局、Kayak 顶部筛选Tab、TripAdvisor 结果卡片设计进行设计。

### 1.2 目标

- [ ] 搜索结果页在 1.5s 内完成首屏渲染（LCP）
- [ ] 筛选操作后结果局部刷新（无整页跳转），响应 ≤ 300ms
- [ ] Web 端三列卡片网格，Mobile 端单列列表，布局无错位
- [ ] 空结果状态有引导行动，用户不会在空页面迷失

### 1.3 非目标（本期不做的）

- 不包含结果卡片内的收藏按钮（F03-PRD-收藏夹系统负责）
- 不包含结果卡片内的评分显示（等评价系统上线后填充）
- 不包含地图视图（F01-PRD-地图搜索负责）
- 不包含广告/推广内容插入

## 2. 竞品对标分析

### 2.1 竞品功能对比

| 竞品 | 结果页布局 | 筛选器设计 | 借鉴点 |
|------|----------|----------|-------|
| Booking.com | 左侧固定筛选栏 + 右侧结果列表 | 多维度可展开，实时更新结果数 | 筛选栏粘性定位；每个筛选选项显示命中数量 |
| Kayak | 顶部水平筛选Tab + 下方结果 | Tab切换类型，下拉选择更多筛选 | 横向筛选Tab（类型切换）；排序下拉置于右侧 |
| TripAdvisor | 全宽列表（单列） + 右侧地图 | 顶部筛选横条，点击展开菜单 | 结果卡片展示评分和评价数量；"查看地图"按钮悬浮 |
| Skyscanner | 卡片列表，顶部筛选行 | 单行筛选，重要筛选项直接展示 | 结果数量显示在标题旁；排序和筛选同行布局 |

### 2.2 差距分析

JOINUS.COM 当前各内容列表页（如圣地列表、祖庭列表）均为独立页面，没有跨类型统一结果页，用户从搜索到查看结果的路径被割裂。需要建立统一的搜索结果页，整合4类实体展示。

### 2.3 差异化优势

结果卡片按信仰体系用主题色区分（佛教橙/道教绿/伊斯兰金等），让用户在视觉上快速感知内容的宗教归属，这是旅游平台独有的信仰导向 UI 表达。

### 5.3 关键交互流程

```
1. 用户点击类型Tab
   → 本地更新激活Tab样式（立即）
   → 携带type参数重新请求 /api/search
   → 骨架屏替换当前列表（200ms内）
   → 新结果渲染

2. 用户勾选宗教筛选
   → Tag区域显示已选宗教
   → 防抖300ms后携带religion参数重请求
   → 结果更新，显示命中数更新

3. 用户点击结果卡片
   → 按类型路由跳转:
     holy-site → /holy-sites/{id}
     temple    → /temples/{id}
     patriarch → /patriarchs/{id}
     teaching  → /teachings/{id}
```

### 5.4 三态设计

| 状态 | 触发条件 | 表现 |
|------|---------|------|
| Loading | 搜索请求进行中 | 显示4个卡片骨架屏（灰色动画），筛选栏保持可见但半透明 |
| Empty | 查询无结果 | 居中显示"找不到内容"插图（暗色卡通人物望远镜），文字"没有找到与"XX"相关的内容"，下方显示"热门推荐"卡片3-4个 |
| Error | API请求失败 | 错误图标 + "搜索失败，请检查网络" + "重新搜索"按钮 |

## 6. API 契约

### 6.1 使用的 API 端点

| 端点 | 方法 | 用途 |
|------|------|------|
| `GET /api/search` | GET | 获取搜索结果（含筛选/排序/分页） |
| `GET /api/search/hot` | GET | 空结果时获取热门推荐 |

### 6.2 关键请求示例

```
GET /api/search?q=少林寺&type=holy-site&religion=buddhism&sort=relevance&page=1&pageSize=20
```

**响应字段说明（结果卡片需要的字段）:**

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 实体ID，用于构建详情页路由 |
| `type` | string | 实体类型：holy-site/temple/patriarch/teaching |
| `name` | string | 英文名称 |
| `nameZh` | string | 中文名称 |
| `religion.nameZh` | string | 所属信仰中文名 |
| `religion.color` | string | 信仰主题色（用于卡片左边框颜色） |
| `country` | string | 国家代码 |
| `description` | string | 简介摘要（截取前120字符） |
| `imageUrl` | string | 缩略图URL |
| `score` | number | 相关度评分（0-1，仅sort=relevance时有效） |

## 8. 技术约束

- **路由**: Next.js App Router，路径 `/search`，使用 `searchParams` 接收参数
- **状态管理**: URL参数作为筛选状态的唯一真实来源（URL-driven state），支持浏览器前进/后退
- **性能**: 筛选变更时使用 `useTransition` 避免UI卡顿；结果列表使用虚拟化（超过50条时）
- **移动端**: 375px宽度无横向滚动；类型Tab横向滚动（overflow-x: auto），不换行
- **类型安全**: 结果卡片使用 `SearchResultItem` 接口，禁止 any 类型（[R-01]）
- **兼容性**: Web 支持 Chrome 100+, Safari 15+；Mobile 支持 iOS 14+, Android 10+
