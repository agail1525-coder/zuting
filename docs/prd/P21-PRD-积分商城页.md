## 一、竞品对标分析

### 1.1 竞品页面对标

| 竞品 | 页面 | 核心设计亮点 | 参考要素 |
|------|------|-------------|---------|
| 京东积分商城 | mall.jd.com | 大图卡片+积分醒目+分类清晰+已售数 | 整体卡片设计 |
| 美团积分商城 | 积分兑换 | 每日限量标签+即将售罄提示+分类Tab | 稀缺感设计 |
| 星巴克 | 星奖赏 | 积星进度可视化+阶梯达成感 | 积分余额展示 |
| Booking Genius | Genius Rewards | 简洁权益卡片+一键兑换无摩擦 | 兑换流程 |
| 航司里程商城 | 积分兑换 | 里程估值显示+过期提醒 | 积分价值感知 |

### 1.2 竞品设计学习点

- **京东**: 商品卡片用图片说话，积分价用大红字，原价用灰色划线，视觉层次分明
- **美团**: "仅剩XX份"标签制造紧迫感，推动快速决策
- **星巴克**: 进度条可视化，让用户看到"再XX积分就能兑换"的目标
- **Booking**: 兑换流程极简，确认弹窗信息充分，无额外步骤
- **航司**: "积分≈¥XX"帮用户感知积分价值，减少决策摩擦

## 二、页面定位

### 2.1 目标用户

- 有积分余额希望消费的用户
- 好奇积分能换什么的新用户
- 会员用户寻找专属权益兑换

### 2.2 页面目标

1. 展示所有可兑换商品，帮助用户找到合适商品
2. 清晰告知用户自己的积分余额和兑换能力
3. 极简兑换流程，降低转化摩擦
4. 激励积分消费，带动平台互动和黏性

### 2.3 三问测试

- Q1 目标用户: 有积分余额的注册用户
- Q2 帮做什么决策: 决定用积分兑换哪个商品
- Q3 去掉会变差: 积分将失去消费场景，用户持有积分无处消费，降低积分价值感知

## 三、用户故事

```
作为一个有5000积分的用户，
进入积分商城后，我希望第一眼就知道我能兑换什么，
而不是看了一堆我兑换不起的东西。

验收标准:
  - 积分余额在页面顶部醒目展示
  - "可兑换"商品在视觉上有明显区分(按钮可点击，灰色=不可兑)
  - 默认展示"全部"分类，按积分价升序排列(先显示能兑的)
  - 积分不足的商品在卡片右上角显示"还差XX积分"
```

```
作为一个想兑换优惠券的用户，
我希望快速找到"优惠券"分类的商品，
不想在所有商品里翻找。

验收标准:
  - 分类Tab快速切换，点击即时过滤
  - 每个分类Tab显示该分类商品数量
  - 切换Tab时有平滑过渡动画
```

```
作为一个准备兑换的用户，
我希望兑换前看到清晰的确认信息，
包括扣多少积分、剩多少积分、得到什么。

验收标准:
  - 兑换确认弹窗包含: 商品图片/名称/积分费用/兑换后余额/权益说明
  - 确认按钮提交中显示Loading状态
  - 成功后Toast提示"兑换成功"，并展示跳转引导
  - 失败时明确提示失败原因
```

```
作为一个刚兑换了优惠券的用户，
我希望立即知道如何使用这张券。

验收标准:
  - 兑换成功弹窗/页面显示"去使用"按钮
  - 点击跳转到"我的优惠券"或直接进入行程选择
  - 兑换记录立即出现在"我的兑换"列表
```

## 五、商品卡片设计规范

### 5.1 商品卡片元素

```
┌──────────────────────────────────┐
│  [角标: 🔥热门 / ⚡限量 / 🆕新品]│
│                                  │
│     [商品封面图 16:9比例]        │
│     图片宽度100%，高度固定       │
│                                  │
│  商品名称(最多2行，超出省略号)   │
│                                  │
│  ┌──────────────────────────┐   │
│  │  XXXX 积分               │   │  ← 积分价(大红字, 20px+)
│  │  ~~原价¥XX~~             │   │  ← 原价划线(灰色, 12px)
│  └──────────────────────────┘   │
│                                  │
│  剩余 XXX 份 / 已售 XXX 件       │  ← 库存/销量(12px灰)
│                                  │
│  [立即兑换] / [积分不足] / [已售罄] / [已拥有]  │
└──────────────────────────────────┘
```

### 5.2 按钮状态设计

| 状态 | 按钮样式 | 说明 |
|------|---------|------|
| 可兑换 | 金色填充按钮 "立即兑换" | 用户积分充足+有库存 |
| 积分不足 | 灰色边框按钮 "积分不足" | 点击提示还差XX积分 |
| 已售罄 | 灰色填充 "已售罄" | 不可点击 |
| 已拥有 | 灰色填充 "已拥有" | 同类型已兑换(适用勋章) |
| 下架 | 不显示该商品 | |

### 5.3 角标类型

```
🔥 热门     — soldCount >= 500
⚡ 限量     — stock > 0 AND stock <= 50
🆕 新品     — createdAt 7天内
⏰ 即将结束 — endAt 24小时内
🎯 推荐     — 运营手动标记
```

### 5.4 库存展示规则

```
stock = -1 (无限库存): 不展示库存/已售信息
stock > 100          : 显示"已售 {soldCount} 件"
stock 51-100         : 显示"剩余 {stock} 份"
stock 11-50          : 显示"⚡ 仅剩 {stock} 份" 橙色
stock 1-10           : 显示"🔴 仅剩 {stock} 份" 红色闪烁
stock = 0            : 显示"已售罄"，按钮置灰
```

## 七、交互规范

### 7.1 页面交互流程

```
进入页面
  ↓
并行加载: 积分余额 + 商品列表(默认全部)
  ↓
用户点击分类Tab
  ↓
切换商品列表(加载动画)
  ↓
用户点击"立即兑换"
  ↓ (积分充足时)
展示兑换确认弹窗
  ↓
用户点击"确认兑换"
  ↓
Loading状态(禁用按钮防重复)
  ↓
成功: 展示兑换成功弹窗 + 更新积分余额 + 更新商品状态
失败: 显示错误Toast (积分不足/售罄/其他)
```

### 7.2 分类Tab交互

| 操作 | 行为 |
|------|------|
| 点击Tab | 即时切换商品列表，骨架屏Loading |
| 滑动Tab栏 | 横向滚动(分类多时) |
| Tab数量 | 最多7个，超过则横向滚动 |

### 7.3 弹窗交互

| 操作 | 行为 |
|------|------|
| 点击弹窗外区域 | 关闭弹窗(确认弹窗需确认) |
| 点击X | 关闭弹窗 |
| 确认兑换点击 | 禁用按钮，显示Loading |
| 成功 | 关闭确认弹窗，打开成功弹窗 |
| 失败 | 弹窗内显示错误信息 |

### 7.4 状态处理

| 状态 | 展示 |
|------|------|
| Loading | 商品卡片骨架屏(2列x3行) |
| Error | 居中错误提示+重试按钮 |
| 空数据(某分类) | 居中"该分类暂无商品" |
| 兑换Loading | 按钮转圈，禁止重复点击 |

## 八、API调用规划

### 8.1 页面初始化

```
并行调用:
1. GET /api/user/me → 获取积分余额(从全局状态，可能不需要额外请求)
2. GET /api/points-mall/products?page=1&pageSize=20 → 商品列表

商品列表响应中 canExchange 字段由后端根据用户积分计算
```

### 8.2 交互触发

```
切换分类Tab    → GET /api/points-mall/products?category={cat}&page=1&pageSize=20
商品列表上拉   → GET /api/points-mall/products?category={cat}&page={next}&pageSize=20
点击确认兑换   → POST /api/points-mall/exchange { productId }
查看我的兑换   → GET /api/points-mall/my-exchanges?page=1&pageSize=20
```

### 8.3 兑换成功后更新

```
POST /exchange 成功后:
1. 更新全局积分余额状态 (减去 pointsSpent)
2. 更新商品卡片库存显示 (soldCount+1, stock-1)
3. 若商品类型=BADGE, 标记该商品为"已拥有"
4. 展示成功弹窗
```

## 十一、技术约束

### 11.1 移动端 (Expo React Native)

- 商品列表: FlatList + numColumns=2 双列网格
- 下拉刷新: RefreshControl
- 上拉加载: onEndReached + onEndReachedThreshold
- 弹窗: Modal 组件 + 动画
- 积分数字: Animated.Value 做计数动画(余额更新时)
- 图片: expo-image + placeholder

### 11.2 小程序 (Taro)

- 商品列表: 使用 Grid 布局
- 分类: ScrollView 横向滚动
- 弹窗: Taro Modal 或自定义 View
- 图片: Taro Image + lazyLoad

### 11.3 Web端 (Next.js)

- 商品列表: CSS Grid，responsive columns
- 分类Tab: 滚动Tab组件
- 弹窗: Dialog/Modal组件(Headless UI 或 Radix)
- 积分数字动画: CSS transition + requestAnimationFrame

### 11.4 性能约束

- 商品列表首屏加载 < 1s
- 图片懒加载，placeholder占位
- 兑换弹窗打开动画 < 200ms
- 兑换接口响应后UI更新 < 300ms
- 商品列表支持虚拟滚动(超过100条时)

### 11.5 安全约束

- 兑换按钮防抖(500ms) + 提交中禁用
- 积分余额展示从服务端获取，不信任客户端传入
- 兑换成功后强制刷新余额，不仅靠本地计算

## 十二、验收标准

### 12.1 视觉验收

| 检查项 | 标准 |
|--------|------|
| 积分余额 | 金色大字，醒目可见 |
| 商品卡片 | 图片清晰，积分价红色醒目，原价灰色划线 |
| 可兑换按钮 | 金色填充，视觉突出 |
| 不可兑换按钮 | 灰色，不可误导点击 |
| 售罄状态 | 卡片有半透明遮罩或明显售罄标识 |
| 角标 | 位于卡片左上角，醒目但不遮挡主要信息 |
| 整体配色 | 深色背景+金色点缀，符合殿堂风格 |

### 12.2 功能验收

| 测试用例 | 预期结果 |
|---------|---------|
| 进入积分商城 | 1秒内显示积分余额和商品列表 |
| 切换分类Tab | 商品列表正确按分类过滤 |
| 积分充足兑换 | 确认弹窗显示正确，兑换成功后积分减少 |
