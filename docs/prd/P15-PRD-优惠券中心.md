## 1. 页面定位

优惠券中心是用户管理个人优惠权益的专属页面，同时承担平台促活与拉新功能。通过"可领取→领取→使用"完整闭环，提升用户在平台的活跃度和复购率。

### 1.1 竞品对标

| 竞品 | 优惠券体系亮点 | 借鉴点 |
|------|--------------|--------|
| Agoda 优惠中心 | 专属会员券 + 限时领取 + 紧迫感倒计时 | 限时领取机制、已用尽警告 |
| 美团优惠券 | 三 Tab 结构（可领/我的/已用），卡片设计清晰 | Tab 分类、卡片视觉规范 |
| 飞猪/携程 | 优惠券包 + 可叠加说明 + 适用范围标签 | 使用条件清晰展示 |
| Booking Genius | 会员专属折扣，不以"券"形式展示但逻辑相同 | 身份绑定折扣的设计思路 |

```
US-01: 作为新用户，我希望在注册后引导我领取新人优惠券，感受平台福利

US-07: 作为用户，我希望手动输入优惠码也能领取/兑换券
```

## 4. 数据模型

### 4.1 Coupon 表（优惠券模板）

```prisma
model Coupon {
  id            String       @id @default(cuid())
  name          String       // "新人专享券"
  description   String?
  code          String?      @unique  // 手动兑换码（可选）
  discountType  DiscountType // FIXED | PERCENT
  discountValue Decimal      // 满减金额 或 折扣百分比(0-100)
  minAmount     Decimal?     // 最低消费门槛
  maxDiscount   Decimal?     // PERCENT时最高折扣额
  entityType    EntityType?  // TRIP | ROUTE | null(全场通用)
  entityIds     String[]     // 适用实体ID（空=全场）
  totalQuota    Int          @default(0)  // 0=不限量
  claimedQuota  Int          @default(0)  // 已领取数量
  isPublic      Boolean      @default(true)  // 是否在优惠券中心展示
  validDays     Int?         // 领取后有效天数（与 expiredAt 二选一）
  expiredAt     DateTime?    // 固定过期时间（与 validDays 二选一）
  isActive      Boolean      @default(true)
  createdAt     DateTime     @default(now())
  updatedAt     DateTime     @updatedAt

  userCoupons   UserCoupon[]

  @@index([isPublic, isActive])
}
```

### 4.2 UserCoupon 表（用户已领优惠券）

```prisma
model UserCoupon {
  id          String           @id @default(cuid())
  couponId    String
  userId      String
  status      UserCouponStatus @default(AVAILABLE)
  claimedAt   DateTime         @default(now())
  usedAt      DateTime?
  expiredAt   DateTime         // 实际过期时间 = max(Coupon.expiredAt, claimedAt + validDays)
  orderId     String?          // 使用时关联的订单

  coupon      Coupon           @relation(fields: [couponId], references: [id])

  @@index([userId, status])
  @@unique([couponId, userId])  // 同一优惠券每个用户只能领一次（可改为允许多次）
}

enum UserCouponStatus {
  AVAILABLE  // 未使用
  USED       // 已使用
  EXPIRED    // 已过期
}
```

## 5. API 契约

### 5.1 获取可领取优惠券列表

```
GET /api/coupons/available
```

**说明:** 公开接口，无需登录；但登录后会标记"已领取"状态

**Query Params:**
```
page?:     number (default: 1)
pageSize?: number (default: 20, max: 50)
```

**Response 200:**
```json
{
  "items": [
    {
      "id": "coupon_xxx",
      "name": "新人专享100元券",
      "description": "新注册用户专属，全场行程可用",
      "discountType": "FIXED",
      "discountValue": 100,
      "minAmount": 1000,
      "maxDiscount": null,
      "entityType": null,
      "expiredAt": "2026-06-30T23:59:59Z",
      "remainingQuota": 500,
      "isClaimed": false,    // 当前用户是否已领取（需登录）
      "claimCondition": "新注册用户"  // 领取条件说明
    }
  ],
  "total": 8,
  "page": 1,
  "pageSize": 20
}
```

### 5.2 领取优惠券

```
POST /api/coupons/:id/claim
Authorization: Bearer {token}
```

**Response 201:**
```json
{
  "userCouponId": "uc_xxx",
  "couponId": "coupon_xxx",
  "expiredAt": "2026-06-30T23:59:59Z",
  "message": "领取成功，有效期至2026年6月30日"
}
```

**Response 409 (已领取):**
```json
{
  "statusCode": 409,
  "message": "ALREADY_CLAIMED",
  "detail": "您已领取过该优惠券"
}
```

**Response 409 (已售罄):**
```json
{
  "statusCode": 409,
  "message": "QUOTA_EXHAUSTED",
  "detail": "该优惠券已被领完"
}
```

### 5.3 兑换码领取

```
POST /api/coupons/redeem
Authorization: Bearer {token}
```

**Request Body:**
```json
{ "code": "WELCOME2026" }
```

**Response 201:** 同 5.2 成功响应

**Response 404:**
```json
{ "statusCode": 404, "message": "INVALID_CODE", "detail": "优惠码无效或已过期" }
```

### 5.4 获取我的优惠券

```
GET /api/coupons/my
Authorization: Bearer {token}
```

**Query Params:**
```
status?: 'AVAILABLE' | 'USED' | 'EXPIRED'
page?:   number
pageSize?: number (max: 50)
```

**Response 200:**
```json
{
  "items": [
    {
      "id": "uc_xxx",
      "coupon": {
        "id": "coupon_xxx",
        "name": "新人专享100元券",
        "discountType": "FIXED",
        "discountValue": 100,
        "minAmount": 1000,
        "entityType": null
      },
      "status": "AVAILABLE",
      "claimedAt": "2026-03-29T10:00:00Z",
      "expiredAt": "2026-06-30T23:59:59Z",
      "isExpiringSoon": true,   // 距过期 ≤ 3天
      "daysLeft": 2,
      "orderId": null
    }
  ],
  "total": 3,
  "page": 1,
  "pageSize": 20
}
```

## 7. 交互细节

### 7.1 领取按钮状态机

```
初始: [立即领取] — 蓝色按钮
领取中: [领取中...] — Loading
已领取: [已领取 ✓] — 绿色，禁用
已售罄: [已抢完] — 灰色，禁用
未登录点击: 跳转登录页，登录后自动回跳并继续领取
```

### 7.2 "去使用"跳转逻辑

```
点击[去使用 →]:
  → 跳转至行程列表页，URL 参数携带 couponId
  → 行程列表页展示 banner: "优惠券已选中，结账时自动应用"
  → 进入结账页时，自动预填该优惠券
```

### 7.3 过期提醒规则

```
expiredAt - now() <= 3天  → 橙色"即将过期(X天后)"标签
expiredAt - now() <= 1天  → 红色"今日到期"标签
expiredAt < now()         → 移入"已过期"分组（在"我的券"Tab 底部折叠展示）
```

## 8. 技术约束

```
Web路由:   /coupons
Mobile:    /(tabs)/profile/coupons 或 /coupons（独立页）

[R-63]   领取/兑换/查询我的券 必须 JwtAuthGuard
[R-64]   列表接口 pageSize @Max(50)
[R-68]   GET /api/coupons/my 只返回当前登录用户的优惠券

GET /api/coupons/available 为公开接口（无需登录），但登录态下会标记 isClaimed
防刷: 领取接口加 Rate Limiting，同一用户同一优惠券 @@unique 约束防重复领取

[B7]  金额展示: (value ?? 0).toFixed(2)
[B8]  列表: Array.isArray(data?.items) ? data.items : []
```

## 9. 验收标准

- [ ] 三 Tab 切换正常，各 Tab 数据独立请求，切换后有 loading 状态
- [ ] 可领取 Tab：展示公开优惠券，登录用户已领取的显示"已领取 ✓"
- [ ] 领取成功后当前按钮立即变为"已领取"，无需刷新页面
- [ ] 配额耗尽时领取，提示"已抢完"并置灰按钮
- [ ] 兑换码输入框：输入有效码 → 成功提示 + 移入"我的券"；无效码 → 错误提示
- [ ] 我的券 Tab：按过期时间升序排列，即将过期的在前
- [ ] 距过期 ≤ 3 天展示橙色标签，≤ 1 天展示红色标签
- [ ] 未达门槛的券展示"还差 ¥X 才能使用"提示，按钮变为"去凑单"
- [ ] 已使用 Tab：展示使用记录，含订单号和节省金额
- [ ] 各 Tab 空状态展示引导语和跳转按钮
- [ ] 未登录访问时，领取按钮跳转登录页，登录后回跳继续操作
- [ ] 移动端卡片布局完整，文字不溢出，按钮可点击区域足够大
