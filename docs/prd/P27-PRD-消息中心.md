# P27 消息中心页面 产品需求文档

> 版本: v1.0 | 日期: 2026-03-29 | 状态: Sprint E1 规划中 | 依赖: M30-PRD-实时消息系统
> 路由: `/messages` (Web) | `/(tabs)/messages` (Mobile) | `/pages/messages/index` (小程序)

---

## 1. 需求背景与目标

### 1.1 背景

消息中心是实时消息系统的核心前端页面，承载用户查看所有对话、收发消息、管理咨询的功能。参考 Airbnb Inbox 的左右分栏布局和 WhatsApp Web 的即时交互体验，为用户提供高效的沟通界面。

### 1.2 目标

| 目标 | 指标 | 说明 |
|------|------|------|
| 页面加载速度 | <1.5s | 首屏聊天室列表完全渲染 |
| 消息发送成功率 | >99% | 发送后确认送达 |
| 用户日均停留时长 | >3min | 消息中心页面停留时间 |

### 1.3 范围

| 范围 | v1 (Sprint E1) | v2 |
|------|----------------|-----|
| 聊天室列表 | 头像+名字+最后消息+未读数+时间 | 列表搜索+置顶+免打扰 |
| 聊天窗口 | 消息气泡+输入框+发送 | 表情面板+@提及+引用回复 |
| 消息类型 | TEXT+IMAGE+SYSTEM | 语音+视频+文件+位置 |
| 搜索 | 聊天室名称搜索 | 全文消息搜索 |
| 新建对话 | 选择用户创建 | 最近联系人+通讯录 |
| 响应式 | 桌面分栏/移动全屏切换 | — |

## 2. 用户故事

### US-27-01: 查看聊天列表

**作为** 用户，**我希望** 打开消息中心看到所有对话，**以便** 快速找到需要回复的消息。

**验收标准:**
- 聊天室按最后消息时间倒序排列
- 每个聊天室显示: 头像、名称、最后一条消息预览(截断30字)、时间、未读数Badge
- 未读消息的聊天室加粗显示
- MERCHANT_INQUIRY类型显示商家图标，GUIDE_CHAT显示导游图标
- 空状态: 显示"暂无消息，去探索圣地吧"引导文案+按钮

### US-27-02: 收发文字消息

**作为** 用户，**我希望** 在聊天窗口中发送和接收文字消息，**以便** 与商家/导游即时沟通。

**验收标准:**
- 我发送的消息显示在右侧(蓝色气泡)，对方消息在左侧(灰色气泡)
- 发送后消息即时出现(乐观更新)，发送失败显示红色感叹号+重试
- 新消息自动滚动到底部，查看历史消息时不自动滚动(显示"N条新消息"浮窗)
- 输入框支持Enter发送，Shift+Enter换行
- 系统消息居中显示，灰色背景圆角卡片

### US-27-03: 发送图片消息

**作为** 用户，**我希望** 在聊天中发送图片，**以便** 分享朝圣照片或截图给对方。

**验收标准:**
- 输入框左侧"+"按钮或图片图标触发图片选择
- 支持从相册选择或拍照 (Mobile/小程序)
- 图片发送前显示缩略图预览，确认后发送
- 聊天中显示缩略图，点击查看大图(全屏预览)
- 图片上传进度条

### US-27-04: 查看在线状态

**作为** 用户，**我希望** 看到对方是否在线，**以便** 决定是否等待即时回复。

**验收标准:**
- 聊天窗口顶部对方名称旁显示在线状态: 绿点(在线) / 灰点(离线) / 黄点(离开)
- 在线时显示"在线"文字，离线时显示"xx分钟前在线"
- 对方正在输入时，状态区域显示"正在输入..."动画(三个跳动圆点)

### US-27-05: 已读回执

**作为** 用户，**我希望** 知道对方是否已读我的消息，**以便** 了解沟通进度。

**验收标准:**
- 我发送的消息下方显示状态: "已发送"(单勾) → "已读"(双勾/蓝色)
- 进入聊天室时自动发送已读回执
- 聊天室列表的未读数实时更新

### US-27-06: 搜索聊天室

**作为** 用户，**我希望** 搜索聊天室，**以便** 快速找到特定商家或导游的对话。

**验收标准:**
- 列表顶部搜索框，输入关键字实时过滤
- 匹配聊天室名称和参与者名称
- 无匹配结果时显示"未找到相关对话"

### US-27-07: 新建对话

**作为** 用户，**我希望** 主动创建新对话，**以便** 向特定商家或导游发起咨询。

**验收标准:**
- 列表顶部"+"按钮打开新建对话浮层
- 选择对话类型: 商家咨询 / 导游咨询
- 商家咨询: 搜索并选择商家
- 导游咨询: 搜索并选择导游
- 若已存在活跃聊天室则直接跳转，不重复创建

## 5. 交互细节

### 5.1 消息发送流程 (乐观更新)

```
1. 用户点击发送 →
2. 消息立即渲染在列表中 (临时ID，status=sending)
3. WebSocket emit('send_message') →
4. 收到服务端 'message_received' 回执 →
5. 替换临时ID为服务端ID，status=sent
6. 若5秒内未收到回执 → status=failed，显示红色感叹号
7. 用户点击感叹号 → 重试发送
```

### 5.2 历史消息加载

```
1. 进入聊天室 → 加载最近30条消息
2. 用户向上滚动到顶部 → 触发加载更多 (cursor分页)
3. 加载中显示顶部spinner
4. 加载完成后保持滚动位置不跳动
5. 无更多消息时显示"已加载全部消息"
```

### 5.3 新消息通知

```
1. 当前聊天室收到新消息 → 自动追加到底部，自动滚动
2. 正在查看历史消息(距底部>200px) → 不自动滚动，显示"N条新消息"浮窗
3. 点击浮窗 → 滚动到最新消息
4. 非当前聊天室收到新消息 → 更新列表未读数+最后消息
5. 页面不在前台 → 更新 document.title 显示未读数 "[3] 消息中心"
```

### 5.4 响应式切换

```
桌面端 (>= 768px):
- 左右分栏同时显示
- 左侧宽300px固定，右侧自适应
- 未选择聊天室时右侧显示插画占位

移动端 (< 768px):
- 默认显示聊天室列表 (全屏)
- 点击聊天室 → 全屏进入聊天窗口 (push路由或状态切换)
- 左上角返回按钮 → 回到列表
- 使用 React状态 或 路由参数 控制视图切换
```


```
- 最近10个聊天室的最新100条消息缓存在React Query中
```

## 8. API调用映射

| 用户操作 | API/事件 | 说明 |
|----------|----------|------|
| 打开消息中心 | `GET /api/chat/rooms` | 加载聊天室列表 |
| 点击聊天室 | `GET /api/chat/rooms/:id/messages` + `ws:join_room` | 加载消息+订阅实时推送 |
| 发送消息 | `ws:send_message` | WebSocket发送 |
| 上拉加载历史 | `GET /api/chat/rooms/:id/messages?cursor=xxx` | REST分页 |
| 进入聊天室 | `POST /api/chat/rooms/:id/read` + `ws:read_receipt` | 标记已读 |
| 输入中 | `ws:typing` | 实时通知对方 |
| 新建对话 | `POST /api/chat/rooms` | REST创建 |
| 搜索聊天室 | 前端本地过滤 | 不走API |

## 9. 错误处理

| 场景 | 处理方式 |
|------|---------|
| WebSocket连接失败 | 显示顶部Banner"连接断开，正在重连..."，自动重试(指数退避) |
| 消息发送失败 | 消息气泡显示红色感叹号，点击重试 |
| 聊天室列表加载失败 | 显示错误提示+重试按钮 |
| 历史消息加载失败 | 顶部显示"加载失败，点击重试" |
| Token过期 | WebSocket断开，跳转登录页 |
| 对方已离开聊天室 | 显示"对方已离开对话"，输入框可用但标注 |
