## 1. 背景与目标

### 1.1 背景

JOINUS.COM 已有 M10 地图与导航模块（基于 Leaflet + CartoDB 暗色瓦片），展示全球 60 处圣地的地理分布。然而现有地图为静态展示，无法与搜索结合——用户搜索"日本神社"后，只能看到文字列表，无法在地图上看到这些神社的地理关系。地图搜索将搜索结果叠加在交互地图上，帮助朝圣者从地理维度规划行程，对标 Airbnb 的地图搜索（最受用户喜爱的功能之一）和 Booking.com 的地图模式。

### 1.2 目标

- [ ] 搜索结果页提供"地图视图"切换，在地图上显示结果标记点
- [ ] 拖动地图或缩放后，自动搜索当前可视范围内的内容
- [ ] 点击地图标记弹出信息卡片，可直接跳转详情页
- [ ] 地图渲染时间（首次加载）< 2s，标记点渲染 < 500ms
- [ ] 基于现有 Leaflet 集成，不引入新地图库

### 1.3 非目标（本期不做的）

- 不包含路线规划（两点之间的导航，Phase D 地图增强）
- 不包含实时位置追踪（朝圣中的导航功能）
- 不包含离线地图缓存（Phase D PWA 离线支持）
- 不包含 3D 地图视角
- 不包含用户自定义标记点

## 2. 竞品对标分析

### 2.1 竞品功能对比

| 竞品 | 地图搜索特点 | 优点 | 缺点 | 借鉴点 |
|------|-----------|------|------|-------|
| Airbnb | 地图占屏幕50%+列表并排，拖动地图自动搜索可视区域，标记显示价格泡泡 | 地图与列表实时联动，拖动即搜索；标记点悬停展示预览卡 | 标记点密集时重叠严重 | 拖动地图触发搜索；悬停弹出预览卡；列表与地图联动高亮 |
| Booking.com | 结果页右侧有地图，标记点按价格显示，点击展示酒店简介弹窗 | 弹窗设计简洁；聚合处理好；支持从地图标记直接预订 | 地图尺寸偏小 | 标记点点击展示Popup卡片；Popup含图片+名称+行动按钮 |
| TripAdvisor | 地图模式切换，标记用数字编号对应列表顺序 | 地图与列表数字编号联动，容易对应 | 无聚合，密集时难点击 | 列表-地图双向联动高亮（hover列表项在地图高亮标记） |
| Google地图 | 搜索结果在地图显示，自动聚合，视口变化自动刷新 | 聚合算法成熟；视口搜索即时 | 移动端点击精准度低（标记太密） | 缩放时动态聚合/展开；可视区域变化触发搜索 |

### 2.2 差距分析

现有 JOINUS.COM 地图（/map 路由）只展示全部60个圣地，无法按搜索词/信仰类型过滤，也无法从搜索结果页切换到地图视图。用户需要规划朝圣路线时，必须在搜索结果页和地图页之间来回切换，体验割裂。

### 2.3 差异化优势

地图标记按信仰体系使用不同颜色和图标（如佛教用莲花图标+橙色，伊斯兰用星月图标+绿色），让信仰地理分布在地图上一目了然，这是通用旅行平台不具备的宗教文化地理可视化。

## 4. 功能清单

### 4.1 核心功能（P0 — 必须有）

- [ ] 地图/列表切换按钮：搜索结果页顶部右侧，"列表视图 / 地图视图"切换
- [ ] 搜索结果标记点：在地图上显示当前搜索结果（含筛选条件）的全部实体标记
- [ ] 标记点颜色：按信仰主题色区分（沿用 packages/config 中的信仰颜色配置）
- [ ] 标记点点击 Popup：显示实体名称、类型标签、信仰、简短描述、"查看详情"按钮
- [ ] Popup"查看详情"跳转：按实体类型路由到对应详情页
- [ ] 地图底图：沿用现有 CartoDB 暗色瓦片（Leaflet 配置复用 M10）
- [ ] 加载态：标记点加载时 Popup 位置显示 Spinner

### 4.2 增强功能（P1 — 应该有）

- [ ] 可视区域搜索：地图拖动/缩放停止后（1s防抖），自动请求 `/api/search/map?bounds=...`
- [ ] 标记点聚合：缩放级别 ≤ 8 时聚合为数量气泡（使用 Leaflet.markercluster 或自实现）
- [ ] 列表-地图联动：在列表视图时悬停卡片，对应地图标记高亮；地图Popup打开时列表对应卡片滚动到可见区
- [ ] 筛选同步：切换地图视图时，当前筛选条件（宗教/类型）继续生效，地图只显示过滤后结果
- [ ] 实体类型图标：圣地用圆形标记，祖庭用六边形标记（用SVG自定义Leaflet DivIcon）

### 4.3 优化功能（P2 — 可以有）

- [ ] 地图搜索专用URL：`/search/map?q=佛教&bounds=...`，支持分享当前地图视角
- [ ] 标记点数量上限提示：超过100个标记时显示"当前视野内结果过多，请缩放或筛选"
- [ ] 我的位置：地图右下角"定位到我的位置"按钮（需浏览器定位授权）
- [ ] 地图视角记忆：离开地图视图再回来时，恢复上次的地图中心点和缩放级别

### 5.3 关键交互流程

```
1. 用户点击"地图视图"按钮
   → 地图容器渲染，底图瓦片加载
   → 请求 /api/search/map?q={q}&type={type}&religion={religion}&bounds={全球范围}
   → 按信仰颜色渲染标记点

2. 用户拖动地图到新区域（1s防抖后）
   → 获取新的可视范围bounds
   → 请求 /api/search/map?bounds={sw_lat},{sw_lng},{ne_lat},{ne_lng}&{当前筛选}
   → 地图标记点更新（平滑过渡）

3. 用户点击某个标记点
   → 弹出 Popup 卡片（含图片/名称/简介/按钮）
   → 若列表视图可见（PC双栏），对应卡片滚动到可见区并高亮

4. 用户点击 Popup 中"查看详情"
   → 跳转到对应详情页（和列表视图卡片点击行为一致）
```

### 5.4 三态设计

| 状态 | 触发条件 | 表现 |
|------|---------|------|
| Loading | 地图标记点数据加载中 | 地图中心显示旋转 Spinner，地图底图正常渲染（底图与标记点异步） |
| Empty | 当前可视区域无搜索结果 | 地图中央显示浮层提示"当前区域暂无内容，试试移动地图或放大视野" |
| Error | 地图API请求失败 | 地图顶部Banner："标记点加载失败" + "重试" 按钮，地图底图保持可用 |

## 6. API 契约

### 6.1 新增 API 端点

#### `GET /api/search/map`

地图搜索接口，返回可视区域内实体的标记点数据（轻量，只含位置和基本信息）。

**请求参数:**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `q` | string | 否 | 搜索关键词（可选，不传时按筛选条件搜全部） |
| `swLat` | number | 是 | 可视区域西南角纬度 |
| `swLng` | number | 是 | 可视区域西南角经度 |
| `neLat` | number | 是 | 可视区域东北角纬度 |
| `neLng` | number | 是 | 可视区域东北角经度 |
| `type` | string | 否 | 实体类型：all/holy-site/temple（祖师/祖训无坐标，排除） |
| `religion` | string | 否 | 信仰slug，多值逗号分隔 |
| `limit` | number | 否 | 最大返回数量，默认100，最大200 |

**响应格式:**

```json
{
  "markers": [
    {
      "id": "clxxx",
      "type": "holy-site",
      "name": "少林寺",
      "nameEn": "Shaolin Temple",
      "lat": 34.5081,
      "lng": 112.9329,
      "religion": "佛教",
      "religionColor": "#FF6B35",
      "imageUrl": "https://cdn.joinus.com/holy-sites/shaolin-thumb.jpg",
      "description": "禅宗祖庭，中国武术圣地..."
    }
  ],
  "total": 8,
  "truncated": false
}
```

**字段说明:**

| 字段 | 说明 |
|------|------|
| `truncated` | 若结果超过limit，值为true，提示客户端显示"缩放以查看更多" |

**错误码:**

| HTTP状态码 | 错误码 | 说明 |
|------------|-------|------|
| 400 | INVALID_BOUNDS | bounds参数范围非法（如swLat > neLat） |
| 400 | BOUNDS_TOO_LARGE | 可视区域过大（超过全球经纬度范围1.5倍） |

## 7. 数据模型

### 7.1 涉及的现有 Prisma 模型

地图搜索使用 HolySite 和 Temple 表的 `lat` 和 `lng` 字段（现有字段）进行地理范围查询。

**查询逻辑（Prisma Where 条件）:**

```typescript
// 可视区域内查询（PostgreSQL 支持）
where: {
  lat: { gte: swLat, lte: neLat },
  lng: { gte: swLng, lte: neLng },
  // 叠加其他筛选条件...
}
```

### 7.2 性能优化索引

```sql
-- 为地图搜索添加坐标联合索引
CREATE INDEX IF NOT EXISTS idx_holysite_coords ON "HolySite" (lat, lng);
CREATE INDEX IF NOT EXISTS idx_temple_coords ON "Temple" (lat, lng);
```

## 8. 技术约束

- **地图库**: 沿用已有 Leaflet（`react-leaflet`），不引入 Mapbox/Google Maps（有费用）
- **底图**: CartoDB暗色瓦片（`https://{s}.basemaps.cartocdn.com/dark_all/{z}/{x}/{y}{r}.png`）
- **标记点**: 使用 Leaflet `DivIcon`（SVG自绘），支持信仰颜色和类型图标
- **聚合**: 使用 `react-leaflet-cluster` 库（如已有）或自实现简单聚合
- **性能**: 地图视口内最多渲染100个标记点；超出时显示截断提示；使用 `useMemo` 缓存标记点列表
- **Mobile适配**: 地图组件在 React Native 中使用 `react-native-maps`（与Web Leaflet分开实现），底图使用相同CartoDB URL
- **SSR**: Next.js 中 Leaflet 必须动态加载（`dynamic(import, { ssr: false })`），避免 SSR hydration 错误
- **类型安全**: 标记点使用 `MapMarker` 接口，禁止 any 类型
