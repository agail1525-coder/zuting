## 关键经验

### 1. Sentry集成 — 避免构建时依赖
**问题**: D6写的Sentry集成使用`import('@sentry/react')`动态导入，但:
- TypeScript编译时会检查类型 → 找不到模块报错
- Vite/Rollup构建时会解析动态import → 找不到模块报错
- Next.js webpack也会解析 → 同样报错

**解决**: 必须完全避免在代码中出现包名字符串：
```typescript
// 错误 — Vite/webpack都会解析
import('@sentry/react')

// 正确 — 运行时构建字符串，绕过静态分析
const pkg = ['@sentry', 'react'].join('/');
const loader = new Function('m', 'return import(m)');
loader(pkg).then(...)
```

**铁律**: 可选依赖(optional peer dependency)必须用运行时字符串拼接+new Function绕过打包器。

### 2. 并行Agent效率
- 6路Agent并行，总节省时间约70%
- D2(邮件短信)两次连接断开(ECONNRESET)，但D3(OAuth)附带完成了D2的全部工作

### 3. 支付网关架构
- Mock模式自动降级: ENV变量未设置时自动进入mock模式
- 每个网关独立可启用: 可以只开Stripe做国际版
- Webhook签名验证是安全底线: WeChat(HMAC-SHA256), Alipay(RSA2), Stripe(HMAC)

### 4. OAuth无Passport.js
- 直接用fetch实现OAuth比引入passport.js更轻量
- 前端通过`/auth/providers` API做feature flag，未配置的OAuth自动隐藏

### 5. 搜索实现
- Prisma `contains` + `mode: 'insensitive'` 在PostgreSQL上走全表扫描
- 当前数据量(196条)无需全文索引，到1万条时考虑pg_trgm或Elasticsearch
