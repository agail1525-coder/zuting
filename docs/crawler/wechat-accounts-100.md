## 投喂 SOP

```bash
# 1. 微信里搜公众号,点进历史消息
# 2. 找到代表作(最好带游记/深度/标题醒目),点右上 ··· → 复制链接
# 3. 粘贴到 scripts/crawler/feeds/wechat/urls.txt,每行一条
# 4. 提交
git add scripts/crawler/feeds/wechat/urls.txt
git commit -m "chore(crawler): 微信公众号投喂批次 N(X 条)"
git push

# 5. 生产立即生效: SFTP 上传 urls.txt (deploy 脚本下次全量会自动拉)
#    手动追加快速方式: scp urls.txt root@120.24.31.151:/opt/zuting/scripts/crawler/feeds/wechat/
#    爬虫每天 13:00 CRON 自动抓,或 Admin 页手动触发 wechat-mp-urls-feed 运行
```

## 质量控制

- **优先级 S** (账号 1-40): 官方寺院/协会/旅游局 — 抓取必留,合规最稳
- **优先级 A** (41-80): 学术/深度媒体 — 内容质量高,持续产出
- **优先级 B** (81-100): 小众/在地视角 — 差异化内容,酌情投喂
- 单个账号最多投喂 20 篇/月,避免数据倾斜
- 作者脱敏 (CW-36): 适配器已落地,昵称 sha256 哈希
- 文章里遇到疑似广告/卖课/引流微信号 → urls.txt 打 `#` 注释掉
