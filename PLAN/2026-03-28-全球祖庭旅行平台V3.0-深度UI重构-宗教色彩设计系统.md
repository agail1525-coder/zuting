# ZUTING V3.0 Deep UI Redesign Plan

## Context

V2.0 completed the platform restructuring from religious app to cultural travel platform (4 waves, committed). But the current UI has critical problems:

1. **Images missing everywhere** — HolySite/Temple/Patriarch cards show emoji placeholders, seed data has no imageUrl populated, `OptimizedImage` component exists but is imported by ZERO cards
2. **Light/Dark theme schizophrenia** — Homepage uses white `bg-[#FAFAF8]`, but Header/Footer/HolySites/Map are dark theme. Feels like two different apps
3. **Color system fragmented** — Religion colors defined in 4 separate places with conflicting values (seed.ts, religions.ts config, ReligionCard.tsx hardcoded map, home-client.tsx CATEGORY_COLORS)
4. **Missing professional travel features** — No image gallery, no reviews on detail pages, no booking UX, no nearby attractions, detail pages are skeleton-level (holy-site detail is 75 lines with emoji hero)

Reference: CEO provided Ctrip screenshots showing professional travel platform patterns (image galleries, reviews with photos, map integration, AI trip planner).
