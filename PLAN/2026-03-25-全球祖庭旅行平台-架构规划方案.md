# 全球祖庭旅行平台 — 架构规划方案

## Context

**现状**: E:\ZUTING\ 目前是一个 Python/tkinter 桌面修行助手（3,892行代码），拥有丰富的宗教文化内容（12大信仰、60圣地、36祖庭、40+祖师、30印修炼体系、32音频、158张图片）。

**目标**: 将其升级为类似作业郎规模的全平台应用（Web + APP + 微信小程序 + 管理后台），帮助100万人走祖庭，建立全球宗教文化和平使者网络。

**参考**: 作业郎项目（React Native + NestJS + Next.js + PostgreSQL 全栈架构）+ 2026年全网最佳实践调研。

---

## 技术栈选型

| 层级 | 技术 | 理由 |
|------|------|------|
| **Monorepo** | Turborepo + pnpm | 增量构建快100x，磁盘效率最高 |
| **Web前端** | Next.js 15 + React 19 + TailwindCSS | SSR/SSG、SEO友好、Vercel原生部署 |
| **移动端** | React Native + Expo 54 | 一套代码 iOS/Android，EAS云构建 |
| **微信小程序** | Taro 4.x + React | 支持微信/支付宝/抖音等多平台小程序 |
| **管理后台** | Vite + React + Ant Design | 轻量快速，内部使用 |
| **后端API** | NestJS + TypeScript | 模块化单体，与作业郎一致，团队可复用经验 |
| **数据库** | PostgreSQL 16 + PostGIS | 关系型 + 地理空间查询（圣地GPS） |
| **ORM** | Prisma | 类型安全、自动迁移 |
| **缓存/队列** | Redis + BullMQ | 会话、缓存、异步任务 |
| **对象存储** | 阿里云OSS / AWS S3 | 图片、音频等大文件 |
| **认证** | JWT + 微信OAuth + Google OAuth | 多端统一认证 |
| **国际化** | i18next | 12种语言 |
| **部署** | Docker Compose → K8s | 渐进式，先Docker后K8s |

## 核心数据模型（Prisma Schema）

```prisma
// === 用户体系 ===
model User {
  id            String    @id @default(cuid())
  phone         String?   @unique
  email         String?   @unique
  wechatOpenId  String?   @unique
  googleId      String?   @unique
  nickname      String
  avatar        String?
  role          UserRole  @default(PILGRIM)
  language      String    @default("zh-CN")
  createdAt     DateTime  @default(now())

  practices     Practice[]
  trips         Trip[]
  posts         Post[]
  journal       JournalEntry[]
}

enum UserRole {
  PILGRIM       // 朝圣者（普通用户）
  GUIDE         // 导游/志愿者
  AMBASSADOR    // 和平使者
  ADMIN         // 管理员
}

// === 宗教数据 ===
model Religion {
  id          String   @id @default(cuid())
  name        String   @unique          // "佛教"
  nameEn      String                    // "Buddhism"
  symbol      String?                   // 宗教符号
  color       String?                   // 主题色
  holySites   HolySite[]
  temples     Temple[]
  patriarchs  Patriarch[]
  teachings   Teaching[]
}

model HolySite {
  id            String   @id @default(cuid())
  name          String                  // "菩提伽耶"
  nameEn        String                  // "Bodh Gaya"
  country       String
  latitude      Float
  longitude     Float
  utcOffset     Float
  description   String   @db.Text
  imageUrl      String?
  soundEffect   String?
  religionId    String
  religion      Religion @relation(fields: [religionId], references: [id])

  teachings     SiteTeaching[]
  trips         TripSite[]
}

model Temple {
  id            String   @id @default(cuid())
  name          String
  nameEn        String?
  country       String
  foundingDate  String?
  description   String   @db.Text
  imageUrl      String?
  latitude      Float?
  longitude     Float?
  religionId    String
  religion      Religion @relation(fields: [religionId], references: [id])
}

model Patriarch {
  id            String   @id @default(cuid())
  name          String
  nameEn        String?
  dates         String?                 // "563 BCE - 483 BCE"
  title         String?
  biography     String   @db.Text
  coreTeaching  String   @db.Text
  imageUrl      String?
  religionId    String
  religion      Religion @relation(fields: [religionId], references: [id])
}

model Teaching {
  id            String   @id @default(cuid())
  name          String
  originalText  String   @db.Text       // 原文
  sourceText    String?
  translationCn String?  @db.Text       // 中文翻译
  religionId    String
  religion      Religion @relation(fields: [religionId], references: [id])
}

// === 三十印修炼 ===
model Seal {
  id        Int       @id                // 1-30
  name      String                       // "一众之誓印"
  series    SealSeries
  poem      String    @db.Text           // 偈语
  essence   String    @db.Text           // 印义
  practice  String    @db.Text           // 修行法
  vow       String    @db.Text           // 愿文
  color     String?                      // 系列色
}

enum SealSeries {
  CHUYIN      // 初印系 1-10
  ZHONGYIN    // 中印系 11-20
  YINGUOYIN   // 印果印 21-23
  CHENGDAOYIN // 成道印 24-26
  GUIYUANYIN  // 归源印 27-30
}

// === 修行记录 ===
model Practice {
  id        String   @id @default(cuid())
  userId    String
  user      User     @relation(fields: [userId], references: [id])
  sealId    Int
  date      DateTime @default(now())
  duration  Int?                         // 分钟
  note      String?  @db.Text
}

// === 旅行规划 ===
model Trip {
  id          String     @id @default(cuid())
  userId      String
  user        User       @relation(fields: [userId], references: [id])
  title       String
  startDate   DateTime?
  endDate     DateTime?
  status      TripStatus @default(PLANNING)
  sites       TripSite[]
  createdAt   DateTime   @default(now())
}

enum TripStatus {
  PLANNING
  CONFIRMED
  IN_PROGRESS
  COMPLETED
}

model TripSite {
  id        String   @id @default(cuid())
  tripId    String
  trip      Trip     @relation(fields: [tripId], references: [id])
  siteId    String
  site      HolySite @relation(fields: [siteId], references: [id])
  order     Int
  visitDate DateTime?
  notes     String?  @db.Text
}

// === 社区 ===
model Post {
  id        String   @id @default(cuid())
  userId    String
  user      User     @relation(fields: [userId], references: [id])
  content   String   @db.Text
  images    String[]
  siteTag   String?                      // 关联圣地
  likes     Int      @default(0)
  createdAt DateTime @default(now())
}
```

## 分阶段实施计划

### Phase 1: 基础设施 + 数据迁移（MVP-0）— 第1-2周
**目标**: 搭建monorepo骨架，迁移数据到PostgreSQL

**任务**:
1. 初始化 monorepo（Turborepo + pnpm）
2. 创建 `packages/shared-types/` — 定义所有TypeScript类型
3. 搭建 `services/api/` — NestJS项目 + Prisma Schema
4. Docker Compose（PostgreSQL + Redis）
5. 编写 `seed.ts` — 将 religions.py 的数据迁移为种子数据
6. 基础CRUD API（Religion、HolySite、Temple、Patriarch、Teaching、Seal）
7. 将现有 desktop/ 文件移入 `desktop/` 子目录

**交付物**: API可运行，Swagger文档可访问，数据已入库

---

### Phase 2: Web端 MVP — 第3-5周
**目标**: 用户可通过浏览器浏览圣地、了解祖庭

**任务**:

2. 首页（Landing Page）— 12大信仰展示 + 全球地图
3. 圣地浏览页 — 60圣地卡片列表 + 筛选
4. 圣地详情页 — 图文、GPS地图、天气、祖训
5. 祖庭浏览页 + 详情页
6. 祖师浏览页 + 详情页
7. 三十印修炼介绍页
8. `packages/api-client/` — API请求封装
9. 多语言框架（i18next，先中英双语）
10. 响应式设计（TailwindCSS）

**交付物**: 可公开访问的Web站点，展示所有宗教文化内容

## MVP（最小可行产品）定义

**MVP = Phase 1 + Phase 2**，约5周，交付：
- 可访问的Web站点展示12大信仰×60圣地×36祖庭×40祖师
- 全球地图上浏览圣地
- 后端API + PostgreSQL数据库
- 中英双语
- 响应式设计（手机浏览器也能用）

**为什么先做Web**: SEO传播最快，无需应用商店审核，响应式可临时替代APP。

## 关键架构决策

1. **共享逻辑不共享UI** — packages只放types/utils/hooks/api-client，各端UI独立实现
2. **保留桌面应用** — 现有Python桌面端移入desktop/子目录，继续独立运行
3. **NestJS模块化单体** — 先不微服务，等用户量上来再拆
4. **PostGIS地理查询** — "附近圣地"功能天然支持
5. **Taro而非uni-app** — 中大型项目Taro更成熟，且支持React语法统一技术栈
6. **渐进式国际化** — MVP先中英，后续逐步加12种语言
