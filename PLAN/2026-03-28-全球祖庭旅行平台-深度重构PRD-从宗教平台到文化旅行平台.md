# 全球祖庭旅行平台 — 深度重构PRD

> **版本**: v2.0 | **日期**: 2026-03-28 | **代号**: Phoenix (涅槃重生)
> **核心转型**: 宗教修行平台 → **国际级文化旅行平台**
> **竞品参考**: 携程/去哪儿/飞猪/马蜂窝/穷游/TripAdvisor
> **差异化**: 以祖庭和圣地为核心IP的深度文化路线旅行 + AI智能规划

---

## 一、战略定位重构

### 1.1 核心问题

当前平台存在**致命定位偏差**：

| 维度 | 当前状态 | 目标状态 |
|------|---------|---------|
| 平台定性 | 宗教修行平台 | **文化旅行平台** |
| 首页感受 | 佛殿氛围、宗教符号满天飞 | 旅行灵感、目的地探索 |
| 核心动作 | "修行"、"朝圣" | **"出发"、"探索"、"体验"** |
| 内容调性 | "诸恶莫作，众善奉行" | "千年智慧，一路同行" |
| 用户画像 | 佛教徒/修行者 | **全球文化旅行爱好者** |
| 商业模式 | 无（空壳） | 路线预订+文化体验+导游服务 |
| 修行模块 | 首屏Tab核心入口 | **注册用户可选功能，后期定向开放** |

### 1.2 新定位一句话

> **"走祖庭，看世界"** — 全球首个以宗教祖庭和文化圣地为IP的深度路线旅行平台

### 1.3 差异化价值

| 携程/去哪儿 | 马蜂窝/穷游 | **祖庭旅行(我们)** |
|------------|-----------|-----------------|
| 机酒预订为核心 | UGC攻略为核心 | **文化路线为核心** |
| 价格驱动 | 内容驱动 | **IP驱动(祖庭+圣地)** |
| 标准化产品 | 自由行攻略 | **主题深度路线** |
| 通用AI客服 | 无AI | **AI文化导游+智能路线规划** |

#### 路线数据模型 (新增)

```prisma
model Route {
  id          String   @id @default(cuid())
  slug        String   @unique
  title       String          // 六祖慧能路线
  titleEn     String          // Sixth Patriarch Huineng Route
  subtitle    String          // "追随六祖足迹，体验禅宗精髓"
  category    RouteCategory   // ZEN, BUDDHIST, TAOIST, CHRISTIAN, ISLAMIC, CROSS_CULTURAL
  difficulty  RouteDifficulty // EASY, MODERATE, CHALLENGING
  duration    Int             // 天数
  nights      Int             // 住宿晚数
  coverImage  String          // 封面图
  images      String[]        // 图片集
  highlights  String[]        // 亮点标签: ["禅宗文化","素斋体验","山居生活"]
  description String          // 详细介绍(Markdown)
  itinerary   Json            // 逐日行程JSON [{day:1, title:"抵达韶关", sites:[], meals:[], accommodation:""}]
  priceFrom   Int             // 起价(分) - 展示用
  included    String[]        // 包含: ["门票","导游","交通"]
  excluded    String[]        // 不含: ["机票","个人消费"]
  tips        String[]        // 贴士: ["建议穿舒适步行鞋"]
  season      String          // 最佳季节: "春秋两季(3-5月/9-11月)"
  groupSize   String          // 建议人数: "2-8人小团"
  status      RouteStatus     // DRAFT, PUBLISHED, ARCHIVED
  rating      Float?          // 评分
  reviewCount Int      @default(0)
  bookCount   Int      @default(0)

  // 关联
  religionId  String?
  religion    Religion? @relation(fields: [religionId], references: [id])
  sites       RouteSite[]     // 路线包含的圣地(有序)
  reviews     Review[]

  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt
}

model RouteSite {
  id       String   @id @default(cuid())
  routeId  String
  route    Route    @relation(fields: [routeId], references: [id])
  siteId   String
  site     HolySite @relation(fields: [siteId], references: [id])
  day      Int      // 第几天到达
  order    Int      // 当天顺序
  duration String?  // 建议停留时间 "2小时"
  note     String?  // 该站点特别说明
}

enum RouteCategory {
  ZEN
  BUDDHIST
  TAOIST
  CHRISTIAN
  ISLAMIC
  HINDU
  JEWISH
  CROSS_CULTURAL
  CULTURAL_HERITAGE
}

enum RouteDifficulty {
  EASY        // 城市为主，交通便利
  MODERATE    // 有山路，需一定体力
  CHALLENGING // 高原/偏远，需充分准备
}

enum RouteStatus {
  DRAFT
  PUBLISHED
  ARCHIVED
}
```

### 2.3 周边文化生态

每个圣地/祖庭不再只是一个"宗教景点"，而是一个**完整的旅行目的地**：

```
圣地/祖庭详情页结构:
├── 基本信息 (名称、位置、GPS、简介)
├── 文化故事 (历史、人物、典故) — 替代原"宗教教义"
├── 参观攻略 (开放时间、门票、交通、最佳季节)
├── 周边美食推荐 (素斋、特色小吃、当地名菜)
├── 住宿推荐 (寺院民宿、周边酒店、禅修营)
├── 文化体验 (抄经、禅坐、茶道、素食烹饪)
├── 所属路线 (包含此站点的推荐路线)
├── 游客评价 (真实游客点评+照片)
└── 周边景点 (非宗教景点推荐)
```

## 四、AI规划师深度设计

### 4.1 从"问答助手"到"旅行规划师"

**旧小鸿**: 问一答一的百科问答
**新小鸿**: 对话式路线规划 + 智能推荐 + 行程定制

#### 核心对话场景

```
场景1: 路线推荐
用户: "我想走一条禅宗路线，3-5天，预算5000以内"
小鸿: "推荐「六祖慧能路线」5天4晚，走访新兴国恩寺→韶关南华寺→广州光孝寺，
       起价¥3,280/人，含门票+导游+交通。要看详细行程吗？"
       [查看路线详情] [调整条件] [其他推荐]

场景2: 目的地咨询
用户: "南华寺什么季节去最好？附近有什么好吃的？"
小鸿: "南华寺四季皆宜，秋季(9-11月)最佳。推荐：
       🍜 寺内素斋(午餐¥30) — 百年传承，必尝
       🥘 韶关烧鹅 — 「东街烧鹅」步行15分钟
       🫖 南华禅茶 — 寺内禅茶室，免费品茗
       要帮你规划包含南华寺的路线吗？"

场景3: 行程定制
用户: "帮我规划一个7天的跨文化路线，从西安到伊斯坦布尔"
小鸿: "为您定制「丝路文明之旅」7天行程：
       Day1: 西安 — 大雁塔+碑林(佛教/儒学)
       Day2: 西安 — 化觉巷清真大寺(伊斯兰)
       Day3: 飞往乌鲁木齐 — 艾提尕尔清真寺
       Day4: 喀什古城 — 丝路文化融合
       Day5: 飞往伊斯坦布尔
       Day6: 圣索菲亚大教堂(基督教→伊斯兰)
       Day7: 蓝色清真寺+大巴扎
       预算约¥15,000/人。要调整吗？"
       [保存为行程] [修改天数] [更换目的地]
```

## 十、风险与原则

### 红线原则
1. **不删任何现有数据** — 12大信仰/60圣地/27祖庭等数据全部保留，只改展示方式
2. **不破坏现有功能** — 行程/订单/AI聊天等已验证功能保留并增强
3. **修行模块不删除** — 仅从首屏导航移除，注册用户个人中心仍可访问
4. **渐进式迁移** — 旧URL做301重定向，不造成SEO断裂

### 已知风险
1. **路线数据冷启动** — 初期路线少，需要运营快速填充内容
2. **视觉大改** — 从深色到亮色是颠覆性变化，需要CEO确认方向
3. **i18n工作量大** — 所有新文案需要多语种翻译
4. **周边信息缺失** — 美食/住宿等需要真实数据采集
